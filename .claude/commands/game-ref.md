레퍼런스 게임을 분석하여 구조화된 보고서를 작성합니다.

Arguments: $ARGUMENTS

## Instructions

### Phase 0: 인자 파싱

$ARGUMENTS에서 게임 제목과 선택적 시스템명을 파싱합니다.

**종합 분석 모드**: `/game-ref 하데스` → 게임 전체 분석
**시스템 집중 모드**: `/game-ref 하데스 전투시스템` → 특정 시스템 깊이 분석

인자가 비어있으면 사용자에게 게임 제목을 물어보세요.

### Phase 1: 웹 검색으로 정보 수집

다음 검색을 병렬로 수행하세요 (WebSearch 도구 사용):

1. `"<게임명> game mechanics systems gameplay"` — 핵심 메커니즘
2. `"<게임명> review analysis game design"` — 리뷰/분석
3. `"<게임명> art style visual sound design"` — 아트/사운드

검색 결과에서 유용한 페이지를 WebFetch로 읽어 상세 정보를 수집합니다.
참고할 소스: Steam 페이지, 나무위키, 게임 위키, GDC 발표, 유튜브 리뷰

한글 제목이면 영문명도 함께 검색하세요.

### Phase 2: 사용자 보충 질문

웹 검색 결과를 간단히 요약하고, 부족한 부분에 대해 사용자에게 질문합니다:
- "이 게임을 직접 플레이하셨나요? 인상 깊었던 시스템이 있나요?"
- "Project_Sun에 특히 참고하고 싶은 요소가 있나요?"
- 시스템 집중 모드라면 해당 시스템에 대한 구체적 질문

### Phase 3: 보고서 작성

수집된 정보를 아래 템플릿으로 구조화합니다.

**파일 경로**: `Docs/Study/ReferenceGames/<Title>-YYYY-MM-DD.md`
- Title은 영문명 사용 (한글 게임이면 영문명으로 변환)
- 예: `Hades-2026-03-24.md`, `Zelda-TotK-2026-03-24.md`

#### 시스템 집중 모드:

위 템플릿에서:
- 섹션 1은 간략하게 (핵심 컨셉 2-3줄)
- 섹션 2는 지정된 시스템만 매우 상세하게 분석:
  - 서브 메커니즘 세분화
  - 수치/밸런스 분석
  - 상태 다이어그램
  - 구현 시 필요한 컴포넌트 구조 제안
- 섹션 3은 해당 시스템 관련 비주얼/사운드만
- 섹션 5는 해당 시스템의 적용 포인트만

### Phase 4: 저장 및 안내

1. 보고서를 `Docs/Study/ReferenceGames/<Title>-YYYY-MM-DD.md`에 저장
2. 사용자에게 저장 경로와 핵심 적용 포인트 요약 전달
3. 관련 시스템 기획이 필요하면 `/시스템기획`을 안내
4. 바로 구현하고 싶다면 `/구현`을 안내

## Examples
- `/game-ref 하데스` — 하데스 종합 분석
- `/game-ref 젤다 왕국의 눈물` — 젤다 TotK 종합 분석
- `/game-ref 하데스 전투시스템` — 하데스 전투 시스템 집중 분석
- `/game-ref Hollow Knight 맵디자인` — 할로우 나이트 맵 디자인 집중 분석
