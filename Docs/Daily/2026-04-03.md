 1. Vision.md (v2.6) — 제작 방향성

  한줄 요약: 낮에 내정과 탐사로 대비하고, 밤에 압도적 물량 공세를 가벼운 전술 조작으로 막아내는 턴제 기지 경영 + 실시간 타워디펜스

  장르/플랫폼: Base Building + Tower Defense + Turn-Based (3태그 화이트 스페이스, Steam 20~40개 수준) / PC Steam $14.99~$19.99

  코어 루프 (1턴 = 하루):
  - 낮(턴제, 무제한 시간): 건설 탭 → 관리 탭 → 탐험 탭 → 인카운터 이벤트
  - 밤(실시간, 일시정지/1x/2x): 타워디펜스 + 분대(3~5개) 이동/공격/정지 마이크로
  - 결과 판정: 보상 or 건물 손상/시민 부상 → 다음 낮

  핵심 설계 원칙:
  - 복잡도 4~4.5/10 — 건설은 Thronefall 미니멀, 인력은 디아블로 소켓식 직관적 깊이
  - 타워 70~80% + 분대 20~30% — 마이크로 없이도 전투 진행, 일시정지 보장
  - 건물 10종 + 본부(특수), 시민 5~15명, 자원 3종(기초/고급/유물)
  - 패배 조건: 본부 파괴 = 게임오버 (점진적 패배, 시민 이탈 없음)

  메타 구조: 옴니버스 기지(세션 1~2시간) → 시대(3~5기지) → 전체 서사. 텍스트 콜백(MVP) → 초기 조건 변형 → 공유 NPC 순서로 단계적 구현.

  타겟 유저: Mastery(Strategy) + Achievement(Completion). 28~38세 PC 전략 게임 경험 10년+.

  7. Exploration.md (v0.2) — 탐사/원정 시스템

  핵심: 노드 그래프 맵(20~40노드/기지)에서 원정대 2~3명 파견. 핵심 비용 = 인력의 부재.

  노드 유형 6종: 자원(40~50%), 생존자(15~20%), 기술(10~15%), 유물(10~15%), 정찰(5~10%), 적 웨이브(5~10%)

  위험도 3단계: 안전(부상 0%, x1.0) → 보통(10~20%, x1.5) → 위험(30~50%, x2.0)

  이동 모델: 직선 이동(인접 노드만), 이동 1턴 + 탐사 1~3턴 + 귀환 1턴. 귀환 시 인접 미발견 노드 안개 해제.

  인카운터: 노드 도달 시 중요 인카운터(Scythe식 3선택지) 발생. 원정 중 랜덤 인카운터(이진 선택, 20~50종) 확률 발생.

  정찰 효과: 탐사 수준에 따라 밤 웨이브 미리보기의 정보 상세도 증가 (적 방향만 → 구성 비율 → 특수 적 유무).
