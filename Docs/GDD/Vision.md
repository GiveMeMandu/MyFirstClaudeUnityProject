## 1. 핵심 컨셉

### 1.1 한줄 요약
> 내정과 탐사로 대비하여 압도적 물량 공세를 막아내는 턴제 기지 경영 게임

### 1.2 엘리베이터 피치
플레이어는 허름한 기지의 지휘관이 되어, 낮 동안 건물을 짓고 인력을 배치하며 원정대를 보내 정보를 수집한다. 밤이 되면 어마어마한 물량의 적이 밀려오고, 낮 동안의 준비가 생존을 결정한다. 매 턴의 소소한 선택들이 쌓여 결정적 순간의 쾌감을 만들어내며, "한 턴만 더"를 멈출 수 없게 만든다. 각 기지는 독립된 이야기를 가지지만, 모든 기지의 이야기가 하나의 시대를 관통하는 서사로 연결된다.

### 1.3 장르 포지셔닝
| 항목 | 내용 |
|---|---|
| 메인 장르 | 턴제 기지 경영 시뮬레이션 |
| 서브 장르 | 타워 디펜스 + 탐험 |
| 레퍼런스 게임 | Frostpunk, Thronefall, They Are Billions, Reigns, Civ5 |
| 차별점 | 내정-탐사-방어 3기둥 턴제 루프 + 옴니버스 기지 메타구조 + 행정관 기반 정보 접근 단계 |

### 1.4 타겟 유저
- **코어 타겟**: 타워 디펜스와 4X 게임을 좋아하면서, 지나치게 복잡한 게임에 피로감을 느낀 유저
- **확장 타겟**: Frostpunk/Thronefall 팬, 캐주얼 경영 시뮬 유저
- **타겟이 즐기는 게임**: Frostpunk, Thronefall, They Are Billions, Civilization, Reigns
- **플랫폼**: PC (Windows) / Steam
- **상용화 모델**: 패키지 판매 (프리미엄)

## 2. 핵심 재미 이론

### 2.1 단기적 재미 (30초~5분)
즉각적으로 느끼는 재미 요소 — **호기심과 선택의 긴장감**
- **선택의 고통**: 건물 슬롯에 무엇을 배치할지, 인력을 어디에 할당할지, 매 순간의 의사결정이 "사람을 미치게" 만드는 쾌감
- **간결한 인카운터**: Reigns 스타일의 빈번한 이진 의사결정으로 리듬감 유지
- **건물 배치 피드백**: 스케일 애니메이션 + 파티클 + 사운드로 한 클릭의 권능감 전달
- **정보 발견**: 탐사/정찰 결과로 새로운 정보가 드러날 때의 호기심 충족

### 2.2 중기적 재미 (5분~1시간)
한 세션에서 느끼는 성장과 전략 검증 — **안도감과 뿌듯함**
- **점진적 해금**: 관리 탭 → 탐험 탭 → 방벽 확장 순서로 기능이 열리며 가능성 확대
- **방벽 확장 = 영역 + 기능**: 기지의 물리적 성장이 곧 새로운 시스템 해금
- **전략 검증**: 낮의 준비가 밤의 결과로 입증될 때의 "내 선택이 맞았다!" 뿌듯함
- **물량 방어 카타르시스**: 어마무시한 적 물량을 막아냈을 때의 안도감과 성취감

### 2.3 장기적 재미 (1시간~)
반복 플레이, 리텐션 요인 — **호기심과 마스터리**
- **옴니버스 내러티브**: 각 기지 세션의 이야기가 연결되어 전체 시대의 서사를 형성
- **시대 전환**: 한 시대의 기지들을 모두 클리어하면 다음 시대로 → 쇠퇴한 기지, 기술 발전 등 변화
- **최적화 탐색**: 더 매끄러운 운영을 시도하려는 동기 (복잡도 4이지만 최적화 여지 존재)
- **다른 선택 시도**: 패배 시 다른 기술 트리/정책 선택으로 재도전
- **기지별 고유 기술 트리**: 매 기지마다 다른 전략을 요구

### 2.4 재미의 우선순위
| 순위 | 재미 요소 | 포기 가능 여부 |
|---|---|---|
| 1 | 선택의 긴장감 (매 턴 유의미한 의사결정) | 절대 불가 |
| 2 | 물량 방어 카타르시스 (DOTS 대규모 전투) | 절대 불가 |
| 3 | "한 턴만 더" 중독성 (다음 턴에 대한 호기심) | 절대 불가 |
| 4 | 옴니버스 내러티브 연결 | 상황에 따라 축소 가능 |
| 5 | 시대 전환 메타 구조 | 필요시 후순위 |

## 3. 디자인 필터 (판단 기준)

### 3.1 수박 껍질 이론 (진입 장벽)
> "유저가 핵심 재미에 도달하기 전에 이탈하게 만드는 장벽은 무엇이고, 어떻게 최소화할 것인가?"

- **허용 가능한 장벽**: 건물/직책 슬롯에 시민 배치, 탐험대 구성, 방어 타워 배치 등 인력 배분의 복잡도
- **제거할 장벽**: 초반의 과도한 정보 노출, 복잡한 자원 관리, RTS식 마이크로 컨트롤
- **장벽 해소 전략**:
  - **행정관 능력 기반 정보 접근 단계**: Lv0(좋음/보통/나쁨) → Lv1(숫자) → Lv2(적성 정보) — 게임이 진행될수록 정보가 정교해지며 자연스럽게 복잡도 학습
  - **탭 점진적 해금**: 1일차 건설만 → 2일차 관리 탭 → 이후 탐험 탭
  - **QoL은 시민 능력으로 해금**: 원정대 필터링/정렬 등이 행정 능력치에 연동

### 3.2 결핍-충족 이론
> "유저에게 무엇이 부족하다고 느끼게 하고, 그것을 어떻게 채워주는가?"

- **의도적 결핍**:
  - 인력 부족: 내정/탐사/방어 모두 사람이 필요하지만 항상 부족
  - 정보 부족: 적 구성을 완전히 알 수 없음 (탐사로 부분 해소)
  - 자원 제한: 간결한 자원(~3종)이지만 항상 선택을 강제
- **충족 수단**:
  - 방어 성공 보상으로 자원/인력 획득
  - 탐사로 기술 해금/적 정보 획득
  - 방벽 확장으로 새로운 기능/슬롯 개방
- **충족 주기**: 매 턴(낮→밤→보상) 단위의 빠른 피드백 루프

### 3.3 기능 판단 필터
새로운 기능/시스템을 추가할 때 다음 질문에 답해야 함:

| # | 질문 | 기준 |
|---|---|---|
| 1 | 핵심 재미를 강화하는가? | "예"가 아니면 보류 |
| 2 | 진입 장벽을 높이지 않는가? | 높인다면 행정관 정보 단계 등 해소 방안 필수 |
| 3 | 내정-탐사-방어 3기둥 중 하나에 속하는가? | 속하지 않으면 우선순위 하향 |
| 4 | 구현 비용 대비 플레이어 경험 가치는? | 저가치라면 후순위 |
| 5 | 빼도 게임이 성립하는가? | "예"라면 코어가 아님 |

## 4. 핵심 플레이 루프

### 4.1 매크로 루프 (전체 게임)
```
[시대 1]
  ├─ 기지 A (세션 1) ─ 클리어
  ├─ 기지 B (세션 2) ─ 클리어   ← 옴니버스: 각 기지 독립 서사, 전체 연결
  └─ 기지 C (세션 3) ─ 클리어
        ↓
[시대 2] ─ 기술 발전/쇠퇴 반영
  ├─ 기지 D ...
  └─ ...
```

### 4.2 마이크로 루프 (하루 = 1턴)
```
[낮 페이즈]                              [밤 페이즈]
┌─────────────────────────────┐      ┌─────────────────────┐
│  건설 탭: 슬롯 건설/업그레이드    │      │  자동 전투           │
│  관리 탭: 인력 배치, 연구, 정책   │  →   │  DOTS 웨이브 공세     │
│  탐험 탭: 원정대 편성/파견       │      │  타워 디펜스          │
│  인카운터: 간결한 이진 선택       │      │  (플레이어 간섭 보류)  │
└─────────────────────────────┘      └─────────────────────┘
                                              ↓
                                     [전투 보상 획득]
                                     [다음 낮으로 →]
```

### 4.3 루프 간 연결
- **탐사 → 내정**: 원정 결과로 기술/정책 해금 방향 결정, 적 정보로 방어 배치 계획
- **내정 → 방어**: 건물 배치/업그레이드/인력 할당이 밤 전투력 직결
- **방어 → 다음 턴**: 전투 보상(자원/인력)이 다음 턴의 내정/탐사 자원
- **방벽 확장**: 자원 투자 → 기지 영역 확대 → 새 슬롯/기능 해금 → 성장 가속

### 4.4 인카운터 이원화
| 유형 | 빈도 | 형식 | 적용 |
|---|---|---|---|
| 일상 인카운터 | 빈번 (매 턴) | Reigns식 이진 선택 | 소소한 변수, 리듬감 |
| 중요 인카운터 | 탐사/특정 조건 | Scythe식 3선택지 (+조건부 4번째) | 방향성 결정, 큰 보상/대가 |

## 6. 아트/사운드 방향성

### 6.1 비주얼 톤
- **렌더링 스타일**: 스타일라이즈드 (Thronefall 참고 — 로우폴리/셀셰이딩 계열, 개발 효율 + 시각적 매력 양립)
- **색감/톤**: 낮은 따뜻한 톤(황금빛) → 밤은 차가운 톤(청색/보라) — Thronefall의 낮밤 대비 참고
- **레퍼런스 이미지**: Thronefall(색감 전환), Frostpunk(산업 미학), 반지의 제왕(방어전 스케일감)

### 6.2 UI/UX 방향
- **정보 밀도**: 중간 — 너무 미니멀하지 않되(관리하는 느낌 필수), 행정관 정보 단계로 점진적 노출
- **인터랙션 스타일**: 탭 기반 전환(건설/관리/탐험), 건물 배치 시 즉각적 피드백
- **핵심 원칙**: 초반 단순 → 점진적 복잡도 — 행정관 능력이 UI 복잡도를 제어

### 6.3 사운드 방향
- **BGM 톤**: 낮은 평온/희망적, 밤은 긴박/위협적 — 페이즈 전환 시 음악 전환으로 감정 앵커
- **효과음 원칙**: 건물 배치의 "쿵" 권능감, 적 물량의 압도적 사운드스케이프, 방어 성공의 안도 팡파르
- **Thronefall 참고**: 경고 호른(밤 시작) + 안도 팡파르(밤 종료)
