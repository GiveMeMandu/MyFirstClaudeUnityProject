## 1. 개요

### 목적
기지 운영의 방향성을 결정하는 비가역적 선택 시스템. 내정/탐사/방어 3개 카테고리에 걸친 정책 트리를 통해 플레이어에게 장기적 전략 분기를 제공하며, 선택의 무게감과 리플레이 가치를 창출한다.

### 핵심 경험
- **비가역적 선택의 무게감**: 일부 정책은 한쪽을 선택하면 반대쪽이 영구 잠김 — "이 결정이 이후 턴에 어떤 영향을 미칠지"의 긴장감
- **기지 정체성 형성**: 같은 기지라도 정책 조합에 따라 완전히 다른 운영 스타일
- **점진적 권한 확장**: 턴이 진행될수록 새로운 정책 노드가 해금되며 기지의 방향성이 구체화
- **트레이드오프 체감**: 무료지만 비가역 — 자원 비용 대신 기회비용으로 무게감 전달

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 원형 정책 트리 (Radial Policy Tree)

3등분된 원형 UI에서 내정/탐사/방어 카테고리별 정책 트리를 관리:

```
         [탐사]
        /      \
       /   🔍   \
      /          \
[내정] ── ⬡ ── [방어]
      \   🏛️   /
       \      /
        \ 🛡️/
```

- **내정 (Domestic)**: 자원 생산, 건설, 인력 관련 정책
- **탐사 (Exploration)**: 원정, 정보 수집, 발견 관련 정책
- **방어 (Defense)**: 전투, 타워, 방벽, 병력 관련 정책

각 카테고리는 독립적으로 진행되며, 카테고리 내에서만 분기/잠김이 발생한다.

#### 2.2 노드 유형

| 유형 | 설명 | 비율 (PoC) |
|---|---|---|
| **일반 노드** | 선택 시 효과 부여, 잠김 없음. 해금만 되면 언제든 제정 가능 | 약 절반 |
| **분기 노드** | A vs B 이진 선택. 한쪽 제정 시 반대쪽 영구 잠김 | 약 절반 (카테고리당 1~2개) |

#### 2.3 해금 조건

- **턴 수 기반**: 특정 턴에 도달하면 새로운 정책 노드가 해금
- **이전 노드 제정**: 트리 구조상 선행 노드를 제정해야 후속 노드 해금
- 두 조건이 AND로 결합 (턴 도달 + 선행 노드 제정)

```
턴 1: 카테고리별 루트 노드 해금 (3개)
턴 3: 각 카테고리 2단계 노드 해금 (루트 제정 시)
턴 5: 각 카테고리 3단계 노드 해금 (2단계 제정 시)
```

#### 2.4 제정 비용

- **무료**: 정책 제정에 자원 비용 없음
- **비용 = 비가역성**: 분기 노드에서의 영구 잠김이 실질적 비용
- 정책 효과 자체가 희망/불만을 변동시킬 수 있음 (제정 비용이 아닌 효과)

#### 2.5 정책 효과 유형

정책은 **영구 패시브 효과**로 적용 (한번 제정하면 기지 종료까지 유지):

| 효과 유형 | 설명 | 예시 |
|---|---|---|
| 자원 수정자 | 자원 생산/소모량 변동 | 기초 자원 생산 +20% |
| 인력 수정자 | 인력 효율/용량 변동 | 건물 배치 효율 +15% |
| 희망/불만 변동 | 즉발 또는 매 턴 희망/불만 변동 | 불만 -5 즉발, 매 턴 희망 +1 |
| 건설/기술 해금 | 특정 건물 또는 업그레이드 해금 | '감시탑 Lv3' 업그레이드 해금 |
| 전투 수정자 | 타워/방벽/민병대 성능 변동 | 타워 공격력 +10% |
| 인카운터 수정자 | 특정 인카운터 발생 조건 변동 | 중요 인카운터 추가 선택지 해금 |

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
턴 진행 (턴 수)          →     PolicyManager              →     새 노드 해금 알림
플레이어 노드 선택       →     (정책 제정 + 잠김 처리)    →     영구 효과 적용
정책 효과 조회 요청      →     PolicyEffectResolver        →     수정자 값 반환
                                                           →     희망/불만 변동
                                                           →     건설/기술 해금 알림
```

### 상태 다이어그램

```
[Locked] ──(턴 도달 + 선행 노드 제정)──→ [Unlocked]
                                              │
                                     플레이어 선택
                                              │
                                 ┌────────────┼────────────┐
                                 ▼            ▼            ▼
                           [Enacted]    [BranchLocked]  (일반 노드)
                           (제정됨)     (영구 잠김)    → [Enacted]
```

노드 상태:
- **Locked**: 해금 조건 미충족, 선택 불가
- **Unlocked**: 해금됨, 선택 가능
- **Enacted**: 제정됨, 효과 활성
- **BranchLocked**: 분기 반대편 선택으로 영구 잠김

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| 노드 해금 턴 간격 | 정책 선택 빈도 | 2~3턴 간격 |
| 자원 수정자 크기 | 정책 체감 강도 | ±10~30% |
| 인력 수정자 크기 | 운영 방식 변화 정도 | ±10~20% |
| 희망/불만 변동량 | 사회적 비용/보상 체감 | ±3~10 |
| 전투 수정자 크기 | 방어 전략 영향 | ±10~25% |

### 밸런스 기준

- **선택 빈도**: 2~3턴마다 새로운 정책 노드 해금 → "한 턴만 더"
- **체감 강도**: 정책 제정 즉시 눈에 띄는 변화 (최소 10% 수정자)
- **분기 균형**: A vs B 양쪽 모두 유효한 전략이어야 함 — 정답 없는 선택
- **카테고리 균형**: 3개 카테고리 모두 비슷한 수준의 영향력

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 선행 노드 미제정 시 후속 노드 | Locked 상태 유지, UI에서 선행 조건 표시 |
| 분기 노드 양쪽 모두 Locked 후 해금 | 양쪽 모두 Unlocked로 전환, 플레이어 선택 대기 |
| 모든 정책 제정 완료 | 정책 UI 비활성화, "모든 정책 제정 완료" 알림 |
| 정책 효과 대상 시스템 미구현 | 효과 무시 + 경고 로그, 시스템 구현 후 자동 적용 |
| 게임 중간 로드 시 정책 상태 | PolicyManager가 제정된 노드 목록 직렬화/역직렬화 |
| 같은 효과 유형의 다중 정책 | 수정자는 가산 (Additive) — 예: +10% + +20% = +30% |
| 분기 노드의 후속 노드 | 분기 잠김 시 해당 분기의 후속 노드도 모두 BranchLocked |

## 7. PoC 정책 노드 설계

### 내정 (Domestic) 카테고리

```
[배급 정책] ──→ [노동 기준] ──→ [생산 방침]
 (일반)         (분기)          (일반)
```

| 노드 | 유형 | 해금 턴 | 효과 |
|---|---|---|---|
| 배급 정책: 균등 배급 | 일반 | 1 | 기초 자원 생산 +15%, 희망 +3 |
| 노동 기준: 표준 근무 | 분기 A | 3 | 건물 배치 효율 +20%, 불만 -2/턴 |
| 노동 기준: 연장 근무 | 분기 B | 3 | 기초 자원 생산 +30%, 불만 +3/턴 |
| 생산 방침: 자급자족 | 일반 | 5 | 고급 자원 생산 +10% |

### 탐사 (Exploration) 카테고리

```
[정찰 규정] ──→ [원정 방침] ──→ [정보 활용]
  (일반)         (분기)          (일반)
```

| 노드 | 유형 | 해금 턴 | 효과 |
|---|---|---|---|
| 정찰 규정: 기본 정찰 | 일반 | 1 | 인카운터 발생 확률 -10%, 탐사 보상 +15% |
| 원정 방침: 안전 우선 | 분기 A | 3 | 원정대 피해 -30%, 탐사 속도 -20% |
| 원정 방침: 속도 우선 | 분기 B | 3 | 탐사 속도 +30%, 원정대 피해 +20% |
| 정보 활용: 전술 공유 | 일반 | 5 | 타워 공격력 +10% (탐사→방어 시너지) |

### 방어 (Defense) 카테고리

```
[방어 태세] ──→ [민병대 정책] ──→ [요새화]
  (일반)          (분기)           (일반)
```

| 노드 | 유형 | 해금 턴 | 효과 |
|---|---|---|---|
| 방어 태세: 경계 강화 | 일반 | 1 | 타워 사거리 +15%, 방어 자원 소모 -10% |
| 민병대 정책: 자원 민병 | 분기 A | 3 | 방어 시 자원 소모 -25%, 인력 효율 -10% |
| 민병대 정책: 징집 민병 | 분기 B | 3 | 인력 효율 +15%, 불만 +2/턴, 타워 데미지 +20% |
| 요새화: 방벽 보강 | 일반 | 5 | 방벽 HP +25%, 방벽 수리 비용 -15% |
