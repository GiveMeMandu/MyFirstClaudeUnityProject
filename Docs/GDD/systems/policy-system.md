# 정책/법률 시스템

- **작성일**: 2026-03-26
- **상태**: 기획 완료
- **slug**: policy-system

---

## 1. 개요

### 목적
일정 턴 주기마다 이진 분기의 비가역적 정책 결정을 강제하여, 기지의 운영 방향을 결정하는 시스템. 한쪽을 선택하면 다른 쪽은 영구 차단되므로, 매 선택마다 긴장감과 무게감을 제공한다.

### 핵심 경험
- **결정의 무게감**: "이 정책을 선택하면 돌이킬 수 없다"는 비가역성에서 오는 긴장감
- **전략적 방향성**: 정책 선택이 쌓여 기지의 운영 스타일이 결정됨 (군사형/경제형/균형형 등)
- **파급 효과의 발견**: 초반 정책이 후반 정책 선택지에 영향 → "앞선 결정이 지금 여기서 이렇게 돌아오는구나"

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 정책 결정 포인트

기지당 5~10개의 정책 결정 포인트가 존재하며, N턴 간격으로 순차 발생한다.

| 속성 | 설명 | PoC 기본값 |
|---|---|---|
| 발생 주기 | N턴마다 정책 결정 이벤트 | 3턴마다 |
| 기지당 정책 수 | 시나리오 동안 총 결정 횟수 | 5~7개 |
| 첫 정책 턴 | 정책 시스템 시작 턴 | 3턴 |
| 결정 타이밍 | 낮 페이즈 시작 시 강제 팝업 | 낮 시작 직후 |

#### 2.2 정책 데이터 구조

각 정책은 이진 분기(A/B)로 구성된다.

| 정책 속성 | 설명 | 예시 |
|---|---|---|
| ID | 고유 식별자 | `policy_resource_priority` |
| 이름 | 정책 제목 | "자원 운영 방침" |
| 설명 | 정책 배경 상황 설명 | "기지의 자원이 부족해지고 있습니다..." |
| 분기 A 이름 | 선택지 A 제목 | "효율 우선" |
| 분기 A 설명 | 선택지 A 효과 설명 | "자원 생산 +25%, 건설 비용 +15%" |
| 분기 A 효과 | 영구 적용 효과 목록 | [{ProductionBonus, +0.25}, {BuildCostMod, +0.15}] |
| 분기 B 이름 | 선택지 B 제목 | "절약 우선" |
| 분기 B 설명 | 선택지 B 효과 설명 | "건설 비용 -20%, 자원 생산 -10%" |
| 분기 B 효과 | 영구 적용 효과 목록 | [{BuildCostMod, -0.20}, {ProductionBonus, -0.10}] |
| 선행 정책 | 이 정책이 나타나기 위한 이전 정책 선택 조건 | policy_01_A (1번 정책에서 A 선택 시) |
| 트리 레벨 | 정책 트리에서의 깊이 | 1, 2, 3... |

#### 2.3 정책 트리 구조

정책은 단순 선형이 아닌, 트리 구조로 분기한다. 이전 정책의 선택이 다음 정책의 선택지를 결정한다.

```
[정책 1] 자원 운영 방침
  ├── A: 효율 우선 ──→ [정책 2A] 노동 정책
  │                      ├── A: 장시간 노동 ──→ [정책 4A]...
  │                      └── B: 교대 근무 ──→ [정책 4B]...
  └── B: 절약 우선 ──→ [정책 2B] 비축 정책
                         ├── A: 전략 비축 ──→ [정책 4C]...
                         └── B: 균등 분배 ──→ [정책 4D]...

[정책 3] 방어 방침 (정책 1과 독립)
  ├── A: 군사 강화
  └── B: 수비 강화
```

- 일부 정책은 이전 정책과 독립적 (별도 트리 라인)
- 트리 깊이: 최대 3~4 레벨
- 동일 레벨의 독립 정책은 같은 턴에 발생하지 않음

#### 2.4 비가역성 규칙

| 규칙 | 설명 |
|---|---|
| 영구 선택 | A를 선택하면 B는 영구 차단. 번복 불가 |
| 영구 효과 | 선택한 분기의 효과는 시나리오 종료까지 유지 |
| 연쇄 차단 | B 분기에서만 도달 가능한 하위 정책도 모두 차단 |
| 정보 공개 | 선택 전 양쪽 효과를 모두 공개 (숨김 없음, 초보자 스트레스 완화) |
| 확인 단계 | 선택 후 "정말로 [A]를 선택하시겠습니까?" 확인 팝업 |

#### 2.5 정책 효과 유형

| 효과 유형 | 대상 시스템 | 예시 |
|---|---|---|
| ProductionBonus | 자원 시스템 | 전체 자원 생산량 ±N% |
| SpecificProductionBonus | 자원 시스템 | 특정 자원(기초/고급/방어) 생산량 ±N% |
| BuildCostMod | 건설 시스템 | 건설 비용 ±N% |
| DefensePowerMod | 방어 시스템 | 방어 건물 데미지 ±N% |
| DefenseHPMod | 방어 시스템 | 건물 HP ±N% |
| WorkerEfficiencyMod | 인력 시스템 | 인력 슬롯 효과 ±N% |
| HealingSpeedMod | 인력 시스템 | 부상 회복 속도 ±N턴 |
| ResearchSpeedMod | 기술 트리 | 연구 소요 턴 ±N턴 |
| TechUnlock | 기술 트리 | 특정 기술 노드 잠금 해제 |

#### 2.6 정책 결정 UI 흐름

1. 정책 발생 턴의 낮 시작 시 전체 화면 팝업 표시
2. 상단: 정책 제목 + 상황 설명
3. 좌측: 분기 A (이름, 설명, 효과 목록)
4. 우측: 분기 B (이름, 설명, 효과 목록)
5. 하단: "현재 적용 중인 정책 효과 요약" 표시
6. 선택 후 확인 팝업 → 확정 → 효과 즉시 적용
7. 정책 미결정 시 다른 행동 불가 (강제 결정)

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
턴 시작 이벤트          →     PolicyManager              →     정책 결정 팝업 표시
플레이어 분기 선택       →     (선택 기록 + 차단 처리)      →     비가역 선택 확정
선택 확정              →     (효과 적용)                 →     각 시스템에 보너스/패널티
                              (하위 정책 갱신)            →     다음 정책 선택지 결정
                              (기술 트리 연동)            →     잠금 해제 이벤트 발생
```

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| decisionInterval | 정책 결정 빈도 | 2~4턴 간격 |
| firstDecisionTurn | 초반 적응 시간 | 2~4턴 |
| 기지당 정책 수 | 결정의 총 횟수 | 5~10개 |
| 효과 수치 (보너스) | 정책 임팩트 | +10~+40% |
| 효과 수치 (패널티) | 트레이드오프 강도 | -5~-25% |
| 정책 트리 깊이 | 연쇄 분기 복잡도 | 2~4 레벨 |
| 독립 정책 라인 수 | 병렬 의사결정 축 | 2~3개 |

### 밸런스 기준

- **의미 있는 트레이드오프**: 양쪽 분기 모두 장단점이 명확. "정답"이 없어야 함
- **체감 가능한 변화**: 정책 선택 직후 다음 턴에 효과가 느껴져야 함 (최소 +15% 이상)
- **누적 효과 관리**: 같은 방향 정책이 누적되면 특화 강화, 반대 방향이면 균형 유지. 극단적 누적은 ±50% 이내
- **초보자 배려**: 양쪽 효과를 숫자까지 모두 공개. "선택 취소 불가" 경고를 확인 팝업에 명시
- **파급 효과의 깊이**: 1번 정책 선택이 3~4번 정책 선택지에까지 영향 → "내 선택이 미래를 바꿨다" 체감

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 정책 결정 턴에 전투 발생 | 낮 시작 시 정책 먼저 결정 → 이후 낮 행동 → 밤 전투. 정책 결정이 항상 선행 |
| 정책 미결정 상태에서 턴 종료 시도 | 턴 종료 차단. "정책 결정이 필요합니다" 토스트 표시 |
| 선행 정책이 아직 결정 안 된 하위 정책 | 해당 턴에 발생하지 않음. 선행 정책 결정 후 다음 발생 주기에 등장 |
| 정책 효과로 자원이 음수가 되는 경우 | 효과는 생산 배율이므로 보유량에 직접 영향 없음. 생산량은 최소 0 (음수 생산 불가) |
| 정책 효과가 0%로 만드는 경우 | 생산 배율 최소값 0.1 (90% 감소까지만 허용) |
| 모든 정책이 결정된 이후 | PolicyManager가 더 이상 팝업을 트리거하지 않음. 기존 효과만 유지 |
| 저장/로드 시 정책 상태 | decisions 딕셔너리 + activeEffects를 직렬화하여 저장 |
| 기지 전환 시 정책 | 기지별 독립 정책 트리. 각 기지의 결정 상태 개별 저장 |
| 정책 효과와 기술 효과 중첩 | 곱연산으로 중첩. 예: 정책 +25% * 기술 +20% = 총 +50% |
