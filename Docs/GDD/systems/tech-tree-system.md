## 1. 개요

### 목적
기지의 기술 발전 경로를 결정하는 전략 시스템. 카테고리별 DAG 기반 연구 트리에서 노드를 선택·연구하여 건물 업그레이드 분기, 숨겨진 슬롯 공개, 능력치 보너스, 새로운 기능을 해금한다. 연구 건물에 배치된 인력이 매 턴 진행도를 생산하며, 한 번에 하나의 연구만 진행 가능하다.

### 핵심 경험
- **전략적 고민**: "이걸 먼저 연구하면 더 유리할까?" — 제한된 턴에서 어떤 경로를 선택할지의 긴장감
- **장기 투자의 보상**: 여러 턴에 걸친 연구가 완료되었을 때의 성취감
- **탐색 욕구**: 잠긴 노드가 무엇인지 궁금하여 선행 연구를 서두르는 동기

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 연구 트리 구조
- **카테고리별 개별 DAG**: 연구 노드는 카테고리(예: 자원, 방어, 탐사 등)로 분류되며, 각 카테고리가 독립된 DAG 그래프를 가진다
- **노드 간 관계**: 분기(1→다수)와 합류(다수→1) 모두 가능. 선행 연구 복수 조건 지원
- **가시성**: 트리 전체 구조는 항상 보이나, 잠긴 노드는 불투명/잠금 아이콘으로 표시. 기본 노드(선행 조건 없음)는 처음부터 공개
- **PoC 범위**: 범용 트리 1개, 5~8개 노드

#### 2.2 연구 진행 흐름

```
[연구 건물 완성] → [연구 노드 선택] → [착수 비용 지불] → [연구 진행중]
                                                            │
                                                   매 턴 종료 시 진행도 증가
                                                   (인력 수 × 인당 생산량)
                                                            │
                                                   진행도 ≥ 필요량 → [완료]
                                                            │
                                                   다음 낮 시작 시 완료 알림
                                                            ↓
                                                   [효과 적용 + 후속 노드 해금]
```

#### 2.3 연구 전환
- 진행 중인 연구를 다른 노드로 전환 가능
- **전환 시 기존 노드의 진행도는 보존** (나중에 재개 가능)
- **착수 비용은 환불되지 않음** — 이미 지불했으므로 재개 시 추가 비용 없음
- 새 노드로 전환 시 해당 노드의 착수 비용을 지불해야 함 (최초 1회)

#### 2.4 연구 완료 효과
| 효과 종류 | 설명 |
|---|---|
| 건물 업그레이드 분기 해금 | 특정 건물의 새로운 업그레이드 경로 활성화 |
| 숨겨진 건물 슬롯 공개 | 맵에 이미 존재하는 숨겨진 슬롯을 드러냄 (위치는 레벨 디자이너가 사전 배치) |
| 능력치 보너스 | 글로벌 또는 특정 건물의 효율/스탯 증가 |
| 건물 내 슬롯 추가 | 특정 건물의 인력 슬롯 수 증가 |
| 새로운 기능 해금 | 새로운 시스템 기능 활성화 |

#### 2.5 연구 시작 조건
1. 연구 건물이 **완성** 상태여야 함 (건설중/파괴됨 불가)
2. 선행 연구가 **모두 완료**되어 있어야 함
3. 착수 비용을 지불할 **자원이 충분**해야 함

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
연구 노드 선택 (UI)      →     TechTreeManager            →     착수 비용 차감
턴 종료 트리거           →     (진행도 계산)               →     진행도 업데이트
다음 낮 시작 트리거      →     (완료 체크)                →     완료 알림 + 효과 적용
연구 전환 명령           →     (현재 연구 저장 + 새 연구)  →     연구 대상 변경
```

### 상태 다이어그램 (연구 노드)

```
[Locked] ── 선행 연구 완료 ──→ [Available]
                                    │
                              노드 선택 + 비용 지불
                                    │
                                    ▼
                              [InProgress] ←── 재개
                                    │              │
                               전환 (보존)          │
                                    │              │
                                    ▼              │
                              [Paused] ─────────────┘

                              [InProgress]
                                    │
                               진행도 충족
                                    │
                                    ▼
                              [Completed]
```

## 4. 밸런스 가이드

### 조정 가능 파라미터 (PoC 기준)

| 파라미터 | 위치 | PoC 기본값 | 조정 방법 |
|---|---|---|---|
| 노드별 필요 진행도 | TechNodeSO | 2~4 | Inspector |
| 인당 연구 포인트 | TechTreeManager | 1.0 | Inspector |
| 노드별 착수 비용 | TechNodeSO | 기초 5~15 | Inspector |

### 밸런스 기준 (PoC)
- **2턴마다 1개 연구 완료** 가능한 수준 (인력 1명 기준: 필요 진행도 2)
- 착수 비용은 건설 비용의 30~50% 수준
- 기본 노드 2~3개는 선행 조건 없이 즉시 연구 가능

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 연구 건물 파괴됨 (연구 진행 중) | 진행도 유지, 생산 중단. 건물 복구 후 자동 재개 |
| 연구 건물에 인력 0명 | 진행도 증가 없음 (0 × 생산량 = 0) |
| 모든 노드 연구 완료 | 연구 선택 불가 상태이나 트리 UI는 열람 가능 |
| 잠긴 노드 클릭 | 해금 조건(선행 연구 목록) 툴팁 표시, 연구 시작 불가 |
| 착수 비용 부족 | "자원 부족" 경고, 연구 시작 불가 |
| 연구 전환 후 재개 | 기존 진행도에서 이어서 진행, 추가 비용 없음 |
| 연구 건물 건설중/수리중 | 연구 진행 불가 (생산량 0), 완성 시 자동 재개 |
