# 기술 트리 시스템

- **작성일**: 2026-03-26
- **상태**: 기획 완료
- **slug**: tech-tree-system

---

## 1. 개요

### 목적
기지별 고유 기술 트리를 통해 건물 업그레이드, 생산 효율, 방어 강화 등을 연구하는 장기 성장 시스템. 탐사에서 발견한 설계도와 정책 선택이 특정 노드를 해금하므로, 매 세션마다 다른 전략적 경로를 강제한다.

### 핵심 경험
- **전략적 분기**: 제한된 턴 안에 모든 기술을 연구할 수 없음 → 어떤 경로를 선택할지 매 세션 고민
- **탐사의 보상**: 탐사에서 발견한 설계도가 잠긴 기술 노드를 해금 → 탐사에 투자하는 동기 부여
- **성장의 실감**: 연구 완료 시 즉각적인 효과 체감 (건물 업그레이드 해금, 생산량 증가 등)

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 기술 노드 구조

각 기술 노드는 DAG(Directed Acyclic Graph) 구조로 연결되며, 기지당 15~25개 노드로 구성된다.

| 노드 속성 | 설명 | 예시 |
|---|---|---|
| ID | 고유 식별자 | `tech_adv_mining` |
| 이름 | 표시 이름 | "고급 채굴 기법" |
| 설명 | 효과 설명 텍스트 | "Mine 생산 효율 +30%" |
| 카테고리 | 기술 분야 | 경제 / 방어 / 건설 |
| 연구 비용 | 기초/고급 자원 소모량 | 기초 80, 고급 40 |
| 연구 턴 수 | 연구 소요 턴 | 3턴 |
| 선행 기술 | 완료 필요한 기술 ID 목록 | [tech_basic_mining] |
| 잠금 조건 | 설계도 ID 또는 정책 ID | blueprint_03 또는 null |
| 해금 효과 | 연구 완료 시 적용 효과 | EffectType + 수치 |

#### 2.2 노드 상태

| 상태 | 조건 | UI 표시 |
|---|---|---|
| Locked | 잠금 조건 미충족 (설계도/정책 필요) | 자물쇠 아이콘, 회색 |
| Unavailable | 선행 기술 미완료 | 어둡게, 선행 기술 표시 |
| Available | 선행 기술 완료 + 잠금 해제 | 밝게, 연구 시작 가능 |
| Researching | 연구 진행 중 | 진행 바 표시 |
| Completed | 연구 완료 | 체크 표시, 효과 활성 |

#### 2.3 연구 프로세스

1. 플레이어가 연구소 건물에 인력을 배치
2. Available 상태의 기술 노드를 선택하여 연구 시작
3. 연구 시작 시 자원 비용 즉시 차감
4. 매 턴 종료 시 연구 진행도 1턴 증가 (연구소에 인력이 배치된 경우에만)
5. 연구 턴 수 충족 시 연구 완료 → 효과 즉시 적용
6. 동시 연구: 연구소 1개당 1개 기술만 연구 가능

| 연구 진행 조건 | 설명 |
|---|---|
| 연구소 건물 존재 | 연구소가 건설 완료 상태여야 함 |
| 인력 배치 | 연구소에 최소 1명 배치 |
| 연구 대상 선택 | Available 노드 중 하나 선택 상태 |

#### 2.4 잠금 해제 메커니즘

일부 기술 노드는 선행 기술 완료만으로는 연구할 수 없다. 추가 잠금 조건이 필요하다.

| 잠금 유형 | 해제 방법 | 예시 |
|---|---|---|
| 설계도 | 탐사에서 설계도 아이템 발견 | "고대 투석기 설계도" → 투석기 기술 해금 |
| 정책 | 특정 정책 분기 선택 | "군사 우선 정책" → 방어 기술 상위 노드 해금 |
| 없음 | 선행 기술만으로 연구 가능 | 대부분의 기본 노드 |

#### 2.5 해금 효과 유형

| 효과 유형 | 설명 | 예시 |
|---|---|---|
| BuildingUnlock | 새 건물 타입 해금 | 투석기 타워 건설 가능 |
| BuildingUpgrade | 기존 건물 업그레이드 해금 | Arrow Tower Lv.2 업그레이드 가능 |
| ProductionBonus | 자원 생산 효율 증가 | Mine 생산량 +30% |
| DefenseBonus | 방어 수치 강화 | 전체 타워 데미지 +15% |
| WorkforceBonus | 인력 효율 증가 | 건설 슬롯 효과 +20% |
| CostReduction | 건설/업그레이드 비용 절감 | 방어 건물 비용 -20% |

#### 2.6 기지별 고유 트리

| 속성 | 설명 |
|---|---|
| TechTreeDataSO | 기지별 기술 트리 정의 SO |
| 노드 목록 | 기지마다 다른 노드 구성 |
| 노드 위치 | UI 상 노드 배치 좌표 |
| 공통 노드 | 전 기지 공유 기본 노드 (5~8개) |
| 고유 노드 | 기지 특화 노드 (10~17개) |

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
연구 시작 명령           →     TechTreeManager            →     자원 차감 + 연구 상태 변경
턴 종료 트리거          →     (연구 진행도 갱신)           →     진행도 +1 또는 연구 완료
설계도 획득 (탐사)       →     (잠금 해제 검사)            →     Locked → Unavailable 전환
정책 선택 (정책 시스템)   →     (잠금 해제 검사)            →     Locked → Unavailable 전환
연구 완료              →     (효과 적용)                 →     보너스 활성화 / 건물 해금
```

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| 노드 연구 비용 (기초) | 연구 진입 문턱 | 30~150 |
| 노드 연구 비용 (고급) | 고급 자원 압박 | 10~80 |
| 노드 연구 턴 수 | 연구 속도 | 2~6턴 |
| 기지당 총 노드 수 | 트리 복잡도 | 15~25개 |
| 잠금 노드 비율 | 탐사/정책 연동 강도 | 전체의 20~40% |
| 공통 노드 수 | 기지 간 공유 기반 | 5~8개 |
| 효과 수치 (생산 보너스) | 연구 가치 체감 | +15~+40% |
| 효과 수치 (방어 보너스) | 방어 투자 가치 | +10~+30% |

### 밸런스 기준

- **연구 완료 페이스**: 20턴 시나리오에서 전체 노드의 40~60%만 연구 가능 → 매 세션 다른 경로 선택 강제
- **자원 압박**: 연구에 자원을 투입하면 건설/업그레이드가 지연 → 트레이드오프 체감
- **첫 연구 완료**: 3~4턴 차에 첫 기술 완료 → 초반 성장 체감
- **잠금 노드 가치**: 잠금 조건이 있는 노드는 일반 노드보다 효과가 1.5~2배 강력 → 탐사/정책 투자 동기
- **연구소 인력 기회비용**: 연구소에 인력 1명 = 자원 생산이나 방어에 못 넣는 비용 → 연구 타이밍 고민

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 연구 중 연구소 파괴 | 연구 진행 일시 정지. 연구소 재건 후 이어서 진행 (진행도 유지) |
| 연구 중 인력 회수 | 해당 턴 연구 진행 안 됨. 인력 재배치 시 다음 턴부터 진행 재개 |
| 자원 부족으로 연구 시작 불가 | "자원 부족" 토스트 메시지. 연구 시작 차단 |
| 선행 기술 미완료 상태에서 연구 시도 | UI에서 비활성화. 선행 기술 목록 툴팁 표시 |
| 잠금 조건 미충족 노드 | 자물쇠 아이콘 + "설계도 필요" 또는 "정책 필요" 텍스트 |
| 연구소 2개 이상 건설 | 각 연구소마다 독립적으로 1개씩 동시 연구 가능 |
| 연구 완료 시 효과 대상 건물 없음 | 효과는 등록만 해두고, 해당 건물 건설 시 자동 적용 |
| 기지 전환 시 기술 트리 | 기지별 독립 트리이므로 각 기지의 연구 상태 개별 저장 |
| 모든 선행 기술 완료 + 잠금 미해제 | Locked 상태 유지. 잠금 해제 조건 표시 |
