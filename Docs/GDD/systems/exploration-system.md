## 1. 개요

### 목적
내정-탐사-방어 3기둥 중 "탐사"를 담당하는 핵심 시스템. 원정대를 편성하여 노드 그래프 맵을 탐험하고, 자원 발견/적 정찰/인카운터/기술 해금 등의 보상을 획득한다. 탐사에 인력을 투입하면 기지 내정 인력이 줄어드는 트레이드오프를 통해 매 턴 전략적 선택을 강제한다.

### 핵심 경험
- **호기심**: 안개에 가려진 미지의 노드에 무엇이 있을지 궁금함
- **트레이드오프**: 원정대에 인력을 보내면 기지 내정/방어 인력이 감소
- **정보 우위**: 적 정찰로 밤 전투를 유리하게 대비
- **발견의 쾌감**: 자원 노드, 기술 해금 아이템 등 보상 획득 시 성취감

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 노드 그래프 맵

시나리오(기지)마다 **고정 배치**된 노드 그래프 맵이 존재한다. 레벨 디자이너가 노드 위치, 유형, 연결 관계를 SO에서 정의한다.

- 게임 시작 시 **안개(Fog)**로 가려져 있음
- 원정대가 노드에 도착하면 해당 노드 + **인접 노드**까지 안개 해제
- 방문 완료한 노드는 **비활성화** (재방문 불가)
- PoC 기준 노드 수: 약 10개/시나리오

> **확장 가능성**: 추후 상점 등 재방문 가능한 노드 유형 추가 예정. 노드 데이터에 `isRevisitable` 플래그를 포함하여 확장성 확보.

#### 2.2 노드 유형 (PoC)

| 유형 | 역할 | 도착 시 효과 |
|---|---|---|
| **자원 노드** (Resource) | 자원 획득 | 기초/고급/방어 자원 즉시 획득 |
| **정찰 노드** (Recon) | 적 웨이브 정보 공개 | N턴 후 웨이브 구성을 미리 확인 가능 |
| **인카운터 노드** (Encounter) | 중요 인카운터 발생 | 기존 EncounterManager를 통한 3선택지 인카운터 |
| **기술 노드** (Tech) | 기술 트리 해금 조건 충족 | 특정 기술 노드 해금 아이템 획득 (추후 연동) |

- 각 노드는 SO에서 유형과 보상 데이터를 정의
- 노드에 도착하기 전 **힌트 아이콘**이 표시됨 (유형은 알 수 있으나 구체적 보상은 미공개)

#### 2.3 원정대 편성

- **탐사 건물 1개 = 원정대 1팀** 추가 (최대 약 3팀)
- 각 원정대에 **가변 인원** 배치 (최소~최대 인원 범위, SO 정의)
- 원정대에 배치된 인력은 기지 내정 풀에서 차감 → **트레이드오프**
- 원정대 파견에 자원 비용 없음 (인력 할애 자체가 비용)

#### 2.4 이동 메커니즘

```
[낮 페이즈]
  플레이어: 탐사 탭 열기 → 노드 맵 전체 화면 표시
    ├── 원정대별 목적지 노드 지정 (인접 노드 중 선택)
    ├── 귀환 명령 (수동)
    └── 원정대 편성/해산

[턴 종료 시]
  원정대 이동 처리:
    ├── 목적지를 향해 1턴분 이동
    ├── 소요 턴수 ≤ 1 → 도착
    └── 소요 턴수 > 1 → 경로 상에서 진행 중

[다음 낮 시작 시 — 탐사 탭 열기]
  도착한 노드의 이벤트를 순차적으로 표시:
    ├── 자원 노드 → 자원 획득 알림
    ├── 정찰 노드 → 적 정보 공개
    ├── 인카운터 노드 → EncounterManager로 인카운터 팝업
    └── 기술 노드 → 해금 알림
  ※ 여러 원정대가 동시에 도착하면 순차 처리
```

#### 2.5 이동 소요 턴수

- 노드 간 **간선(Edge)**에 소요 턴수가 정의됨 (SO에서 설정)
- 기본 1턴, 먼 노드는 2~3턴
- 귀환 시에도 턴 소요 (최단 경로로 자동 귀환)

#### 2.6 귀환

- **수동 귀환 명령**: 플레이어가 명시적으로 귀환 지시
- 귀환 시 **최단 경로**(방문 완료 노드 경유)로 자동 이동
- 귀환 중에도 턴마다 1간선씩 이동
- 기지 도착 시 인력이 내정 풀로 복귀

#### 2.7 안개(Fog of War)

```
[게임 시작]
  기지 노드 + 인접 노드만 공개 (나머지 안개)

[원정대 도착 시]
  도착 노드 공개 + 인접 노드 힌트 공개
  (힌트: 노드 유형 아이콘 표시, 구체적 보상은 미공개)

[안개 상태]
  ├── Hidden: 존재 자체를 모름
  ├── Hinted: 유형 아이콘만 보임 (연결 경로 표시)
  └── Revealed: 완전 공개 (방문 완료)
```

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
원정대 편성 (인력 배치)   →     ExplorationManager          →     인력 풀 차감
목적지 노드 지정          →     (경로 계산)                  →     이동 경로 설정
턴 종료 트리거            →     (이동 처리)                  →     원정대 위치 갱신
탐사 탭 열기 트리거       →     (도착 이벤트 큐 처리)        →     보상 지급 / 인카운터
귀환 명령                →     (최단 경로 계산)              →     귀환 경로 설정
노드 맵 SO              →     (맵 초기화, 안개 관리)        →     맵 상태 (공개/힌트/안개)
```

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| 원정대 인원 (min/max) | 탐사 투입 인력 부담 | 1~3명 |
| 노드 간 소요 턴수 | 탐사 속도, 인력 묶이는 기간 | 1~3턴 |
| 자원 노드 보상량 | 탐사 동기 | 기초 10~30, 고급 5~15, 방어 3~10 |
| 정찰 노드 정보량 | 정보 우위 가치 | 1~3턴 후 웨이브 정보 |
| 탐사 건물 건설 비용 | 탐사 진입 문턱 | 건설 시스템 SO에서 설정 |
| 노드 맵 크기 | 시나리오 탐사 분량 | 8~15 노드 |

### 밸런스 기준

- **인력 압박**: 원정대 1팀(1~3명) 파견 시 기지 내정 인력이 체감 수준으로 부족해야 함
- **탐사 보상 가치**: 1팀 2턴 탐사의 자원 보상 ≈ 자원 건물 2턴 생산량 이상 (투입 인력 기회비용 상쇄)
- **정찰 가치**: 적 웨이브 구성을 미리 알면 방어 배치를 최적화할 수 있어야 함
- **왕복 비용**: 귀환에도 턴 소요 → 먼 노드 탐사는 인력이 오래 묶이는 리스크

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 탐사 건물 0개 상태에서 탐사 시도 | 탐사 탭에 "탐사 건물을 건설하세요" 안내 표시 |
| 원정 중 탐사 건물 파괴 | 이미 파견된 원정대는 유지 (이미 떠남). 새 원정대 편성 불가 |
| 기지 방어 실패 (본부 파괴) | 원정 중인 팀에 영향 없음 (이미 떠남). GameOver 처리는 TurnManager에서 |
| 원정대원 부상 (인카운터 결과) | 부상 상태로 전환. 기지 귀환 후 치료 필요 |
| 모든 노드 탐사 완료 | 추가 파견 불가. 인력 재배치는 플레이어 판단에 맡김 |
| 귀환 경로 차단 (모든 경로 방문 완료) | 방문 완료 노드도 경유 가능 (이벤트 재발생 없이 통과만) |
| 여러 원정대 동시 도착 | 도착 이벤트를 큐에 넣고 탐사 탭 열기 시 순차 처리 |
| 인카운터 노드에서 전투 트리거 선택 | 탐사 중 전투 없음. 해당 선택지는 기지 돌아간 후 다음 밤에 반영하거나, 전투 트리거 선택지를 탐사 인카운터에서 제외 |
| 원정대 인원이 최소 인원 미달 | 편성 불가. UI에서 최소 인원 안내 |
| 이동 중 목적지 변경 | 이동 중에는 목적지 변경 불가. 도착 후 재지정 |

### 추후 확장 예정
- **재방문 가능 노드** (상점 등): `isRevisitable` 플래그 활용
- **원정대원 능력치/적성**: 일부 인카운터 선택지 성공 확률에 영향 (인력 시스템 개별 시민 데이터 구현 후)
- **절차적 맵 생성**: 시나리오별 고정 배치 외에 랜덤 생성 옵션
- **기술 트리 연동**: 기술 노드 → 기술 트리 해금 조건 충족
