## 1. 개요

### 목적
호기심 충족의 핵심 수단. 원정대를 편성하여 노드 그래프 맵을 탐험하고, 기술 해금/적 정찰/자원 발견/인카운터를 통해 내정 방향을 결정하는 정보의 원천이자, 내정-방어를 잇는 다리 역할.

### 핵심 경험
- **정보 발견의 쾌감**: 미지의 노드를 탐사하여 새로운 기술, 자원, 적 정보를 발견할 때의 호기심 충족
- **인력 트레이드오프**: 원정에 인력을 투입하면 기지 내정/방어 인력이 감소 — 항상 "보낼까 말까"의 긴장
- **귀환의 보람**: 원정대가 보상을 들고 기지로 돌아오는 순간의 성취감
- **전략적 경로 선택**: 안전한 자원 노드 vs 위험하지만 기술 해금이 가능한 노드 사이의 선택

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 원정대 편성

| 항목 | 내용 |
|---|---|
| 편성 단위 | 원정대 1팀 = 인력 N명 (최소 2, 최대 6) |
| 동시 운영 | 최대 2개 원정대 동시 파견 가능 (방벽 확장으로 3개까지 해금) |
| 인력 차감 | 원정대에 배치된 인력은 기지 인력 풀에서 제외 |
| 편성 시점 | 낮 페이즈 탐험 탭에서 편성 |
| 편성 비용 | 없음 (인력 배치 자체가 비용) |

- 원정대 규모에 따른 효과:
  - 인원이 많을수록 위험 노드 생존률 증가
  - 인원이 많을수록 자원 운반량 증가 (인당 운반 한도 존재)
  - 인원이 적으면 은밀 이동 가능 (일부 노드 보너스)

#### 2.2 노드 그래프 맵

| 노드 타입 | 비율 | 탐사 결과 | 위험도 |
|---|---|---|---|
| 자원 노드 | 30% | 기초/고급 자원 획득 | 낮음 |
| 정찰 노드 | 15% | 다음 웨이브 정보 일부 공개 | 낮음 |
| 기술 노드 | 15% | 설계도 발견 → 기술 트리 잠금 해제 | 중간 |
| 인카운터 노드 | 20% | 중요 인카운터 이벤트 발생 | 중간 |
| 위험 노드 | 10% | 고급 보상 + 부상/사망 위험 | 높음 |
| 빈 노드 | 10% | 경유지, 보상 없음 | 없음 |

- 기지당 20~40 노드로 구성
- 기지에서 가까운 노드는 안전, 먼 노드일수록 고위험/고보상
- 노드 간 연결은 1~2개의 경로 (일부 노드는 단방향)
- 탐사 전 노드는 "안개" 상태 — 인접 노드만 타입 아이콘 표시, 먼 노드는 ?로 표시

#### 2.3 이동 규칙

| 규칙 | 내용 |
|---|---|
| 이동 비용 | 인접 노드 이동 = 1턴 소요 |
| 다턴 이동 | 2칸 이상 떨어진 노드 = 경유 노드를 거쳐야 함 (턴 당 1노드씩) |
| 이동 타이밍 | 낮 페이즈에 이동 명령 → 턴 종료 시 이동 실행 |
| 잔류 | 탐사 완료 후 원정대는 해당 노드에 잔류 (즉시 귀환 아님) |
| 귀환 | 기지 노드로 돌아와야 보상 수령. 귀환도 턴 소요 |

#### 2.4 탐사 수행

- 원정대가 노드에 도착하면 **자동으로 탐사 수행** (별도 명령 불필요)
- 탐사 결과는 턴 종료 시 리포트로 표시
- 노드 타입별 탐사 결과:

| 노드 타입 | 탐사 결과 상세 |
|---|---|
| 자원 노드 | 기초 자원 10~30, 고급 자원 5~15 (원정대가 운반) |
| 정찰 노드 | 다음 웨이브의 적 타입 1~2종 공개, 진입 방향 공개 |
| 기술 노드 | 설계도 아이템 획득 → 기술 트리 특정 노드 해금 가능 |
| 인카운터 노드 | 중요 인카운터 이벤트 발동 (인카운터 시스템 연동) |
| 위험 노드 | 전투 판정 → 성공 시 희귀 자원/설계도, 실패 시 인력 부상/사망 |

#### 2.5 자원 운반

- 원정대가 획득한 자원은 **원정대가 소지**
- 기지 자원 풀에 반영되지 않음 (기지 ResourceManager와 분리)
- 원정대가 **기지 노드로 귀환 시** 소지 자원이 기지 자원 풀에 추가
- 인당 운반 한도: 기초 자원 20, 고급 자원 10
- 운반 한도 초과 시 초과분은 해당 노드에 남김 (재방문 시 수거 가능)

#### 2.6 위험 판정

- 위험 노드 진입 시 **생존 판정** 실행
- 판정 공식: `생존 확률 = 기본(60%) + (인원 수 × 8%) — 위험도 보정`
- 판정 결과:

| 결과 | 확률 | 효과 |
|---|---|---|
| 완전 성공 | 생존 확률 × 0.4 | 보상 획득, 피해 없음 |
| 부분 성공 | 생존 확률 × 0.6 | 보상 획득, 인력 1명 부상 (2턴 회복) |
| 실패 | 1 — 생존 확률 | 보상 없음, 인력 1명 사망 |

#### 2.7 적 정찰 상세

- 정찰 노드에서 획득하는 정보 단계:

| 정찰 등급 | 공개 정보 | 조건 |
|---|---|---|
| Lv1 (기본) | 적 진입 방향 (동/서/남/북) | 정찰 노드 1곳 탐사 |
| Lv2 (상세) | 적 타입 1~2종 공개 | 정찰 노드 2곳 탐사 |
| Lv3 (완전) | 전체 웨이브 구성 + 규모 공개 | 정찰 노드 3곳+ 탐사 |

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
원정대 편성 (인력 배치)  →     ExpeditionManager         →     인력 풀 차감
이동 명령              →     (노드 이동 + 턴 소비)       →     원정대 위치 갱신
노드 도착              →     (탐사 수행 + 판정)          →     탐사 결과 리포트
위험 노드 진입          →     (생존 판정)               →     성공/부상/사망 처리
귀환 명령              →     (경로 역추적 + 턴 소비)     →     기지 도착 시 자원 수령
정찰 결과              →     (웨이브 정보 갱신)          →     방어 준비 UI 반영
```

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| 동시 파견 원정대 수 | 탐사 속도 vs 내정 인력 부족 | 2~3 |
| 원정대 최대 인원 | 안전성 vs 내정 인력 부족 | 4~8 |
| 인당 운반 한도 (기초) | 탐사 자원 수익 | 15~30 |
| 인당 운반 한도 (고급) | 고급 자원 희소성 | 5~15 |
| 위험 노드 기본 생존률 | 탐사 리스크 | 50%~70% |
| 인원당 생존률 보너스 | 대규모 원정 인센티브 | 5%~12% |
| 정찰 Lv3 달성 필요 노드 수 | 정찰 투자 대비 보상 | 2~4 |
| 노드 간 거리 (턴) | 탐사 템포 | 대부분 1턴, 일부 2턴 |

### 밸런스 기준

- **왕복 4턴 내 첫 보상**: 기지 인접 자원 노드는 이동 1턴 + 귀환 1턴 = 2턴 만에 자원 수령 가능
- **정찰 투자 보상**: 정찰 2회(4턴 투자) 시 Lv2 정보 → 방어 배치 최적화로 자원 절약 효과
- **위험 노드 딜레마**: 6명 원정대(생존률 108% → 100% 캡) 시 거의 안전하지만 기지 인력 6명 부재
- **탐사 vs 내정**: 3명 원정대 2팀 파견 시 기지 인력 6명 감소 → 자원 건물 2개분 생산량 손실
- **설계도 희소성**: 기술 노드는 맵의 깊은 곳에 배치 → 도달에 3~5턴 투자 필요

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 원정대 전원 사망 | 원정대 해산, 소지 자원 해당 노드에 잔류 (새 원정대 파견 시 수거 가능) |
| 기지 인력 0명인 상태에서 원정대 편성 시도 | 편성 불가 + "가용 인력 부족" 토스트 |
| 원정대가 탐사 중 밤 전환 | 원정대는 노드에서 안전 (밤 전투는 기지에서만 발생) |
| 모든 노드 탐사 완료 | 자원 노드는 재방문 시 감소된 자원 재생산 (50%) |
| 원정대 귀환 중 인력 풀 초과 | 발생하지 않음 — 인력 풀 상한 없음 |
| 운반 한도 초과 자원 | 해당 노드에 잔류, "초과 자원 잔류" 알림 표시 |
| 부상 인력의 원정대 재편성 | 부상 회복 전까지 편성 불가 |
| 원정대 이동 중 턴 종료 차단 (방어 자원 부족) | 원정대 이동은 별도 처리 — 턴 종료 차단과 무관하게 이동은 예약 상태 유지 |
