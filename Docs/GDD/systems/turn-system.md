# 턴 시스템

- **작성일**: 2026-03-25
- **상태**: 기획 완료
- **slug**: turn-system

---

## 1. 개요

### 목적
게임의 핵심 루프(낮→밤→다음 낮)를 관리하는 중앙 오케스트레이터. 모든 시스템의 턴 종료/시작 처리를 조율하고, 밤 인카운터(전투/이벤트)를 결정하며, 시나리오 진행(턴 수, 클리어 조건)을 추적한다.

### 핵심 경험
- **리듬감 있는 루프**: 낮(의사결정) → 밤(결과 확인) → 다음 낮(새 기회)의 반복
- **긴장감 있는 전환**: 페이드 연출과 함께 "밤이 온다"는 압박감
- **한 턴만 더**: 다음 턴에 건설 완료, 전투 대비 등 기대감으로 멈출 수 없는 흐름

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 턴 페이즈 구조

```
[낮 페이즈]
  플레이어 행동 (건설, 인력 배치, 탐사 명령 등)
       │
       │ "턴 종료" 버튼
       ▼
[낮 종료 처리]
  ① 자원 생산 건물 → 자원 추가 (추후 구현)
  ② 손상 건물 자동 회복
  ③ 낮→밤 페이드 연출 (1.5초)
       │
       ▼
[밤 페이즈]
  시나리오 데이터에서 이번 턴 인카운터 결정
  ├── 고정 전투 → BattleManager.StartBattle()
  ├── 랜덤 인카운터 → 확률 테이블에서 결정
  │     ├── 전투 → BattleManager.StartBattle()
  │     ├── 이벤트 → 토스트 메시지 표시
  │     └── 없음 → 스킵
  └── 인카운터 종료 대기
       │
       ▼
[밤 종료 처리]
  ① 전투 결과 통계 토스트 (전투가 있었을 경우)
  ② 밤→낮 페이드 연출 (2초, 해가 뜨는 느낌)
  ③ 턴 카운터 증가
  ④ 최종 턴 클리어 체크
       │
       ▼
[다음 낮 시작 처리]
  ① 건설 진행도 증가 (BuildingManager.ProcessTurn())
  ② 완성 건물 활성화
  ③ 낮 페이즈로 복귀
```

#### 2.2 건설 진행도 타이밍
- **건설 진행도는 다음 낮 시작 시 증가** (밤 전투 직전이 아님)
- 이유: 완성 직후 밤 전투에서 파괴되는 나쁜 경험 방지
- `BuildingManager.ProcessTurn()`은 다음 낮 시작 시점에 호출

#### 2.3 인카운터 결정
- 시나리오 SO에서 턴별 인카운터 유형을 정의
- **고정 전투**: 기획자가 특정 턴에 반드시 전투 발생하도록 지정
- **랜덤 인카운터**: 확률 테이블에 따라 전투/이벤트/없음 결정
- PoC에서 이벤트는 토스트 메시지로만 표시

#### 2.4 페이드 전환 연출
- **낮→밤**: 화면이 점진적으로 어두워짐 (1.5초). 단, 전투 시 적당히 밝게 유지
- **밤→낮**: 화면이 밝아지며 해 뜨는 느낌 (2초)
- PoC에서는 UI 없이 화면 페이드만 구현

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
턴 종료 버튼 클릭       →     TurnManager                →     페이즈 전환 이벤트
시나리오 SO 데이터       →     (인카운터 결정)              →     인카운터 시작/종료
                              (턴 카운트 관리)             →     턴 수 변경 이벤트
BattleManager 콜백      →     (전투 종료 감지)             →     밤 종료 트리거
                              (클리어/패배 판정)           →     게임 종료 이벤트
```

### 상태 다이어그램

```
                    ┌──────────┐
                    │  DayPhase │ ◀─────────────────────────┐
                    └────┬─────┘                            │
                         │ 턴 종료 버튼                      │
                    ┌────▼──────────┐                       │
                    │ DayEndProcess │                       │
                    │ (자원/회복)    │                       │
                    └────┬──────────┘                       │
                         │                                  │
                    ┌────▼──────────┐                       │

                    └────┬──────────┘                       │
                         │                                  │
                    ┌────▼──────────┐                       │
                    │  NightPhase   │                       │
                    │ (인카운터)     │                       │
                    └────┬──────────┘                       │
                         │ 인카운터 종료                     │
                    ┌────▼──────────┐                       │
                    │NightEndProcess│                       │
                    │ (통계 토스트)  │                       │
                    └────┬──────────┘                       │
                         │                                  │
                    ┌────▼──────────┐                       │
                    │ FadeToDay     │ (2초)                  │
                    └────┬──────────┘                       │
                         │                                  │
                    ┌────▼──────────┐                       │

특수 전이:
  NightPhase에서 본부 파괴 → GameOver (Defeat)
```

## 4. 밸런스 가이드

### 조정 가능 파라미터 (ScenarioDataSO)

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| totalTurns | 세션 길이 | 15~30턴 |
| 고정 전투 턴 | 전투 빈도 조절 | 3~5턴 간격 |
| 랜덤 인카운터 확률 | 밤 긴장감 | 전투 20~40%, 이벤트 20~30%, 없음 30~50% |
| fadeToNightDuration | 전환 체감 | 1.0~2.0초 |
| fadeToDayDuration | 전환 체감 | 1.5~2.5초 |
| nightAmbientIntensity | 밤 시각적 분위기 | 0.2~0.4 |

### 밸런스 기준

- **전투 빈도**: 연속 3턴 이상 전투 없으면 긴장감 저하. 연속 3턴 이상 전투이면 피로감
- **이벤트 발생률**: 너무 잦으면 성가심, 너무 적으면 존재감 없음
- **페이드 속도**: 빠르면 전환감 부족, 느리면 루즈함. 1.5~2초가 적당

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 턴 종료 버튼 연타 | 페이즈 전환 중 버튼 비활성화 |
| 전투 중 본부 파괴 | BattleManager.OnBattleEnded(Defeat) → TurnManager가 GameOver 처리 |
| 최종 턴 전투 승리 | 전투 종료 → 턴 카운터 증가 → totalTurns 도달 → Victory 엔딩 |
| 최종 턴에 전투 없음 | 밤 인카운터 없이 턴 종료 → totalTurns 도달 → Victory 엔딩 |
| 전투 없는 밤 | 인카운터 없음 → 짧은 대기 후 밤→낮 전환 |
| 이벤트 + 전투 같은 턴 | 이벤트 토스트 먼저 → 확인 후 전투 시작 |
| 시나리오 데이터 없음 | 기본값 사용 (20턴, 매 3턴 전투, 이벤트 확률 0%) |

### 추후 연동 예정 (문서 참고용)

#### 자원 시스템 연동 포인트
- **낮 종료 시**: 자원 생산 건물 → 자원 추가 (+플로팅 UI 이펙트 "+자원아이콘 숫자")
- **전투 시작 시**: 활성화된 방어 건물 슬롯의 소모량 만큼 방어 자원 차감
- **전투 미발생 시**: 방어 자원 차감 없음
- 자원 생산 UI는 낮→밤 전환 중에 표시 (플로팅 텍스트)

#### 인력 관리 연동 포인트
- **턴 종료 시**: 인력 미배치 슬롯 경고 표시 (선택적 — 무시하고 진행 가능)
