# 인카운터 시스템

- **작성일**: 2026-03-26
- **상태**: 기획 완료
- **slug**: encounter-system

---

## 1. 개요

### 목적
매 턴 의사결정을 제공하여 리듬감과 내러티브를 부여하는 이벤트 시스템. 일상 인카운터(이진 선택)와 중요 인카운터(3선택지)로 이원화하여 소소한 변수와 큰 방향성 결정을 모두 제공.

### 핵심 경험
- **리듬감**: 매 턴 낮 시작 시 일상 인카운터로 소소한 변수 제공
- **선택의 무게**: 중요 인카운터에서 코스트 있는 큰 보상 vs 안전한 작은 보상
- **반드시 선택**: 거부 불가 — 선택지 중 하나는 골라야 함
- **버프의 전략**: N턴간 지속되는 버프로 다음 전투/생산에 영향

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 인카운터 이원화

| 유형 | 발생 시점 | 형식 | 용도 |
|---|---|---|---|
| 일상 인카운터 | 낮 시작 시 (확률) | 이진 선택 (A or B) | 소소한 변수, 리듬감 |
| 중요 인카운터 | 밤 페이즈 (시나리오 정의) | 3선택지 | 방향성 결정, 큰 보상/대가 |

#### 2.2 일상 인카운터 (Reigns식)
- 낮 시작 시 확률적으로 등장
- **이진 선택**: 두 선택지 중 하나를 반드시 선택 (거부 불가)
- 즉발 효과 (자원/인력 변동) 또는 버프 효과 (N턴간 지속)

#### 2.3 중요 인카운터 (Scythe식)
- 밤 페이즈에서 시나리오 데이터에 의해 발생
- **3선택지**:
  1. 코스트 없는 작은 보상 (항상 선택 가능)
  2. 코스트 있는 큰 보상 (자원 부족 시 선택 불가)
  3. 조건 충족 시 선택 가능한 특수 보상 (특정 건물 Active 등)

#### 2.4 빈도 조절 (Pity 시스템)
일상 인카운터의 과도한 연속 출현/미출현 방지:

```
실제 확률 = baseChance + pityBonus - fatiguePenalty
pityBonus   = 미발생 연속 턴 × pityIncrement  (상한: maxChance)
fatiguePenalty = 연속 발생 턴 × pityDecrement  (하한: minChance)
```

| 상황 | 보정 | 결과 확률 (base=0.4) |
|---|---|---|
| 1턴 미발생 | +0.15 | 0.55 |
| 2턴 미발생 | +0.30 | 0.70 |
| 3턴 미발생 | +0.45 → 캡 0.9 | 0.90 |
| 연속 1회 발생 후 | -0.10 | 0.30 |
| 연속 2회 발생 후 | -0.20 | 0.20 |

#### 2.5 버프 시스템
인카운터 선택 결과로 N턴간 지속되는 버프/디버프 부여:

| 버프 종류 | 효과 | PoC 포함 |
|---|---|---|
| 생산력 증가 | 자원 건물 생산량 +X% | O |
| 공격력 증가 | 타워 데미지 +X% | O |
| 방어 자원 절반 소모 | 방어 자원 소모 50% | O |

- 같은 종류 버프 중복 시 더 강한 것으로 갱신 (스택 X)
- 활성 버프는 상단 UI에 표시, 마우스 호버 시 설명

#### 2.6 선택지 결과 유형

| 효과 유형 | 설명 |
|---|---|
| 즉발 자원 변동 | 기초/고급/방어 자원 ±N |
| 즉발 인력 변동 | 인력 추가 또는 부상 |
| 버프 부여 | N턴간 지속 효과 |
| 전투 트리거 | 선택 후 전투 발생 |

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
낮 시작 트리거           →     EncounterManager          →     일상 인카운터 팝업
밤 시나리오 데이터       →     (pity 확률 + 풀 선택)     →     중요 인카운터 팝업
플레이어 선택지 클릭     →     (효과 적용)               →     자원/인력 변동 + 버프
턴 경과                 →     BuffManager               →     버프 타이머 감소/만료
```

### 핵심 변수

| 변수 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| baseEncounterChance | float | 0.4 | 일상 인카운터 기본 발생 확률 |
| pityIncrement | float | 0.15 | 미출현 턴당 확률 증가 |
| pityDecrement | float | 0.1 | 출현 턴당 확률 감소 |
| minChance | float | 0.1 | 최소 발생 확률 |
| maxChance | float | 0.9 | 최대 발생 확률 |

| 연동 대상 | 인터페이스 | 타이밍 |
|---|---|---|
| **턴 시스템** | 낮 시작 시 일상 인카운터 트리거 | 턴 전환 시 |
| **턴 시스템** | 밤 중요 인카운터 (기존 EncounterDataSO 대체/확장) | 밤 페이즈 |

| **버프 시스템** | 생산/공격/방어 보너스 적용 | 매 턴 |

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| baseEncounterChance | 일상 인카운터 빈도 | 0.3~0.5 |
| pityIncrement/Decrement | 연속 출현/미출현 조절 | 0.1~0.2 |
| 일상 인카운터 효과 크기 | 게임 리듬 영향 | 자원 ±5~15, 버프 2~3턴 |
| 중요 인카운터 보상 크기 | 방향성 결정 무게감 | 자원 ±20~50 |
| 버프 지속 턴 | 전략 영향 기간 | 2~4턴 |
| 버프 효과량 | 체감 강도 | 20%+ 보너스 |

### 밸런스 기준

- **빈도**: 매 2~3턴에 1회 일상 인카운터 (체감)
- **일상 효과**: "있으면 좋고 없으면 아쉬운" 수준
- **중요 효과**: "게임 흐름이 바뀌는" 수준
- **버프 체감**: 활성 시 눈에 띄는 차이 (20%+ 보너스)

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 코스트 선택지의 자원 부족 | 해당 선택지 비활성화 (선택 불가) |
| 조건 미충족 선택지 | 해당 선택지 비활성화 + 조건 텍스트 표시 |
| 인카운터 풀 비어있음 | 인카운터 미발생, 경고 로그 |
| 버프 중복 | 같은 종류 → 더 강한 것으로 갱신 (스택 X) |
| 전투 직전 버프 | 밤 인카운터 선택 후 전투 시작 → 버프 전투에 반영 |
