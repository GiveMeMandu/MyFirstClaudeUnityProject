# 건설 시스템

- **작성일**: 2026-03-24
- **상태**: 기획 완료
- **slug**: construction-system

---

## 1. 개요

### 목적
낮 페이즈에서 기지의 생산력/방어력/탐사력을 결정하는 핵심 의사결정 시스템. 플레이어는 고정 슬롯에 건물을 활성화하고 업그레이드하여 밤의 방어전을 대비한다.

### 핵심 경험
- **"언제 건설할까"의 타이밍 판단**: 지금 자원을 건설에 투입할지, 방어 운영에 남길지
- **"어떤 분기로 갈까"의 비가역적 선택**: 업그레이드 분기 선택의 긴장감과 뿌듯함
- **인력 배분의 퍼즐**: 건설에 몇 명을 투입할지 vs 다른 곳에 배치할지 매 턴 결정
- **기지 성장의 시각적 만족감**: 허름한 기지가 점차 확장되어가는 과정

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 슬롯 구조
- 모든 건물 위치는 **고정 슬롯**으로 맵에 배치됨
- 각 슬롯에 **어떤 건물이 지어지는지 사전 결정**되어 있음 (플레이어가 건물 종류를 선택하지 않음)
- 게임 시작 시 모든 슬롯이 보이지 않고, **인접 위치로 점진적 확장** (Thronefall 방식)
- 방벽 건물도 일반 건설과 동일한 슬롯 시스템으로 동작

#### 2.2 건물 카테고리
| 카테고리 | 역할 | 인력 배치 시 기능 |
|---|---|---|
| 자원 | 매 턴 자원 생산 | 인력 배치 수에 따라 산출량 결정 |
| 방어 | 밤 전투 시 자동 공격 | 인력 배치 수에 따라 화력 결정 + 운영 자원 소모 |
| 연구 | 연구 포인트 획득 | 인력 배치 시 연구 진행 |
| 행정 | QoL 기능, 정보 접근 단계 | 행정관 능력치에 따라 기능 차등 |
| 탐사 | 원정대 편성/파견 | 탐사 관련 기능 활성화 |
| 방벽 | 기지 경계 확장 | 확장 시 새 슬롯 노출 + 방어 범위 증가 |

#### 2.3 건설 프로세스
1. **건설 명령**: 낮 페이즈에서 슬롯 선택 → 건설 시작 (자원 소비)
2. **인력 투입**: 건설 중인 슬롯에 인력 배치 (기본 1명, 연구로 2~3명까지 확장)
3. **진행도 증가**: 인력이 배치된 턴에만 진행도 증가. 미배치 시 정지
4. **완성**: 필요 턴 수 도달 시 건물 활성화 (완성 연출: 스케일 애니메이션 + 파티클 + "쿵" 사운드)

```
[빈 슬롯] → 건설 명령(자원 소비) → [건설중] → 인력 투입 × N턴 → [완성]
                                        ↑                              ↓
                                   취소 가능                     업그레이드 분기 선택
                                 (자원 반환)                    (비가역, 자원 소비)
                                                                       ↓
                                                              [업그레이드 완성]
                                                              (추가 슬롯/기능)
```

#### 2.4 업그레이드 분기
- 건물 완성 시 **기본 분기 1개** 이상 존재
- **연구 트리 연동**: 건물 완성 → 연구 트리에 해당 건물의 분기 노드 추가 → 연구 완료 시 추가 분기 해금
- 분기 선택은 **비가역** (해당 슬롯 한정)
- 상위 티어 업그레이드 시 **추가 인력 슬롯** 생성 → 다른 기능 활성화 가능

#### 2.5 건물 손상/파괴/수리

```
[완성] ── 적 공격 ──→ [손상] ── HP 0 ──→ [파괴]
  ↑                      │                   │
  │                 자동 회복              수리 명령
  │                 (~4턴)              (인력 투입)
  │                      │                   │
  └──────────────────────┘                   ↓
                                        [수리중]
                                     (기능 정지)
                                     (공격 대상 제외)
                                           │
                                     수리 완료
                                           ↓
                                        [완성]
```

- **손상**: HP가 감소했지만 0 이상. 기능 정상 동작. 매 턴(낮 시작 시) 자동 회복 (약 4턴에 전체 회복, 즉 매 턴 25% 회복)
- **파괴**: HP가 0. 기능 정지. 수리 명령 필요 (인력 투입, 자원 비용 없음, 자연 회복 없음)
- **수리 중**: 기능 정지 + 적 공격 대상 제외. 인력 투입으로 가속 가능
- **건설 중인 건물**: 적 공격 대상에서 제외
- **본부 파괴 = 게임오버**

#### 2.6 방어 타워 운영 자원
- 방어 건물은 활성화(인력 배치) 시 **고정 운영 자원 1종** 소모
- 인력 수에 비례하여 소모량 증가 (각 건물의 슬롯 별로 소모되는 고정값이 있음)
- 현재 보유량 < 이미 배치된 방어 타워 자원 소모량 + 배치하고자 하는 슬롯의 이번 턴 소모 예정량 → 해당 방어 타워에 추가 인력 배치 불가
- **핵심 판단**: 방어 타워에 인력을 많이 넣으면 화력↑이지만 운영 자원 빠르게 고갈

### 데이터 흐름

```
[입력]                          [처리]                         [출력]
─────────────                 ─────────────                  ─────────────
플레이어 건설 명령    →     BuildingManager          →     건물 상태 변경
자원 보유량           →     (건설/업그레이드/수리)    →     자원 차감
인력 배치 결정        →     BuildingSlot             →     진행도 증가
                            (상태 머신)              →     완성 이벤트 발행
연구 트리 해금        →     UpgradeTree              →     분기 노드 추가
적 공격 (밤)          →     BuildingHealth           →     HP 감소/파괴
턴 경과               →     AutoRepair               →     손상 자동 회복
```

### 상태 다이어그램

건물 슬롯의 상태 전이:

```
         ┌─────────┐
         │  Hidden  │  (아직 노출되지 않은 슬롯)
         └────┬─────┘
              │ 방벽 확장 or 선행 건물 건설
         ┌────▼─────┐
         │  Empty   │  (빈 슬롯, 건설 가능)
         └────┬─────┘
              │ 건설 명령 (자원 소비)
         ┌────▼─────────┐
    ┌───→│ Constructing │  (건설중, 인력 투입 시 진행)
    │    └────┬─────────┘
    │         │ 진행도 100%
    │    ┌────▼─────┐
    │    │  Active  │  (완성, 기능 동작)
    │    └──┬───┬───┘
    │       │   │ 업그레이드 명령
    │       │  ┌▼──────────┐
    │       │  │ Upgrading  │  (업그레이드중)
    │       │  └──┬─────────┘
    │       │     │ 진행도 100%
    │       │  ┌──▼─────┐
    │       │  │ Active  │  (상위 티어)
    │       │  └─────────┘
    │       │
    │       │ 적 공격 (HP 감소)
    │  ┌────▼─────┐
    │  │ Damaged  │  (손상, 기능 동작, 자동 회복중)
    │  └──┬───┬───┘
    │     │   │ HP = 0
    │     │  ┌▼──────────┐
    │     │  │ Destroyed  │  (파괴, 기능 정지)
    │     │  └──┬─────────┘
    │     │     │ 수리 명령 (인력 투입)
    │     │  ┌──▼─────────┐
    │     │  │ Repairing  │  (수리중, 기능 정지, 공격 대상 제외)
    │     │  └──┬─────────┘
    │     │     │ 수리 완료
    │     └─────┴──→ Active
    │
    │ 취소 (자원 반환)
    └─────────────────
```

### 연동 시스템

| 연동 대상 | 인터페이스 | 방향 |
|---|---|---|
| **인력 관리** (관리 탭) | 건설/수리 슬롯에 인력 배치/회수 요청 | 양방향 |
| **자원 시스템** | 건설/업그레이드 비용 차감, 방어 운영 자원 소모 | 건설 → 자원 |
| **기술 트리** (연구) | 건물 완성 → 연구 트리에 분기 노드 추가. 연구 완료 → 추가 분기 해금 | 양방향 |
| **웨이브 방어** | 밤 전투에서 건물 HP 감소/파괴 이벤트 수신 | 방어 → 건설 |
| **방벽 확장** | 방벽 건물 완성 → 인접 Hidden 슬롯을 Empty로 전환 | 건설 → 슬롯 |
| **턴 시스템** | 낮 페이즈에서만 건설 명령 허용. 턴 종료 시 진행도/자동회복 처리 | 턴 → 건설 |

## 4. 밸런스 가이드

### 조정 가능 파라미터 (기획자용 도구)

**BuildingData ScriptableObject**에 노출하여 Inspector/커스텀 에디터에서 조정:

| 파라미터                         | 조정 목적     | 권장 범위                              |
| ---------------------------- | --------- | ---------------------------------- |
| constructionTurns (티어별)      | 건설 템포 조절  | T1: 1턴, T2: 2턴, T3: 3턴             |
| maxHP (건물별)                  | 방어전 생존성   | 50~300                             |
| autoRepairRate               | 손상 회복 속도  | maxHP / 3 ~ maxHP / 5 (3~5턴 완전 회복) |
| constructionCost (건물별)       | 건설 진입 문턱  | 자원 종류별 비용                          |
| defenseResourceCostPerWorker | 방어 운영 부담  | 1턴 생산량의 20~40%                     |
| upgradeCostMultiplier        | 업그레이드 가성비 | 1.3x ~ 2.0x                        |
| maxWorkers (연구 단계별)          | 건설 가속 상한  | 1 → 2 → 3                          |

### 기획자용 도구 요구사항

1. **BuildingData 커스텀 에디터**: 건물별 티어/비용/소요 턴/HP/자동회복을 테이블 형태로 한눈에 편집
2. **업그레이드 분기 시각화**: 노드 그래프 형태로 분기 구조와 해금 조건 확인/편집
3. **밸런스 프리뷰**: 현재 수치로 N턴 시뮬레이션 시 건물 상태 예측 (선택적)

### 밸런스 기준

- **건설 투자 회수**: 자원 건물은 건설 비용을 3~5턴 내에 회수 가능해야 함
- **방어 운영 비용**: 모든 방어 타워 풀가동 시 자원 생산의 60~80% 소모 → 효율적 배분 강제
- **수리 vs 신규 건설**: 수리(무자원, 인력만)가 신규 건설보다 항상 경제적이어야 함
- **인력 병목**: 전체 인력의 70% 이상을 한 카테고리에 투입하면 다른 카테고리가 기능 부전 느껴야 함

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 건설 중 자원이 부족해짐 | 이미 시작된 건설은 계속 진행. 비용은 건설 명령 시 선차감 |
| 건설 중 취소 | 투입 자원 전액 반환. 슬롯은 Empty 상태로 복귀 |
| 업그레이드 중 취소 | 투입 자원 전액 반환. 기존 티어 Active 상태 유지 |
| 인력 미배치 상태로 턴 종료 | 건설 진행도 증가하지 않음. 경고 표시 없음 (의도적 선택일 수 있음) |
| 동시 다수 건물 건설 | 허용. 단, 각 슬롯에 인력을 따로 배치해야 하므로 인력 병목 발생 |
| 본부 파괴 | **게임오버** 트리거. 본부는 최후 방어선 |
| 손상된 건물 추가 피격 | HP 추가 감소. 0 도달 시 파괴 전환 |
| 수리 중 건물 피격 | 공격 대상 제외이므로 피격되지 않음 |
| 건설 중 건물 피격 | 공격 대상 제외이므로 피격되지 않음 |
| 방어 운영 자원 부족 | 해당 방어 타워에 추가 인력 배치 불가. 기존 배치 인력 재배치 가능 |
| 방어 운영 자원이 턴 중 소진 | 배치된 인원을 빼고 다른 슬롯에 재배치 가능 |
| 모든 건물 파괴 (본부 제외) | 게임오버 아님. 본부만 살아있으면 수리/재건 가능 |
