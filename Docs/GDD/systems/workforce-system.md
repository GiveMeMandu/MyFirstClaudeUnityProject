# 인력 배치 시스템

- **작성일**: 2026-03-25
- **상태**: 기획 완료
- **slug**: workforce-system

---

## 1. 개요

### 목적
기지의 전체 인력 풀을 관리하고, 각 건물의 슬롯에 인력을 배분하는 핵심 의사결정 시스템. 인력은 항상 부족하므로 어디에 배치하느냐가 게임의 핵심 전략이 된다.

### 핵심 경험
- **선택의 긴장감**: 건물은 많은데 인력은 부족. 어디에 배치할지 매 턴 고민
- **인력 배분의 퍼즐**: 자원 생산 vs 방어 활성화 vs 건설 가속 vs 치료 지원 사이의 트레이드오프
- **부상의 압박**: 전투 후 부상자 발생 → 가용 인력 감소 → 다음 턴 운영이 더 빡빡해짐

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 인력 풀
- 기지에 총 N명의 인력이 존재 (PoC: 4명 시작)
- 인력 상태: **건강(Healthy)** / **부상(Injured)**
- 건강한 인력만 건물에 배치 가능
- 미배치 인력은 유휴(Idle) 상태로 아무 효과 없음

#### 2.2 건물 다중 슬롯
각 건물은 1개 이상의 **인력 슬롯(WorkerSlot)**을 보유:

| 건물 카테고리 | 슬롯 예시 | 효과 |
|---|---|---|
| 자원 | 생산 슬롯 | 인력 배치 수에 따라 자원 산출량 결정 |
| 방어 | 공격력 슬롯 | 배치 시 타워 데미지 증가 |
| 방어 | 공격 속도 슬롯 | 배치 시 공격 속도 증가 |
| 건설중 | 건설 슬롯 | 배치 수에 따라 건설 진행도 증가 |
| 연구 | 연구 슬롯 | 배치 시 연구 포인트 획득 |

- 방어 건물은 최소 1명 배치 시 활성화 (0명이면 공격 안 함)
- 슬롯별로 최대 배치 수가 다름 (SO에서 정의)
- 추후 시민별 적합치에 따른 보너스/패널티 예정

#### 2.3 배치 규칙
- **낮 페이즈에서만** 배치/재배치 가능
- 매 턴 자유롭게 재배치 가능
- 턴 종료 시 배치된 상태로 밤에 진입
- 전체 배치 인력 수 ≤ 건강한 인력 수

#### 2.4 부상 시스템
- **부상 발생 조건**:
  - 밤 전투에서 파괴된 건물에 배치되었던 인력 → 부상
  - 이벤트에 의한 부상 (턴 종료 이후 발생)
- **부상 면제**: 방어 건물, 건설 중 건물에 배치된 인력은 이벤트 부상 대상에서 제외
- **치료**:
  - 자연 회복: 3~4턴 후 자동 회복
  - 본부 치료 슬롯: 건강한 인력을 배치하면 부상자 회복이 1~2턴 가속
  - 치료 슬롯에는 부상자가 아닌 건강한 인력을 배치 (간호 역할)
  - 부상자가 슬롯을 차지하는 것이 아님
- **사망 없음**: 어떤 경우에도 인력은 사망하지 않음 (캐주얼 지향)

#### 2.5 인력 획득
- 시나리오 시작 시 초기 인력 수 (SO 정의)
- 이벤트/전투 보상으로 추가 인력 획득 가능

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
플레이어 배치 명령       →     WorkforceManager           →     건물 슬롯 인력 수 변경
전체 인력 풀 상태        →     (배치/회수/부상 관리)       →     BuildingSlot.SetWorkers()
건물 파괴 이벤트         →     (부상 처리)                →     인력 상태 변경
턴 경과                 →     (부상 회복 처리)            →     부상자 → 건강 전환
```

### 상태 다이어그램

```
[Healthy] ── 건물 파괴 / 이벤트 ──→ [Injured]
    ↑                                    │
    │                              자연 회복 (3~4턴)
    │                              또는 치료 가속 (1~2턴 단축)
    │                                    │
    └────────────── 회복 완료 ────────────┘
```

### 연동 시스템

| 연동 대상 | 인터페이스 | 방향 | 타이밍 |

| **방어 시스템** | 타워 활성/비활성 (인력 ≥ 1) | 인력→방어 | 밤 전투 시작 시 |

| **방어 시스템** | 건물 파괴 → 인력 부상 | 방어→인력 | 밤 전투 중 |
| **이벤트 시스템** | 이벤트 → 인력 추가/부상 | 이벤트→인력 | 밤 인카운터 |

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| totalWorkers (초기) | 시작 난이도 | 3~6 |
| naturalHealTurns | 부상 패널티 강도 | 3~5턴 |
| healingBoostTurns | 치료 슬롯 가치 | 1~3턴 단축 |
| 슬롯별 maxWorkers | 건물 효율 상한 | 1~3 |
| 이벤트 인력 획득량 | 성장 속도 | 1~2명/이벤트 |

### 밸런스 기준

- **인력 부족**: 전체 슬롯 수의 50~70%만 채울 수 있어야 함 (항상 선택 필요)
- **부상 영향**: 전투 후 1~2명 부상 시 "다음 턴 운영이 빡빡하다" 체감
- **치료 가치**: 치료 슬롯에 인력 투입 = 다른 곳 못 넣는 기회비용 → 치료 가속이 확실히 의미 있어야 함
- **방어 활성화**: 인력 1명을 방어에 넣으면 자원 생산 1명 감소 → 트레이드오프 체감

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 모든 인력이 부상 | 건물 배치 불가. 자연 회복까지 대기. 매우 위험한 상황 |
| 부상 인력이 배치된 건물 파괴 | 이미 부상 상태이므로 추가 부상 없음 (부상 중첩 없음) |
| 건물 파괴 시 배치 인력 복귀 | 유휴 풀로 복귀 후 부상 상태로 전환 |
| 방어 건물 인력 0명으로 밤 진입 | 해당 타워 공격 안 함. 경고는 표시하되 진행 허용 |
| 이벤트 부상 대상 선정 | 방어 건물, 건설 중 건물에 배치된 인력은 후보에서 제외 |
| 인력 수 초과 배치 시도 | UI에서 가용 인력(건강-배치됨) 초과 배치 불가 |
| 턴 종료 시 미배치 인력 경고 | 경고 표시하되 무시하고 진행 가능 (의도적 유휴 허용) |
| 건물 수리 완료 후 인력 | 수리 완료 시 이전 배치 상태 초기화. 재배치 필요 |
| 치료 슬롯에 인력 배치 중 부상 발생 | 치료사(건강한 인력)는 본부에 배치 중이므로 이벤트 부상 면제 대상 아님(본부는 방어/건설이 아님). 기획 재검토 필요 — PoC에서는 본부 배치 인력도 이벤트 부상 면제 처리 |
