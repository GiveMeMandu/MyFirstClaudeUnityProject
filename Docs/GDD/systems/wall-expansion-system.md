# 방벽 확장 시스템

- **작성일**: 2026-03-26
- **상태**: 기획 완료
- **slug**: wall-expansion-system

---

## 1. 개요

### 목적
기지 성장의 물리적/시각적 체감을 제공하는 핵심 진행 시스템. 자원을 투자하여 방벽을 확장하면 기지 영역이 넓어지고, 새 건설 슬롯과 기능이 해금된다. 동시에 방어 범위 증가라는 트레이드오프를 통해 전략적 판단을 강제한다.

### 핵심 경험
- **공간 확장 = 기능 해금**: 방벽을 넓히면 새로운 건설 슬롯이 노출되는 직관적 진행감
- **성장 vs 생존의 딜레마**: 확장할수록 방어 범위가 넓어져 더 많은 방어 건물/인력이 필요
- **시각적 성장감**: 좁은 기지가 점차 넓어지며 건물이 들어서는 과정의 만족감
- **시나리오별 변주**: 기획자가 SO 데이터로 시나리오마다 다른 확장 경험을 설계 가능

## 2. 시스템 설계

### 핵심 메커니즘

#### 2.1 방벽 레벨 구조

| 레벨 | 역할 | 예시 |
|---|---|---|
| Lv.0 (초기) | 시작 영역. 본부 + 기본 슬롯 2~3개 | 튜토리얼 시작 상태 |
| Lv.1 | 1차 확장. 자원/방어 건물 슬롯 추가 | 자원 건물 2개 + 방어 타워 1개 |
| Lv.2 | 2차 확장. 고급 건물 슬롯 + (선택적) 탐사 해금 | 연구소 + 탐사 건물 슬롯 |
| Lv.3 | 3차 확장. 후반 건물 슬롯 + (선택적) 정책 해금 | 고급 방어 + 행정 건물 슬롯 |
| Lv.4 (최대) | 최종 확장. 모든 슬롯 노출 | 전 슬롯 개방, 풀 방어 구성 가능 |

- 기지당 3~5단계 확장 (시나리오 SO에서 최대 레벨 조절 가능)
- 레벨 0은 항상 존재하며, 최대 레벨은 시나리오별로 다를 수 있음

#### 2.2 확장 비용

| 레벨 | 기초 자원 | 고급 자원 | 비고 |
|---|---|---|---|
| Lv.0 → Lv.1 | 60 | 0 | 첫 확장은 기초 자원만으로 가능 |
| Lv.1 → Lv.2 | 100 | 20 | 고급 자원 투자 시작 |
| Lv.2 → Lv.3 | 150 | 50 | 중후반 투자 |
| Lv.3 → Lv.4 | 200 | 80 | 풀 확장 고비용 |

- 모든 비용은 `WallExpansionDataSO`의 레벨별 데이터에서 설정
- 기획자가 시나리오별로 자유롭게 조정 가능

#### 2.3 확장 조건
- **필수 조건**: 보유 자원 >= 확장 비용
- **선택적 조건**: 기술 연구 완료 (시나리오 SO에서 `requiresResearch` 플래그)
- **선택적 조건**: 특정 턴 도달 (시나리오 SO에서 `minTurn` 설정)
- 조건 미충족 시 확장 버튼 비활성화 + 부족 조건 툴팁 표시

#### 2.4 확장 프로세스

```
[현재 방벽 레벨 N]
       │
       │ 확장 명령 (자원 즉시 차감)
       ▼
[확장 연출 시작]
  ① 방벽 팽창 애니메이션 (1.5초)
  ② 새 영역 밝아지는 효과 (어둡던 영역 → 밝아짐)
  ③ 새 건설 슬롯 Hidden → Empty 전환
  ④ (시나리오 설정 시) 시스템 해금 토스트 표시
       │
       ▼
[방벽 레벨 N+1]
  - 방어 범위 증가 적용
  - 새 슬롯 건설 가능
```

- 확장은 **즉시 완료** (건설처럼 여러 턴 소요되지 않음)
- 자원은 확장 명령 시 선차감
- 확장은 **낮 페이즈에서만** 가능

#### 2.5 슬롯 해금 메커니즘
- 각 방벽 레벨에 해금되는 슬롯 목록이 `WallExpansionLevelData`에 정의
- 슬롯은 `BuildingSlot.State.Hidden` → `BuildingSlot.State.Empty`로 전환
- 기존 건설 시스템의 슬롯 상태 머신을 그대로 활용

#### 2.6 시스템 해금 (시나리오별 선택적)
- 확장 레벨에 연동하여 게임 시스템을 해금할 수 있음
- **해금 가능한 시스템 예시**: 탐사 탭, 연구소 접근, 정책 시스템, 교역 등
- 해금 여부와 레벨은 `WallExpansionLevelData.unlockedFeatures` 배열로 기획자가 설정
- 첫 시나리오: 튜토리얼 느낌으로 Lv.1에서 자원 건물, Lv.2에서 연구, Lv.3에서 탐사 등 점진적 학습

#### 2.7 방어 범위 트레이드오프
- 방벽 확장 시 방어 범위(적 스폰 포인트 수 or 진입 경로)가 증가
- 레벨별 방어 범위 증가량은 SO에서 설정

| 레벨 | 적 진입 경로 수 | 필요 방어 타워 (최소) |
|---|---|---|
| Lv.0 | 1 | 0~1 |
| Lv.1 | 2 | 1~2 |
| Lv.2 | 3 | 2~3 |
| Lv.3 | 4 | 3~4 |
| Lv.4 | 4~5 | 4~5 |

### 데이터 흐름

```
[입력]                              [처리]                           [출력]
─────────────────                 ─────────────────               ─────────────────
확장 명령 (플레이어)      →     WallExpansionManager       →     방벽 레벨 증가
자원 보유량               →     (비용 검증 + 차감)          →     자원 차감 or 부족 경고
WallExpansionDataSO       →     (슬롯 해금 처리)           →     Hidden→Empty 슬롯 전환
                                (시스템 해금 처리)          →     기능 해금 이벤트 발행
                                (방어 범위 갱신)            →     적 스폰 포인트 업데이트
```

## 4. 밸런스 가이드

### 조정 가능 파라미터

| 파라미터 | 조정 목적 | 권장 범위 |
|---|---|---|
| 레벨별 기초 자원 비용 | 확장 진입 문턱 | Lv.1: 40~80, Lv.2: 80~120, Lv.3: 120~180, Lv.4: 160~240 |
| 레벨별 고급 자원 비용 | 고급 자원 수요 조절 | Lv.1: 0, Lv.2: 10~30, Lv.3: 30~70, Lv.4: 60~100 |
| 레벨별 해금 슬롯 수 | 확장당 성장 체감 | 2~4개/레벨 |
| 레벨별 추가 스폰 포인트 | 방어 부담 증가율 | 0~2개/레벨 |
| minTurn (레벨별) | 확장 속도 제한 | Lv.1: 0, Lv.2: 3~5, Lv.3: 7~10 |
| requiresResearch (레벨별) | 연구 연동 강제 여부 | 시나리오별 설정 |
| 확장 연출 시간 | 연출 체감 | 1.0~2.0초 |

### 밸런스 기준

- **첫 확장 타이밍**: 2~3턴 내에 Lv.1 확장 가능해야 함 (초기 자원 100 기준, 비용 60이면 즉시 가능)
- **확장 간격**: 각 확장 사이 최소 2~3턴의 자원 축적 기간 필요 (급확장 방지)
- **방어 압박 비례**: Lv.2 확장 시 방어 타워 1개 추가 건설이 필요한 수준의 적 진입 경로 증가
- **확장 안 하면 안전하지만 정체**: 확장 없이 Lv.0에서 버틸 수 있지만, 자원 생산이 부족하여 후반 웨이브 대응 불가
- **풀 확장 시 리스크**: 모든 방어 경로를 커버하려면 인력/자원의 80% 이상을 방어에 투입 필요

## 5. 엣지 케이스

| 상황 | 처리 방법 |
|---|---|
| 최대 레벨에서 확장 시도 | 확장 버튼 비활성화 + "최대 확장 완료" 표시 |
| 자원 부족 시 확장 시도 | 버튼 비활성화 + "자원 부족" 툴팁 (부족 자원 종류/수량 표시) |
| 연구 미완료 시 확장 시도 | 버튼 비활성화 + "연구 필요: [연구명]" 툴팁 |
| 밤 페이즈에서 확장 시도 | 확장 UI 자체가 비활성화 (낮 전용) |
| 확장 연출 중 턴 종료 시도 | 연출 완료까지 턴 종료 버튼 비활성화 |
| 해금된 슬롯에 이미 건물이 있는 경우 | 발생 불가 — Hidden 슬롯은 건물 없음이 보장됨 |
| 시나리오에 WallExpansionDataSO 없음 | 기본값 사용 (4레벨, 기본 비용, 시스템 해금 없음) |
| 확장 직후 밤 전투 진입 | 새 스폰 포인트 즉시 활성화 — 방어 준비 없이 확장 시 위험 |
