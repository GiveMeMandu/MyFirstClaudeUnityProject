## 3. 핵심 파일 요약

### README 요약
6개월 프로덕션 사용(TypeScript 마이크로서비스 50,000+ 라인)에서 검증된 Claude Code 인프라. 핵심 문제 "스킬이 자동으로 활성화되지 않는다"를 `UserPromptSubmit` 훅 + `skill-rules.json` 조합으로 해결했다. 15~30분 내에 프로젝트에 통합 가능하도록 설계된 레퍼런스 라이브러리.

### 진입점 (핵심 파일 3개)
| 파일 | 역할 |
|---|---|
| `skill-activation-prompt.ts` | UserPromptSubmit 훅. 프롬프트를 분석해 관련 스킬을 자동 제안. 완전 범용 (커스터마이징 불필요) |
| `post-tool-use-tracker.sh` | PostToolUse 훅. Edit/Write 후 수정된 파일 추적 및 컨텍스트 관리. 완전 범용 |
| `skill-rules.json` | 모든 스킬의 트리거 조건 정의. keywords, intentPatterns, fileTriggers, contentPatterns, enforcement 레벨 설정 |

### 설정 파일
- `package.json` (hooks/): tsx 의존성만 포함 — 매우 경량
- `settings.json`: 훅 등록 예시. **그대로 복사하면 안 됨** (모노레포 전용 Stop 훅 포함)

## 4. 라이브러리 평가

### 장점

- **즉시 적용 가능한 핵심 2개 훅**: `skill-activation-prompt` + `post-tool-use-tracker`는 커스터마이징 없이 모든 프로젝트에서 동작
- **500-line 규칙 + 점진적 공개(Progressive Disclosure)**: 스킬을 `SKILL.md`(500줄 미만) + `resources/` 하위 파일로 분리하여 컨텍스트 한계를 우회하는 검증된 패턴
- **skill-rules.json 고급 기능**: `block` vs `suggest` enforcement, `skipConditions` (세션 추적, 파일 마커 `// @skip-validation`, 환경변수), `contentPatterns`(파일 내용 기반 트리거) 등 현재 우리 구현에는 없는 기능 포함
- **에이전트 라이브러리**: `code-architecture-reviewer`, `refactor-planner`, `documentation-architect` 등 즉시 사용 가능한 전문 에이전트 10개
- **개발 문서 패턴**: 컨텍스트 리셋 후에도 프로젝트 지식 보존하는 3파일 구조 (`plan.md` + `context.md` + `tasks.md`)

### 단점 / 주의사항

- **웹 프레임워크 특화**: `backend-dev-guidelines`(Express/Prisma), `frontend-dev-guidelines`(React/MUI v7)은 Unity/C# 프로젝트에 직접 사용 불가
- **Stop 훅 위험성**: `tsc-check.sh`, `trigger-build-resolver.sh`는 특정 모노레포 구조 전용 — 그대로 사용 시 Stop 이벤트가 블록될 수 있음
- **settings.json 직접 복사 금지**: 비존재 서비스명(`blog-api`, `auth-service` 등) 참조 포함

### Project_Sun 도입 적합성

| 컴포넌트 | 적합성 | 이유 |
|---|---|---|
| `skill-activation-prompt` 훅 | ✅ 이미 동일 구현 있음 | 우리 프로젝트에 동일 훅이 존재하며 동작 중 |
| `post-tool-use-tracker` 훅 | ✅ 이미 동일 구현 있음 | Unity 파일 타입 감지까지 추가된 개선 버전 보유 |
| `skill-developer` 스킬 | ✅ **즉시 도입 권장** | 기술 무관 메타-스킬. 스킬 개발 방법론(500-line 규칙 등) 습득에 유용 |
| `skill-rules.json` 고급 패턴 | ✅ **부분 도입 권장** | `block` enforcement, `skipConditions`, `contentPatterns` 등 현재 미구현 기능 참고 |
| 에이전트 10개 | ✅ **선택적 도입 권장** | `code-architecture-reviewer`, `refactor-planner` 등은 Unity C# 코드에도 적용 가능 |
| `dev-docs` 슬래시 커맨드 | ⚠️ 경로 수정 필요 | `dev/active/` 경로 → `Docs/dev-docs/`로 변경 필요 |
| backend/frontend 스킬 | ❌ 부적합 | Node.js/React 특화. Unity/C# 전용 스킬을 별도 제작해야 함 |

## 5. 학습 포인트

### 배울 수 있는 패턴/기법

- **500-line 규칙**: 스킬을 메인 파일 + resources/ 하위 파일로 분리해 컨텍스트 한계 우회
- **점진적 공개(Progressive Disclosure)**: 필요할 때만 리소스 파일 로드
- **Guardrail vs Domain 스킬**: `block`(실수 방지 강제) vs `suggest`(권고) 구분
- **세션 추적 기반 skip**: 같은 세션 내 반복 경고 방지 (`skills-used-{session_id}.json`)
- **파일 마커 skip**: `// @skip-validation` 주석으로 특정 파일 검증 건너뛰기
- **contentPatterns**: 파일 내용 기반 스킬 트리거 (경로 패턴만으로 부족할 때)
- **개발 문서 3파일 패턴**: 컨텍스트 리셋 생존 전략

### 참고할 코드 구조

- `.claude/skills/skill-developer/SKILL.md` — 스킬 작성 베스트 프랙티스 완전 정리
- `.claude/skills/skill-developer/SKILL_RULES_REFERENCE.md` — skill-rules.json 스키마 완전 레퍼런스
- `.claude/skills/skill-developer/TRIGGER_TYPES.md` — 트리거 타입별 상세 가이드
- `.claude/skills/skill-developer/PATTERNS_LIBRARY.md` — 복사해서 쓸 수 있는 패턴 모음
- `.claude/hooks/skill-activation-prompt.ts` — 우리 구현과 비교하여 개선점 파악

### 추천 학습 순서

1. **README.md** — 전체 철학과 컴포넌트 구조 파악
2. **`.claude/skills/skill-developer/SKILL.md`** — 스킬 개발 방법론 핵심 이해
3. **`.claude/skills/skill-developer/SKILL_RULES_REFERENCE.md`** — skill-rules.json 고급 기능 (block, skipConditions 등)
4. **`.claude/skills/skill-developer/TRIGGER_TYPES.md`** — contentPatterns 등 미사용 트리거 타입 학습
5. **`CLAUDE_INTEGRATION_GUIDE.md`** — 컴포넌트 통합 시 주의사항 및 기술 스택 호환성 체크 패턴
6. **에이전트 파일들** (`code-architecture-reviewer.md` 등) — Project_Sun에 도입할 에이전트 선정
