# Sid Meier's Civilization V 레퍼런스 분석

- **분석일**: 2026-03-24
- **장르**: 4X 턴제 전략 (eXplore, eXpand, eXploit, eXterminate)
- **플랫폼**: Windows, macOS, Linux
- **개발사**: Firaxis Games / 2K (퍼블리셔)
- **출시일**: 2010-09-23 (Gods & Kings 2012, Brave New World 2013)
- **엔진**: 자체 엔진 (Lua 스크립팅)
- **평가**: Steam 압도적 긍정적 (95%, 77,839 리뷰) / Metacritic 90

## 1. 게임 개요

### 핵심 컨셉
석기 시대부터 정보화 시대까지 문명을 이끌어가는 턴 기반 전략 게임. 4X 장르의 정수로, **탐험(Explore)→확장(Expand)→개발(Exploit)→정복(Exterminate)** 4가지 축을 중심으로 수십 개의 시스템이 유기적으로 연결된다. 시리즈 5편에서 헥스 그리드와 1유닛/타일(1UPT) 제한을 도입하여 전투 시스템을 근본적으로 혁신했다.

리드 디자이너 Jon Shafer의 설계 철학: **"게임에서 유일하게 중요한 것은 플레이어 머릿속의 경험(the only thing which matters in a game is the experience inside the player's head)"**

### 4X 장르 공통 특징
4X는 1990년대 Master of Orion에서 정립된 전략 게임 하위 장르로, 다음 4가지 핵심 축을 공유한다:

| 축 | 설명 | Civ5에서의 구현 |
|---|---|---|
| **eXplore (탐험)** | 미지의 영역을 탐색하여 자원·지형·상대 문명 발견 | 전쟁의 안개(Fog of War), 정찰 유닛, 자연 경이 발견 보너스 |
| **eXpand (확장)** | 영토를 넓히고 거점을 늘림 | 개척자로 도시 건설, 문화 경계 확장, 도시국가 동맹 |
| **eXploit (개발)** | 자원을 효율적으로 활용하여 문명 발전 | 타일 개선, 기술 연구, 교역로, 전문가 배치 |
| **eXterminate (정복)** | 군사력으로 경쟁자 제거 | 헥스 전투, 도시 포위, 핵무기, 정복 승리 |

4X 게임의 설계 과제:
- **초반 재미 vs 후반 지루함**: 탐험/확장이 끝나면 관리 피로 급증 → Civ5는 BNW에서 관광/세계 의회로 후반 콘텐츠 보강
- **승리 조건 다양성**: 단일 승리 조건은 최적 빌드 고착 → Civ5는 5가지 승리 조건으로 다양한 플레이 스타일 수용
- **시스템 상호의존성**: 개별 시스템이 고립되면 깊이 부족 → Civ5는 자원·기술·정책·외교가 상호 참조

### 핵심 플레이 루프
```
[턴 시작]
    ├─ 도시 관리: 생산 큐, 시민 배치, 전문가 할당
    ├─ 기술 연구: 기술 트리에서 다음 연구 선택
    ├─ 사회 정책: 축적된 문화로 정책 채택
    ├─ 군사/외교: 유닛 이동, 전투, 외교 협상
    ├─ 경제: 교역로 관리, 자원 교환
    └─ 종교/스파이: (G&K) 선교사 파견, 스파이 배치
[턴 종료] → 모든 문명 행동 → 다음 턴
```
- **1턴**: 수 초(초반) ~ 수 분(후반, 대규모 제국)
- **1게임**: 4~12시간 (표준 속도)

### 주요 시스템 목록
| 시스템 | 한줄 설명 |
|---|---|
| 턴제 구조 | 모든 문명이 순차적으로 행동하는 턴 기반 시스템 |
| 기술 트리 | 석기시대→정보화시대까지 ~80개 기술의 선형+분기 연구 |
| 도시 경영 | 타일 활용, 건물 건설, 시민 배치, 생산 큐 관리 |
| 사회 정책 | 문화 포인트로 해금하는 정부/사회 운영 방식 |
| 헥스 전투 | 헥스 그리드 + 1UPT 기반 전술 전투 |
| 외교 | AI 지도자와의 협상, 조약, 도시국가 관계 |
| 자원 관리 | 전략/사치 자원의 수급과 교역 |
| 교역로 | (BNW) 도시 간/문명 간 자동 교역 |
| 종교 | (G&K) 신앙 포인트로 종교 창시/전파 |
| 관광/문화 | (BNW) 걸작/유적 기반 관광 vs 문화 대결 |
| 세계 의회 | (BNW) 결의안 투표, 외교 승리 경로 |

## 2. 시스템별 상세 분석

### 2.1 턴제 구조
- **메커니즘**: 모든 플레이어(AI 포함)가 번갈아 한 턴씩 행동. 한 턴에 모든 유닛 이동, 도시 생산 지정, 외교 협상 등을 수행. 턴 종료 시 생산/연구/문화 등 자원이 누적 계산됨. 게임 속도(빠름/표준/서사시/마라톤)에 따라 연구·생산 비용 배율 조정.
- **플레이어 경험**: 시간 압박 없이 숙고하여 의사결정 가능. "한 턴만 더(One More Turn)" 중독성의 원천. 모든 결정이 장기적 결과를 만들어 전략적 깊이 제공.
- **기술적 구현 추정 (Unity)**: 턴 매니저가 페이즈 상태머신 관리 (PlayerPhase → AIPhase → ResolutionPhase → NextTurn). 각 페이즈에서 이벤트 시스템으로 시스템별 업데이트 트리거. Project_Sun의 4타임(아침/점심/저녁/야간) 구조와 자연스럽게 매핑 가능 — 각 타임이 하나의 "턴"으로 기능.
- **참고 자료**: [Revisiting the Design of Civ 5](https://www.gamedeveloper.com/design/revisiting-the-design-of-civ-5)

### 2.2 기술 트리
- **메커니즘**: ~80개 기술이 시대별(고대→정보화)로 배치. 각 기술은 선행 기술 1~2개를 요구하는 방향 그래프(DAG) 구조. 기술 해금 시 새 건물, 유닛, 개선 시설, 자원 활용, 사회 정책이 열림. 연구 속도 = 도시의 과학(비커) 출력 합계 / 기술 비용.
- **설계 핵심**:
  - **선택의 폭**: 같은 시대 내에서 어떤 기술을 먼저 연구할지 선택 → 문명마다 다른 발전 경로
  - **게이트키핑**: 핵심 기술(예: 철기, 화약)이 유닛/건물 트리를 분기 → 전략적 우선순위 강제
  - **시대 전환**: 일정 기술 해금 시 시대 진입 → UI/음악/비주얼 변화로 진보 체감
- **플레이어 경험**: 매 턴 "다음에 뭘 연구할까"라는 의미 있는 선택. 기술 선점의 군사적/경제적 우위 → 연구 방향이 곧 전략.
- **기술적 구현 추정 (Unity)**: ScriptableObject로 기술 노드 정의 (ID, 이름, 시대, 비용, 선행기술 리스트, 해금 목록). 그래프 탐색으로 해금 가능 기술 필터링. UI는 노드 기반 트리 뷰어(ScrollRect + 줌/팬). 연구 진행도는 float 누적 → 1.0 도달 시 해금 이벤트 발행.
- **참고 자료**: [Carl's Guides - Gameplay Concepts](https://www.carlsguides.com/strategy/civilization5/gameplayconcepts.php)

### 2.3 도시 경영
- **메커니즘**: 도시 중심 타일 + 반경 3칸 내 워킹 타일. 각 타일은 식량/생산/금/과학/문화/신앙 산출. 시민 수 = 일할 수 있는 타일 수. 시민을 전문가 슬롯에 배치하면 타일 산출 대신 특수 보너스 획득. 건물은 산출 보너스 또는 새로운 전문가 슬롯 제공.
- **핵심 자원 산출 체계**:
  | 자원 | 역할 | 도시 성장 영향 |
  |---|---|---|
  | 식량(Food) | 인구 성장 | 잉여 식량 → 인구 증가 → 더 많은 타일/전문가 |
  | 생산(Production) | 건물/유닛 완성 속도 | 높을수록 빠른 건설 |
  | 금(Gold) | 유지비, 구매, 외교 | 적자 시 유닛 해산 |
  | 과학(Science) | 기술 연구 속도 | 인구 비례 기본 산출 |
  | 문화(Culture) | 사회 정책/국경 확장 | 정책 채택 간격 결정 |
  | 신앙(Faith) | 종교 유닛/건물 구매 | (G&K) 종교 시스템 구동 |
- **플레이어 경험**: 도시 하나하나가 미니 경영 시뮬레이션. 식량 vs 생산 vs 과학의 균형 잡기. 도시 전문화 (과학 도시, 생산 도시, 금 도시) 전략.
- **기술적 구현 추정 (Unity)**: 도시 엔티티에 ResourceOutput 컴포넌트(6종 자원). 타일 매니저가 워킹 타일 할당/해제. 턴 종료 시 CityUpdateSystem이 모든 도시의 산출 계산→성장/생산 진행도 업데이트. ECS 활용 시 IComponentData로 자원 데이터, SystemBase로 배치 계산.
- **참고 자료**: [Civilization Wiki - Civilization V](https://civilization.fandom.com/wiki/Civilization_V)

### 2.4 사회 정책
- **메커니즘**: 문화 포인트 누적 → 임계치 도달 시 정책 1개 채택. 10개 정책 트리(전통, 자유, 명예, 신앙, 후원, 미학, 상업, 탐험, 합리, 이념), 각 트리 5~6개 정책. 트리 개시(Opener) → 중간 정책 → 마무리(Finisher) 구조. 마무리 보너스는 강력하여 트리 완성에 인센티브. 이념(자유/질서/독재)은 후반 강력한 보너스 + 외교 영향.
- **설계 핵심**:
  - **기회비용**: 문화 자원은 유한 → 모든 정책을 채택할 수 없음 → 문명 정체성 형성
  - **시너지 설계**: 전통+미학(문화 특화), 자유+합리(과학 특화) 등 조합별 시너지
  - **시대 게이트**: 일부 트리는 특정 시대부터 해금 → 게임 진행에 따른 선택지 점진적 확장
- **플레이어 경험**: 정책 선택이 곧 플레이 스타일 선언. "나는 군사 국가" vs "나는 문화 강국"의 정체성 부여. 이념 선택은 후반 게임의 핵심 결정점.
- **기술적 구현 추정 (Unity)**: PolicyTree ScriptableObject (트리 구조, 각 노드에 효과 리스트). PolicyManager가 문화 임계치 체크 → 채택 가능 정책 리스트 제공. 효과는 스탯 수정자(StatModifier) 패턴으로 도시/유닛에 적용. UI는 트리 형태의 정책 선택 화면.
- **참고 자료**: [Carl's Guides - Social Policies](https://www.carlsguides.com/strategy/civilization5/gameplayconcepts.php)

### 2.5 헥스 전투 시스템
- **메커니즘**: 헥스 그리드 + 1유닛/타일(1UPT) 제한. 근접/원거리 유닛 구분. 지형 보너스(숲 +25% 방어, 언덕 +25% 방어, 강 건너기 페널티). 도시 포위전 메커니즘. 경험치 → 승진(프로모션) 시스템.
- **플레이어 경험**: 이전작의 둠 스택(한 타일에 유닛 쌓기)이 사라지고 진정한 전술적 포지셔닝 등장. 지형 활용, 측면 공격, 병목 방어 등 공간적 사고 요구.
- **기술적 구현 추정 (Unity)**: 헥스 그리드는 큐브 좌표계(Cube Coordinates)로 구현. A* 경로 탐색. 전투 계산은 데미지 공식(공격력 vs 방어력 + 지형 보정 + 승진 보너스). DOTS ECS로 대규모 유닛 시뮬레이션 가능.
- **참고 자료**: [Wikipedia - Civilization V](https://en.wikipedia.org/wiki/Civilization_V)

### 2.6 외교 시스템
- **메커니즘**: AI 지도자별 고유 성격(전쟁광, 평화주의자, 과학 중시 등). 호감도 시스템(양수/음수 수정자 누적). 조약 체결(개방 국경, 연구 협정, 방위 조약, 동맹). 도시국가는 독립 세력으로 금/유닛/퀘스트로 동맹 가능. 세계 의회(BNW)에서 결의안 투표.
- **설계 한계 (디자이너 회고)**: Shafer는 AI를 "변덕스럽고 예측 불가능"하게 만들려 했으나, 결과적으로 플레이어가 외교에 투자할 동기를 잃게 만들었다고 인정. → 교훈: **예측 가능한 규칙 위에 성격을 얹어야** 외교가 전략적 의미를 가짐.
- **참고 자료**: [Revisiting the Design of Civ 5](https://www.gamedeveloper.com/design/revisiting-the-design-of-civ-5)

### 2.7 자원/교역 시스템
- **메커니즘**: 전략 자원(철, 말, 석유 등 - 수량 기반)과 사치 자원(상아, 향신료 등 - 행복도 제공). BNW에서 교역로 추가: 캐러밴/화물선이 도시 간 금/과학/종교 자동 운반. 교역로 개수 제한 → 어디로 보낼지 전략적 선택.
- **설계 한계**: Shafer 회고 — 자원의 자급자족이 너무 쉬워 교역의 필요성이 약화. 건강 시스템 제거로 비전략 자원(식량류) 가치 하락.
- **기술적 구현 추정 (Unity)**: Resource ScriptableObject(타입, 산출량, 행복도 보너스). TradeRoute 시스템이 경로별 수익 계산(거리, 시대, 건물 보너스). 경영 시뮬레이션에서 교역은 핵심 수입원으로 활용 가능.

### 2.8 승리 조건 시스템
- **메커니즘**: 5가지 승리 조건이 동시에 추적됨
  | 승리 조건 | 달성 방법 |
  |---|---|
  | 정복 승리 | 모든 문명의 수도 점령 |
  | 과학 승리 | 우주선 부품 6개 건설 → 발사 |
  | 문화 승리 | (BNW) 모든 문명에 관광 영향력 "지배적" 달성 |
  | 외교 승리 | 세계 의회 투표에서 과반 확보 |
  | 점수 승리 | 2050년(500턴) 시점 최고 점수 |
- **설계 핵심**: 여러 승리 조건이 동시에 열려 있어 게임 중반에도 전략 전환 가능. 각 승리 조건이 서로 다른 시스템 체인을 요구 → 리플레이 가치.
- **플레이어 경험**: "이번에는 문화 승리를 노려볼까"라는 런별 목표 설정. 다른 문명의 승리 경로를 견제하는 메타 전략.

## 3. 아트/사운드 스타일

### 비주얼 스타일
- **아트 디렉션**: 아르데코(Art Deco) 스타일. 1920~30년대 뉴욕 건축(엠파이어 스테이트, 크라이슬러 빌딩)에서 영감. UI 전반에 기하학적 장식, 금색 라인, 대칭적 패턴 적용.
- **지도/지형**: 3D 렌더링 헥스 타일. 시대별 건물 외관 변화. 자연 경이(그랜드 캐니언, 에베레스트 등) 특수 3D 모델.
- **캐릭터**: 지도자를 사실적 3D 모델로 표현 (카툰 아닌 리얼리스틱). 각 지도자의 고유 배경, 의상, 애니메이션.
- **색감/톤**: 지형은 자연스러운 색감, UI는 금/갈색/크림 계열의 고급스러운 톤.

### UI/UX
- **HUD 레이아웃**: 상단 바에 핵심 자원(과학/금/행복/문화/신앙) 요약. 하단에 유닛/도시 정보 패널. 미니맵 우하단.
- **정보 계층**: UI 요소 크기로 중요도 반영 (Shafer의 가장 자랑스러운 성과). 자주 쓰는 기능은 크게, 드문 기능은 숨김.
- **인터랙션**: 클릭 기반 + 단축키. 턴 종료 버튼이 화면 중앙 하단에 큼직하게 배치 → "한 턴만 더" 유도.
- **기술 트리 UI**: 스크롤 가능한 횡방향 트리 뷰. 시대별 색상 구분. 해금 가능 기술 하이라이트.

### 사운드 디자인
- **BGM 전략**: 각 문명 지도자별 고유 테마곡. 해당 문화의 전통 멜로디를 기반으로 작곡 (예: 비스마르크 = 환희의 송가 어두운 변주). 평화/전쟁 상태에 따라 같은 멜로디의 다른 편곡 재생.
- **오케스트레이션**: 심포니 오케스트라 + 민족 악기(코라, 특수 현악기 등) 조합. "세계적 느낌(worldly feel)"을 위해 실제 민족 악기 사용 필수.
- **인터랙티브 오디오**: 전투 시 유닛 타입, 지형, 카메라 위치에 따라 사운드 동적 변화. 화면 4분면별 독립 환경음 생성. 유닛 그룹(최대 15개체) 단위의 컨텍스트 기반 전투 사운드.
- **시대별 음악 변화**: 시대 전환 시 음악 스타일도 함께 진화 → 고대의 간소한 악기에서 현대의 풀 오케스트라로.
- **작곡**: Geoff Knorr, Michael Curran. Logic Pro 기반 제작 → 라이브 오케스트라 녹음.

## 4. 장단점 분석

### 잘한 점
- **4X의 교과서적 구현**: 탐험→확장→개발→정복의 4축이 모두 의미 있게 작동하며, 각 축이 고유한 재미를 제공
- **헥스 + 1UPT 혁신**: 시리즈 최대 전투 혁신. 둠 스택 제거로 전술적 깊이 대폭 향상
- **UI 정보 계층**: 요소 크기로 중요도를 직관적으로 전달. 초보자 접근성과 숙련자 효율 동시 달성
- **문명별 정체성**: 고유 유닛, 건물, 능력 + 고유 음악까지 → 43개 문명 각각의 플레이 경험 차별화
- **BNW의 후반 콘텐츠**: 관광/세계 의회가 4X의 고질적 문제인 "후반 지루함"을 상당 부분 해결
- **기술 트리 설계**: 선형+분기의 DAG 구조가 "가이드된 자유(guided freedom)"를 완벽 구현
- **사회 정책의 기회비용**: 모든 것을 가질 수 없는 설계 → 매 런마다 다른 문명 성격
- **인터랙티브 사운드**: 문명별 테마, 전쟁/평화 변주, 시대별 진화 → 몰입감 극대화
- **"한 턴만 더" 중독성**: 턴제의 명확한 진행 단위 + 매 턴 의미 있는 결정 → 시간 감각 상실급 몰입

### 아쉬운 점
- **AI 품질**: 디자이너 본인이 인정한 최대 약점. 전투 AI가 특히 취약 → 1UPT 전술을 제대로 활용 못함
- **외교 불투명성**: AI 행동이 "변덕스럽고 예측 불가능"하여 외교 투자의 전략적 가치 약화
- **자원 자급자족 문제**: 교역의 필요성이 낮아 경제 시스템 깊이 감소
- **후반 관리 피로**: 도시/유닛 수 증가에 따른 마이크로 매니지먼트 부담 (4X 장르 공통 과제)
- **바닐라 vs 확장팩 격차**: BNW 없는 바닐라는 후반 콘텐츠 빈약 → 확장팩 의존도 높음
- **멀티플레이어 밸런스**: 특정 문명(바빌론, 폴란드 등)의 압도적 강세 → 메타 고착

## 5. Project_Sun 적용 포인트

### 직접 적용 가능
| 요소 | 적용 방법 | 우선순위 |
|---|---|---|
| 4타임 턴제 구조 | Civ5의 턴 구조를 아침/점심/저녁/야간 4타임으로 변환. 각 타임에 행동 포인트 배분 → "한 타임만 더" 중독성. TurnManager가 페이즈 상태머신으로 타임 전환 관리 | 높음 |
| 기술 트리 | DAG 기반 기술 트리 구조 그대로 차용. ScriptableObject 노드 그래프. 기술 해금이 건물/유닛/자원 활용을 열어주는 게이트키핑 구조 | 높음 |
| 자원 산출 체계 | Civ5의 6종 자원(식량/생산/금/과학/문화/신앙) 참고하여 Project_Sun에 맞는 3~5종 핵심 자원 설계. 각 자원이 서로 다른 시스템을 구동 | 높음 |
| 승리 조건 다양성 | 단일 승리가 아닌 복수 승리 조건 → 리플레이 가치. 방어 생존, 경제 달성, 영토 확장 등 | 중간 |
| UI 정보 계층 | 요소 크기 = 중요도 원칙. 상단 바에 핵심 자원 요약, 하단에 상세 정보. 미니맵 배치 | 중간 |

### 변형하여 적용
| 요소 | 변형 내용 | 이유 |
|---|---|---|
| 도시 경영 → 거점 경영 | Civ5의 다중 도시 관리 → Project_Sun에서는 **단일 거점의 심층 경영**. 타일 산출 대신 건물 배치/인력 배분으로 자원 생산. 시민 전문가 시스템을 NPC 역할 배정으로 변환 | 경영 시뮬레이션에서 하나의 기지를 깊이 관리하는 것이 더 적합 |
| 사회 정책 → 운영 방침 | 문화 포인트 대신 **평판/영향력 포인트**로 해금. 방어 중심/경제 중심/탐험 중심 등 방침 트리. 한 트리에 집중하면 보너스, 다방면은 범용적 → 빌드 다양성 | 경영 시뮬의 "경영 스타일" 선택으로 자연스럽게 변환 |
| 턴 진행 속도 | Civ5는 순수 턴제 → Project_Sun은 **실시간이지만 4타임으로 나뉜 하이브리드**. 각 타임 내에서 실시간 이벤트(적 공격, 자원 생산) 진행, 타임 전환 시 결산/업그레이드 시간 | 실시간의 긴박감 + 턴제의 전략적 숙고를 모두 확보 |
| 외교 → NPC 관계 | 문명 간 외교 → **NPC 파벌/상인/용병과의 관계**. Shafer의 교훈 적용: 관계 수치가 투명하게 표시되어 예측 가능한 행동 보장 | Civ5 외교의 실패 교훈을 바로 반영 |
| 교역로 → 거점 물류 | 도시 간 교역 → **자원 수송 라인** 또는 외부 상인 교역. 교역로 개수 제한 → 어떤 자원을 어디서 가져올지 전략적 선택. Civ5의 자급자족 문제를 피하기 위해 핵심 자원을 교역 필수로 설계 | 경영 시뮬에서 물류 관리는 핵심 재미 요소 |
| DOTS 대규모 전투 | Civ5의 1UPT 소규모 전술 → **DOTS 기반 대규모 적 물량**. 헥스 그리드의 공간적 사고는 유지하되, 개별 유닛이 아닌 부대 단위 관리. 수천 엔티티 동시 시뮬레이션 | Dome Keeper 레퍼런스와 결합하여 물량 방어의 스펙터클 구현 |

### 영감만 참고
- **"한 턴만 더" 중독성 설계** — 매 타임(턴)마다 의미 있는 결정이 있어야 다음 타임으로 넘어가고 싶은 욕구 발생
- **시대 전환 연출** — 기술 트리 진행에 따른 비주얼/사운드 변화로 문명 발전 체감 (건물 외관 업그레이드, 음악 스타일 진화)
- **문명별 정체성 시스템** — 향후 플레이어 세력/팩션 시스템 구현 시, 각 팩션에 고유 능력/유닛/건물 부여
- **인터랙티브 사운드 시스템** — 게임 상태(평화/전쟁, 아침/야간)에 따라 BGM 레이어가 동적으로 변화하는 구조
- **Jon Shafer의 설계 교훈**: "단순하고 투명한 시스템이 복잡하고 우아한 시스템보다 낫다", "AI 행동은 예측 가능해야 전략적 의미를 가진다", "자급자족이 쉬우면 교역이 죽는다"
