# Thronefall 레퍼런스 분석

- **분석일**: 2026-03-24
- **장르**: 미니멀리스트 전략 / 타워 디펜스 / 킹덤 빌더
- **플랫폼**: Windows, macOS, Nintendo Switch, iOS
- **개발사**: GrizzlyGames / Mythwright (퍼블리셔)
- **출시일**: 2024-10-11 (얼리 액세스 2023-08)
- **엔진**: Unity
- **평가**: Steam 압도적 긍정적 (96%, 10,511 리뷰)
- **개발 규모**: 2인 팀

## 1. 게임 개요

### 핵심 컨셉
중세 왕국을 낮에 건설하고 밤에 방어하는 미니멀리스트 전략 게임. **"전략 게임에서 불필요한 복잡성을 모두 제거(strip a classic strategy game from all unnecessary complexity)"**하는 것이 핵심 설계 철학. 개발자는 "게임 시간이 부족한 성인을 위한 깊지만 시간을 덜 잡아먹는 경험"을 목표로 했다.

설계 접근법: **"플레이어가 무엇을 느끼길 원하는가?"를 먼저 정하고, 그 감정을 전달하는 메커니즘을 역설계.**

### 핵심 플레이 루프
```
[낮 페이즈 - 건설]                    [밤 페이즈 - 방어]
  │                                      │
  ├─ 골드로 건물 배치 (고정 슬롯)        ├─ 다방향 적 웨이브 진입
  ├─ 기존 건물 업그레이드                ├─ 방어탑/벽/유닛 자동 전투
  ├─ 군사 유닛 배치/재배치               ├─ 왕(플레이어) 직접 전투 참여
  └─ 경제 건물 수입 확인                 └─ 생존 시 골드 보상
         │                                     │
         └────── 밤 생존 → 새 낮 ──────────────┘
                 (반복, 웨이브 에스컬레이션)
```
- **1런**: 15~30분 (맵 하나)
- **핵심 긴장**: 경제 투자(수입 건물) vs 방어 투자(군사 건물) 균형

### 주요 시스템 목록
| 시스템 | 한줄 설명 |
|---|---|
| 낮/밤 사이클 | 낮에 건설·업그레이드, 밤에 웨이브 방어의 2페이즈 루프 |
| 고정 건설 슬롯 | 미리 정해진 위치에만 건물 배치, 자유 배치 없음 |
| 건물 업그레이드 | 기존 건물을 골드로 강화, 여러 단계 업그레이드 가능 |
| 경제 시스템 | 농장/광산/항구 등 수입 건물 → 밤 생존 시 골드 획득 |
| 군사 유닛 | 기사, 궁수, 특수 영웅 등 자동 전투 유닛 |
| 방어 구조물 | 벽, 타워, 성소 등 고정 방어 시설 |
| 왕 전투 | 플레이어가 직접 전장에서 전투 (9종 무기) |
| 퍽/뮤테이터 | 50+ 퍽으로 전략 커스터마이징 + 난이도 변경자 |
| 맵 시스템 | 10개 핸드메이드 맵, 각각 고유 지형/건설 슬롯 배치 |

## 2. 시스템별 상세 분석

### 2.1 낮/밤 사이클
- **메커니즘**: 낮과 밤이 교대하며 각 페이즈에서 다른 활동 수행. 낮에는 건설/업그레이드/배치만 가능, 밤에는 적 웨이브가 진입하여 전투 발생. 밤 생존 시 경제 건물로부터 골드 획득 → 다음 낮에 투자. 웨이브가 진행될수록 적 물량/종류 에스컬레이션.
- **시각적 전환**: 낮→밤 전환 시 조명이 따뜻한 황금빛에서 차가운 청색으로 변화. 건물에 불이 켜지고, 적 진입 경로에 붉은 표시 등장. 밤→낮 전환 시 "생존했다"는 안도감의 시각적 해방.
- **설계 영감**: Kingdom 시리즈에서 영감. 단, Kingdom은 밤에 플레이어가 수동적 → Thronefall은 왕 직접 전투로 능동적 해결.
- **플레이어 경험**: 낮의 "고요한 계획"과 밤의 "혼란스러운 즉흥 대응"의 대비 → 감정 진폭. 밤 생존 후 낮이 돌아오면 "안전하다"는 심리적 보상. 이 리듬이 중독성의 핵심.
- **기술적 구현 추정 (Unity)**: DayNightCycleManager가 페이즈 상태 관리. 조명 시스템: 글로벌 Directional Light의 색온도/강도 Lerp + Post-processing (Color Grading) 전환. 페이즈 전환 이벤트로 건설 UI 활성화/비활성화, 스폰 매니저 시작/정지.
- **참고 자료**: [Game Developer - Mastering Minimalism](https://www.gamedeveloper.com/design/mastering-minimalism-and-layering-complexity-with-strategy-game-thronefall)

### 2.2 고정 건설 슬롯 시스템
- **메커니즘**: 맵에 미리 배치된 건설 노드(슬롯)에만 건물 배치 가능. 각 슬롯은 특정 건물 타입만 수용 (경제 슬롯, 군사 슬롯, 방어 슬롯 등). "무엇을 어디에"가 아닌 **"언제, 어떤 순서로"**가 핵심 의사결정.
- **설계 철학**: 자유 배치 제거가 역설적으로 **"전례 없는 레벨 디자인 가능성(unheard of level design possibilities)"**을 열었다(개발자 인터뷰). 각 맵의 슬롯 배치가 곧 퍼즐 → 맵별 최적 전략이 달라짐. 제약이 창의성을 낳는다(constraint breeds creativity).
- **장점**:
  - 의사결정 단순화 → 분석 마비 방지
  - 밸런스 조정 용이 (맵 디자이너가 난이도 정밀 제어)
  - 건물 배치 애니메이션의 일관된 연출 가능
  - 신규 플레이어 접근성 극대화
- **단점**:
  - 자유도 부족 → 창의적 방어 전략 한계
  - 반복 플레이 시 "최적 빌드 순서" 고착
- **플레이어 경험**: 복잡한 배치 고민 없이 "이 골드를 경제에 쓸까 방어에 쓸까"에 집중 → 깔끔한 의사결정.
- **기술적 구현 추정 (Unity)**: BuildSlot MonoBehaviour (슬롯 위치, 수용 가능 건물 타입 리스트, 현재 건물 참조). BuildManager가 빈 슬롯 하이라이트 + 클릭 시 건물 선택 UI 표시. 건물 배치 시 스케일 애니메이션 + 파티클 + 사운드 → "한 번의 클릭으로 큰 권능을 행사하는 느낌".
- **참고 자료**: [PC Gamer - Build and defend](https://www.pcgamer.com/build-and-defend-a-nice-little-kingdom-in-microstrategy-game-thronefall/)

### 2.3 경제 시스템
- **메커니즘**: 경제 건물(농장, 광산, 항구 등)이 밤 생존 시 골드 생산. 골드는 다음 낮에 건설/업그레이드에 사용. 핵심 트레이드오프: 초반에 경제 건물을 많이 지으면 수입은 높지만 방어가 약해져 밤에 위험. 방어에 투자하면 안전하지만 성장 둔화.
- **설계 핵심**: 자원이 "골드" 단일 통화 → 모든 결정이 같은 자원을 놓고 경쟁 → 기회비용이 명확. 복수 자원 없이도 깊은 전략적 깊이 달성.
- **플레이어 경험**: "이번 밤만 버티면 다음에는 크게 투자할 수 있어" → 성장의 기대감. 경제 투자가 복리처럼 쌓이는 스노우볼링의 쾌감.

### 2.4 퍽/뮤테이터 시스템
- **메커니즘**: 경험치 레벨업으로 퍽 해금 (50+개). 퍽은 수동적 보너스(유닛 강화, 경제 보너스 등). 뮤테이터는 난이도 변경자 — 적 강화/플레이어 약화를 자발적으로 선택하면 추가 보상. 패배해도 경험치 획득 → 절대 좌절하지 않는 진행.
- **복잡성 레이어링**: 초반에는 기본 메커니즘만 → 퍽 해금으로 점진적 옵션 확장 → 숙련 후 뮤테이터로 자발적 난이도 상승. "쉬운 입문, 깊은 마스터리" 구현.
- **플레이어 경험**: 패배가 좌절이 아닌 "경험치 = 진행"으로 느껴짐 → 리트라이 동기. 퍽 조합이 빌드 다양성 제공 → 같은 맵도 다르게 플레이.
- **기술적 구현 추정 (Unity)**: Perk ScriptableObject (ID, 효과 리스트, 해금 레벨). PerkManager가 활성 퍽 관리 → StatModifier 패턴으로 효과 적용. Mutator는 GameModifier로 적 스탯/스폰률 조정. 경험치는 PlayerProfile에 영구 저장.

### 2.5 웨이브 방어 시스템
- **메커니즘**: 밤마다 다방향에서 적 웨이브 진입. 웨이브 수와 적 종류는 맵/난이도별 고정 디자인. 적 유형별 다른 행동 (근접 러시, 원거리, 공성, 비행 등). 방어탑/벽/유닛이 자동 교전. 왕(플레이어)은 직접 전투 참여 가능.
- **적 진입 경로 표시**: 밤 시작 전 적 진입 방향을 시각적으로 미리 알림 → 유닛 재배치 기회 → 전략적 대응 가능.
- **기술적 구현 추정 (Unity)**: WaveManager가 웨이브 테이블(적 타입, 수량, 방향, 타이밍) 관리. 적은 NavMeshAgent 또는 DOTS 기반 경로 이동. 방어탑은 범위 내 적 자동 타겟팅 (OverlapSphere + 우선순위 정렬). 대규모 물량은 ECS로 확장.

### 2.6 맵 디자인
- **메커니즘**: 10개 핸드메이드 맵. 각 맵은 고유한 지형, 건설 슬롯 배치, 적 진입 경로를 가짐. 맵별로 최적 전략이 다름 → 같은 게임이지만 맵마다 새로운 퍼즐.
- **설계 핵심**: 고정 슬롯 시스템 덕분에 디자이너가 맵별 난이도/전략을 정밀 제어 가능. 절차적 생성이 아닌 핸드크래프트 → 각 맵이 고유한 경험.

## 3. 아트/사운드 스타일

### 비주얼 스타일
- **렌더링**: 셀 셰이딩(Cel-shading) + 로우 폴리(Low-poly) 3D. 외곽선(Outline) 강조로 형태 구분.
- **색감/톤**: 절제된 색상 팔레트. 낮에는 따뜻한 황금/녹색, 밤에는 차가운 청색/보라. 적군은 붉은/어두운 톤으로 즉각 구별.
- **캐릭터/환경**: 심플한 3D 모델이지만 실루엣으로 유닛 타입 즉시 인식 가능. 중세 유럽풍 건물이 미니어처/디오라마 느낌.
- **시각적 기능성**: "높은 가독성(high legibility)과 매력적 분위기, 게임플레이의 캐주얼함을 동시에 전달"하는 것이 아트 목표. 정보 전달이 미학보다 우선.
- **낮→밤 전환 연출**: 색온도 점진 변화 + 건물 불빛 점등 + 하늘색 변화 → 2페이즈의 감정적 전환을 시각으로 강화

### UI/UX
- **HUD**: 극도로 미니멀. 골드 수량 + 현재 웨이브 정보 정도만 상시 표시. 건설 시 해당 슬롯 근처에 옵션 팝업.
- **건설 인터페이스**: 슬롯 클릭 → 건설 가능 건물 아이콘 표시 → 선택 → 배치 애니메이션. 전체 과정이 2클릭 이내.
- **정보 과부하 방지**: 전략 게임 특유의 복잡한 패널/수치 최소화. 필수 정보만 직관적 표시.
- **건물 배치 피드백**: 배치 시 메시 스케일 애니메이션 + 파티클 시스템 + 사운드 → "간단한 버튼 하나로 큰 권능을 행사하는(feel like they can exercise great power with a simple button press)" 느낌

### 사운드 디자인
- **BGM 스타일**: 중세풍 오케스트라. 낮에는 평화로운 멜로디, 밤에는 긴장감 있는 전투 음악으로 전환. 트랙 수는 적지만 품질이 높음.
- **효과음 특징**: 건물 배치의 "쿵" 소리 (배치의 무게감/권능감), 군대 집결 나팔 소리 (밤 준비 신호), 전투 중 금속 충돌/화살 소리.
- **페이즈 전환 사운드**: 밤 시작 시 경고 호른 → 긴장감. 밤 종료 시 안도의 팡파르 → 보상감.

## 4. 장단점 분석

### 잘한 점
- **미니멀리즘의 완성**: "제거의 미학"이 게임플레이, 아트, UI 전체에 일관되게 적용 → 모든 요소가 목적을 가짐
- **낮/밤 사이클의 감정 설계**: 계획의 고요함 → 전투의 혼란 → 생존의 안도가 반복되며 감정 진폭 극대화
- **고정 슬롯의 역설적 깊이**: 자유 배치 제거가 레벨 디자인 가능성을 오히려 확장. "무엇을 어디에"가 아닌 "무엇을 먼저"에 집중
- **즉각적 건설 피드백**: 스케일 애니메이션 + 파티클 + 사운드 → 한 클릭의 만족감. Dome Keeper의 "즉각 체감" 원칙과 동일
- **복잡성 레이어링**: 쉬운 입문 → 퍽 해금으로 점진적 깊이 → 뮤테이터로 하드코어 도전. 커브가 자연스러움
- **패배의 긍정적 설계**: 패배해도 경험치 획득 → 좌절 대신 진행감
- **셀 셰이딩의 기능성**: 로우폴리가 "예쁘면서도 정보 전달에 최적화" → 소규모 팀의 아트 효율 + 전략적 가독성 동시 달성
- **2인 팀 100만 카피**: 미니멀리즘이 개발 효율(적은 에셋, 빠른 반복)과 플레이어 경험 모두에 이점 → 인디 개발의 교과서

### 아쉬운 점
- **자유도 부족**: 고정 슬롯으로 인한 창의적 배치 불가 → 전략의 표현적 자유도 희생
- **반복 시 최적 빌드 고착**: 맵별 "정답 순서"가 존재 → 숙련 후 도전감 감소 (뮤테이터로 부분 보완)
- **BGM 부족**: 트랙 수 적음 → 장시간 플레이 시 반복감
- **직접 전투 밸런스**: 왕의 전투력이 너무 높으면 방어 건물 의미 약화, 너무 낮으면 존재감 부족 → 미세한 밸런스 과제
- **콘텐츠 볼륨**: 10맵은 핵심 루프 대비 적은 편 (퍽 조합으로 리플레이 보강)
- **멀티플레이어 부재**: 솔로 전용 (코옵/대전 모드 없음)

## 5. Project_Sun 적용 포인트

### 직접 적용 가능
| 요소 | 적용 방법 | 우선순위 |
|---|---|---|
| 낮/밤 시각 전환 연출 | 4타임(아침/점심/저녁/야간)별 조명·색온도·포스트프로세싱 자동 전환. Thronefall의 황금빛 낮→청색 밤 대비를 참고하되, 4단계 그라데이션으로 확장. 각 타임 전환 시 짧은 컷씬/페이드 연출 → 시간 경과의 시각적 만족감 | 높음 |
| 고정 건설 슬롯 + 업그레이드 분기 | 건물 배치는 Thronefall처럼 고정 슬롯. **차별화: 슬롯에 배치된 건물이 업그레이드 트리에 따라 분기** (예: 농장 슬롯 → Lv2에서 "대규모 농장"(식량 특화) vs "시장"(교역 특화) 선택). Frostpunk 법률의 비가역 분기 + Thronefall의 고정 슬롯 결합 | 높음 |
| 건물 배치 피드백 연출 | 배치 시 스케일 애니메이션 + 파티클 + 사운드 "쿵" → 한 클릭의 권능감. Thronefall 개발자 인사이트: "simple button press = great power" | 높음 |
| 경제 vs 방어 트레이드오프 | 4타임 중 경영 타임(아침/점심)에 투자한 경제가 밤 방어 이후 수입으로 환수. 경제 비중 ↑ → 밤 위험 ↑, 방어 비중 ↑ → 성장 둔화. Thronefall의 단일 통화(골드) 기회비용 구조 차용 | 높음 |
| 적 진입 경로 미리보기 | 밤(방어 타임) 시작 전 적 진입 방향/규모를 시각적으로 미리 표시 → 유닛 재배치 기회 → 전략적 준비감 | 중간 |
| 복잡성 레이어링 | 초반에는 기본 건설/방어만 → 게임 진행에 따라 기술 트리, 법률, 원정대, 퍽 순차 해금. Thronefall의 "쉬운 입문 → 점진적 깊이" 원칙 | 중간 |

### 변형하여 적용
| 요소 | 변형 내용 | 이유 |
|---|---|---|
| 2페이즈(낮/밤) → 4타임 | Thronefall의 낮=건설/밤=방어 → **아침(건설)·점심(경영/교역)·저녁(원정/이벤트)·야간(방어)**. 각 타임에 특화된 활동 배정. 타임별 조명/음악 전환으로 Thronefall의 시각적 만족감을 4배로 확장 | 2페이즈의 단순함 대신 더 다양한 활동과 의사결정 지점 제공 |
| 왕 직접 전투 → 지휘관 모드 | Thronefall의 왕 액션 → **우선 배제**. 대신 지휘관으로서 유닛 배치/명령에 집중. 추후 "영웅 유닛" 직접 조작을 선택적 확장 가능 | 경영 시뮬레이션 정체성 우선. 액션 요소는 단계적 도입 |
| 단일 통화(골드) → 다중 자원 | Thronefall의 골드 단일 통화 → **3~5종 자원(Frostpunk/Civ5 참고)**. 단, 각 자원의 역할이 명확히 분리되어야 Thronefall적 직관성 유지 | 경영 시뮬 깊이를 위해 다중 자원 필요하지만, 역할 명확성으로 복잡성 통제 |
| 퍽/뮤테이터 → 법률+운영방침 | Thronefall의 수동 퍽 → **Frostpunk 법률 + Civ5 사회정책**으로 통합. 뮤테이터(자발적 난이도 상승) → "위기 조건 수락"(적 강화 대신 보상 증가) 이벤트로 변환 | 퍽의 빌드 다양성 + 법률의 내러티브 무게감 결합 |
| 핸드메이드 맵 → 핸드메이드 스테이지 + 절차적 변형 | 10개 고정 맵 → **핸드메이드 스테이지 골격 + 자원/적/이벤트 배치 절차적 변형**. 고정 슬롯 위치는 스테이지별 고정, 자원 분포/적 구성은 런마다 변화 → 리플레이성 보강 | Thronefall의 "최적 빌드 고착" 문제 해결 |
| 패배 시 경험치 → 영구 진행 시스템 | 패배해도 "지식/기술 포인트" 일부 획득 → 다음 런에서 초반 해금 가속. Thronefall의 긍정적 패배 설계 + 로그라이크적 메타 진행 | 좌절감 최소화 + 리트라이 동기 |

### 영감만 참고
- **"제약이 창의성을 낳는다(constraint breeds creativity)"** — 고정 슬롯이 자유 배치보다 더 나은 레벨 디자인을 가능케 한 사례. Project_Sun에서도 제약을 두려워하지 말 것
- **미니멀 UI 철학** — 전략 게임이라도 화면에 수치/패널을 최소화하면 접근성 극대화. 필수 정보만 표시, 상세는 선택적 조회
- **2인 팀 100만 카피의 교훈** — 로우폴리/셀셰이딩 아트가 개발 효율(적은 에셋, 빠른 이터레이션)과 시각적 매력을 동시에 달성. Project_Sun 아트 스타일 결정 시 고려
- **"플레이어가 무엇을 느끼길 원하는가?"를 먼저 정하라** — 메커니즘 전에 감정 설계. 각 시스템의 목적을 "어떤 느낌을 주는가"로 검증
- **페이즈 전환 사운드의 역할** — 경고 호른(밤 시작)과 안도 팡파르(밤 종료)가 감정 전환의 청각적 앵커
