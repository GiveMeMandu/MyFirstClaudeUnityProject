# Frostpunk 레퍼런스 분석

- **분석일**: 2026-03-24
- **장르**: 사회 생존 시뮬레이션 / 도시 건설 / 자원 관리
- **플랫폼**: Windows, macOS
- **개발사**: 11 bit studios (개발/퍼블리싱)
- **출시일**: 2018-04-24
- **엔진**: 자체 엔진 (Liquid Engine)
- **평가**: Steam 매우 긍정적 (92%, 37,869 리뷰) / Metacritic 84
- **비고**: Project_Sun이 가장 많이 참고하는 핵심 레퍼런스

## 1. 게임 개요

### 핵심 컨셉
화산 겨울로 빙하기가 닥친 대체 역사 세계에서, 지구 마지막 도시의 지도자로서 증기 기관 기술로 생존하는 **"최초의 사회 생존 게임(first society survival game)"**. 일반 도시 건설 게임과 달리, **자원 효율이 아니라 도덕적 선택과 사회 통치가 핵심**. 개발사는 "이 게임은 샌드박스 도시 건설이 아니라 정치와 리더십에 관한 스토리 드리븐 게임"이라고 명시.

핵심 질문: **"생존을 위해 인간성을 얼마나 포기할 것인가?"**

### 핵심 플레이 루프
```
[자원 수급] ──석탄/나무/철/식량 채집──→ [도시 건설/확장]
      │                                        │
      │                                   건물 배치, 히트존 관리
      │                                        │
      ↓                                        ↓
[법률 제정] ←──희망/불만 게이지 변동──← [시민 관리]
      │                                        │
      │    ┌──────────────────────────────┐     │
      └───→│  원정대 파견 (Frostland 탐험) │←────┘
            │  생존자/자원/스팀코어 발견    │
            │  이벤트 선택지 → 결과        │
            └──────────────────────────────┘
                         │
               [온도 하강 이벤트] → 위기 → 3막 구조 진행
```
- **1게임**: 8~15시간 (시나리오 모드)
- **시간 단위**: 실시간 진행 (1x~3x 속도), 24시간 주기로 법률 제정 가능

### 주요 시스템 목록
| 시스템 | 한줄 설명 |
|---|---|
| 제너레이터 & 히트존 | 도시 중심 제너레이터의 동심원 열 공급 + 스팀허브 확장 |
| 자원 관리 | 석탄, 나무, 철, 식량, 스팀코어 5종 자원 수급/소비 |
| 법률의 서 (Book of Laws) | 돌이킬 수 없는 법률 제정 → 적응/목적(질서 vs 신앙) 분기 |
| 희망/불만 게이지 | 시민의 사기(Hope)와 불만(Discontent) 이중 지표 |
| 기술 트리 | 난방/자원·식량/건강·주거 3가지 탭, 5티어 연구 |
| 원정대 (Scouting) | Frostland 탐험, 생존자 구출, 자원 발견, 이벤트 선택 |
| 시민 관리 | 노동자/엔지니어/아이 3분류, 작업 배치, 질병/부상 |
| 3막 내러티브 | 학습→내부갈등→최종 폭풍의 서사 구조 |
| 무한 모드 | 인내/건축가/세레니티 3모드, 8맵 |

## 2. 시스템별 상세 분석

### 2.1 제너레이터 & 히트존 시스템
- **메커니즘**: 도시 중심에 제너레이터가 위치. 석탄을 소비하여(기본 6석탄/시간) 동심원 형태의 히트존 생성. 히트존 내 건물만 정상 기능. 기술 연구로 히트존 범위 확대(Range 업그레이드) 및 온도 상승(Power 업그레이드, 석탄 소비 2배씩 증가). 스팀허브를 배치하면 제너레이터와 동일 수준의 보조 히트존 생성.
- **건물 배치 제약**: 동심원 도로가 제너레이터를 중심으로 방사형 확장. 히트존 밖 건물은 난방 없이 생산성 저하/시민 동사. 공간 효율과 열 효율의 트레이드오프.
- **플레이어 경험**: "석탄이 떨어지면 모두 죽는다"는 근본적 공포. 히트존 경계에 건물을 세울지 말지의 고민. 오버드라이브(긴급 과열)는 제너레이터 폭발 리스크 → 극단적 긴장감.
- **기술적 구현 추정 (Unity)**: 제너레이터 위치 중심 원형 영역 계산. 각 건물에 HeatZone 소속 플래그. 온도 시뮬레이션은 글로벌 온도 + 히트존 보너스 + 건물 단열 보너스로 계산. Project_Sun에서는 히트존 대신 **방어 범위(Defense Zone)** 개념으로 변환 가능.
- **참고 자료**: [Frostpunk Wiki - Generator](https://frostpunk.fandom.com/wiki/Generator)

### 2.2 법률의 서 (Book of Laws)
- **메커니즘**: 게임 내 24시간마다 1개 법률 제정 가능. 선택하면 **되돌릴 수 없음**. 이진 분기 — 한쪽을 선택하면 다른 쪽은 영구히 잠김 (예: 아동 노동 vs 아동 교육). 2대 분기:
  - **적응(Adaptation)**: 초반 생존 기본법 (식량 배급, 노동 시간, 의료)
  - **목적(Purpose)**: 중후반 사회 이념 선택
    - **질서(Order)**: 감시→경비대→선전대→독재 (효율적이나 억압적)
    - **신앙(Faith)**: 기도→성소→순례→광신 (단결력 있으나 맹목적)
- **설계 핵심**:
  - **점진적 타락**: 개발자 의도 — "전체주의 국가가 어떻게 탄생하는지" 보여주기. 작은 양보가 큰 억압으로 이어짐
  - **기회비용 극대화**: 모든 법률이 이점과 대가를 동시에 가짐 → 순수한 좋은 선택은 없음
  - **게임 일시정지**: 법률 선택 시 게임이 멈추며, 플레이어에게 숙고 시간 강제
  - **후반 역설**: 최종 법률(독재/광신)은 게임 클리어에 불필요 → 이념적 선택으로 전환
- **플레이어 경험**: "이번 한 번만..."이라는 합리화의 연쇄. 게임 끝에서 자신의 법률 기록을 보며 도덕적 성찰. "이것이 정당했는가?(Was it worth it?)" 질문.
- **기술적 구현 추정 (Unity)**: LawTree ScriptableObject (노드: ID, 이름, 전제조건, 효과, 배타적 쌍 ID). LawManager가 쿨다운(24h) 관리 + 채택 시 효과 적용. 효과는 StatModifier + EventTrigger 조합. UI는 트리 뷰 + 선택 확인 모달.
- **참고 자료**: [Gabriel Chauri - Player's Decisions in the Book of Laws](https://www.gabrielchauri.com/frostpunk-decisions/)

### 2.3 희망/불만 게이지
- **메커니즘**: 두 개의 독립적 게이지가 시민 심리 상태를 추적.
  - **희망(Hope)**: 0이 되면 추방(게임 오버). 긍정적 이벤트, 법률, 식량 공급, 의료 등으로 상승.
  - **불만(Discontent)**: 최대치 도달 시 반란(게임 오버). 과로, 식량 부족, 사망, 가혹한 법률로 상승.
  - 두 게이지는 **독립적으로 변동** — 동시에 희망이 낮고 불만이 높을 수 있음
  - 법률의 영향은 "약간 증가/감소" 같은 상대적 표현으로 표시 → 정확한 수치를 숨겨 전략적 불확실성 유지
- **설계 핵심**: 전통적 행복도 시스템(단일 지표)과 달리, 이중 지표로 더 복잡한 사회 상태 표현. 희망 없이 불만도 없는 상태(무기력)도, 희망 있지만 불만도 높은 상태(혁명 전야)도 가능.
- **플레이어 경험**: 두 게이지를 동시에 관리하는 저글링. 가혹한 결정은 단기 불만 증가이지만 장기 생존 기여 → 단기 vs 장기 트레이드오프.
- **기술적 구현 추정 (Unity)**: MoraleSystem에 Hope/Discontent float 값 (0~1). 각 이벤트/법률/상태가 MoraleModifier를 발행. 임계치 체크 시스템이 게임오버 조건 모니터링. UI는 화면 상단 이중 바 + 변동 피드백 아이콘.
- **참고 자료**: [Game Developer - Emotional Narrative Engagement](https://www.gamedeveloper.com/design/frostpunk-an-analysis-of-emotional-narrative-engagement)

### 2.4 원정대 (Scouting) 시스템
- **메커니즘**: 비콘(Beacon) 건설 후 정찰대 파견 가능. Frostland 맵에서 위치(Location) 간 이동. 각 위치에 도착하면 자원, 생존자, 이벤트 발견.
  - **이동**: 위치 간 이동에 실제 시간 소요 (수 시간~1일+). 이동 중에는 도시 내 인력 부재.
  - **위치 유형**: 자원 매장지, 생존자 캠프, 폐허 도시, 특수 이벤트 장소
  - **이벤트 선택**: 위치 도착 시 선택지 제시 (예: 위험한 동굴 탐색 vs 안전하게 무시, 생존자 호위 vs 자원만 회수)
  - **리스크**: 일부 위치에서 정찰대 전멸 가능 (곰 공격, 함정 등). 확률 기반 + 선택지 의존.
  - **전초기지(Outpost)**: 특정 위치에 영구 기지 설립 → 매일 자원 자동 공급. 핵심 전략 자원.
  - **스팀코어**: 원정으로만 획득 가능한 희귀 자원 → 고급 건물/자동화 장치에 필수
- **설계 핵심**:
  - **FOMO와 리스크-리워드**: 탐험하지 않으면 스팀코어/생존자를 놓침 → 파견 동기. 하지만 인력이 도시에서 빠짐 → 생산력 저하.
  - **내러티브 전달**: 원정 이벤트가 세계관(빙하기의 참상)을 간접 체험시킴
  - **도시 밖의 전략 레이어**: 도시 관리만으로는 부족 → 외부 탐험이 게임의 깊이를 2배로 확장
- **플레이어 경험**: "밖에서 무슨 일이 일어나고 있을까"라는 호기심. 정찰대가 돌아올 때의 기대감과 불안감. 이벤트 선택지에서의 도덕적 고민 (도시와 동일한 딜레마를 외부에서도 체험).
- **기술적 구현 추정 (Unity)**: ExpeditionManager가 파견 팀 상태(이동중/탐색중/귀환중) 관리. Location ScriptableObject(좌표, 이벤트 리스트, 자원, 발견 조건). EventSystem이 선택지 UI 표시 → 결과 적용. 이동 시간은 거리 기반 계산. Project_Sun에서는 **타임 소비**로 변환 — 원정에 1~2타임 소요.
- **참고 자료**: [Frostpunk Wiki - Scouting](https://frostpunk.fandom.com/wiki/Scouting)

### 2.5 자원 관리 시스템
- **메커니즘**: 5종 핵심 자원
  | 자원 | 획득 방법 | 용도 |
  |---|---|---|
  | 석탄(Coal) | 채탄소, 탄광, 원정 | 제너레이터/스팀허브 연료, 생존의 근본 |
  | 나무(Wood) | 제재소, 벌목장, 원정 | 건물 건설, 초기 핵심 자원 |
  | 철(Steel) | 제철소, 원정 | 고급 건물, 기술 연구 |
  | 식량(Food) | 사냥꾼 오두막, 온실 | 시민 생존, 배급 법률에 따라 소비량 변동 |
  | 스팀코어(Steam Core) | 원정 전용 | 고급 자동화 건물 핵심 소재 (대체 불가) |
- **설계 핵심**: 석탄은 "생명줄" — 소비가 멈추면 죽음. 스팀코어는 원정 전용 → 외부 탐험의 가치 보장. 자원 부족이 법률 선택을 강제 (배급량 줄이기, 아동 노동 등).
- **참고 자료**: [Steam - Frostpunk](https://store.steampowered.com/app/323190/Frostpunk/)

### 2.6 기술 트리
- **메커니즘**: 워크숍(Workshop)에서 엔지니어가 연구. 3개 탭(난방 / 자원·식량 / 건강·주거), 각 5티어. 하위 티어 연구 완료 시 상위 해금. 연구 시간은 워크숍 수에 반비례(최대 병렬 연구 불가, 속도 보너스).
- **설계 핵심**: Civ5의 넓은 DAG와 달리, 좁고 깊은 직선+분기 구조. 선택지가 좁아 "무엇을 먼저"의 우선순위 결정이 핵심. 각 티어가 체감적 파워 점프 제공.
- **참고 자료**: [Frostpunk Wiki - Technology Tree](https://frostpunk.fandom.com/wiki/Technology_Tree)

### 2.7 3막 내러티브 구조
- **메커니즘**: 메인 시나리오(A New Home)가 3막으로 구성:
  - **1막 (학습)**: 기본 시스템 학습, 도시 기초 건설, 첫 번째 위기(식량/의료)
  - **2막 (내부 갈등)**: 런던파(도시 이탈 세력) 등장, 사회 분열, 법률 선택 압박
  - **3막 (최종 폭풍)**: 기온 -150°C까지 하강, 모든 시스템을 총동원한 극한 생존
- **설계 핵심**: 각 막이 다른 시스템을 강조 → 1막은 자원, 2막은 법률/희망, 3막은 모든 것의 종합 시험. 최종 폭풍은 "준비한 것이 충분했는가"를 검증하는 클라이맥스.
- **플레이어 경험**: 3막 진입 시 "이번에는 살아남을 수 있을까"라는 극적 긴장. 생존 후 "이것이 정당했는가?" 반성 → 게임의 정서적 여운.

### 2.8 시민 관리
- **메커니즘**: 시민은 노동자(Worker), 엔지니어(Engineer), 아이(Children) 3분류. 각 건물이 특정 유형의 시민을 요구. 질병/동상/사망 시스템. 과로/24시간 교대 법률로 생산성 상승 + 불만/질병 증가.
- **설계 핵심**: 시민이 단순 자원이 아닌 "관리해야 할 사회"임을 체감시킴. 건물에 사람을 배치하는 행위가 "누군가를 위험에 보내는 결정"으로 느껴짐.

## 3. 아트/사운드 스타일

### 비주얼 스타일
- **렌더링**: 3D 탑뷰 렌더링, 줌인 시 디테일한 건물/시민 확인 가능
- **색감/톤**: 극도로 차갑고 어두운 톤. 하얀 눈 + 어두운 금속 + 주황 불빛의 대비. "미니멀한 아트 스타일에서 모든 색의 한 점이 눈 속 온기의 터치처럼 느껴진다"
- **스팀펑크 미학**: 19세기 후반 산업혁명 + 증기 기관 기술. 굴뚝의 연기, 톱니바퀴, 금속 파이프가 주요 시각 요소
- **환경 디테일**: 눈 쌓인 지붕, 깜빡이는 조명, 건물의 점진적 마모 → 세계의 진정성(authenticity) 강화
- **동적 환경**: 온도 변화에 따라 눈 강도, 안개, 바람 효과 변화. 최종 폭풍에서 시각적 극한

### UI/UX
- **HUD**: 상단에 자원 바(석탄/나무/철/식량/스팀코어), 좌하단에 온도계, 하단에 희망/불만 게이지
- **법률 UI**: 전체 화면 전환, 책 형태의 법률 트리 → 몰입감 극대화, 게임 일시정지로 숙고 유도
- **원정 UI**: Frostland 맵 오버레이, 위치 아이콘 + 경로 표시
- **정보 계층**: 자원은 항상 표시, 상세 정보는 건물 클릭 시 패널 → 정보 과부하 방지
- **색상 팔레트**: 어두운 청회색 배경 + 주황/금색 하이라이트(따뜻함=생존의 상징)

### 사운드 디자인
- **BGM 스타일**: 오케스트라 + 앰비언트. 비극적이면서도 웅장한 분위기. Piotr Musiał 작곡. 바이올린/첼로 중심의 서정적 멜로디가 절망 속 희망을 표현.

## 4. 장단점 분석

### 잘한 점
- **도덕적 딜레마의 메커니즘화**: 도덕적 선택이 단순 텍스트가 아니라 실질적 게임플레이 영향(희망/불만/자원)으로 연결. 법률 한 줄이 사회를 바꿈
- **법률의 비가역성**: 되돌릴 수 없는 선택 → 모든 결정의 무게감 극대화. "점진적 타락"을 체험적으로 구현
- **이중 게이지의 긴장감**: 희망과 불만이 독립적으로 움직여 단일 지표보다 복잡하고 현실적인 사회 상태 표현
- **히트존 공간 전략**: 제너레이터 중심 동심원 배치가 도시 건설에 물리적 제약 부여 → 공간 최적화 퍼즐
- **원정대의 깊이 확장**: 도시 밖 탐험이 세계관 전달 + 핵심 자원(스팀코어) 공급 + 도덕적 딜레마를 3중으로 제공
- **3막 구조의 서사적 완결**: 학습→갈등→시험의 구조가 감정적 아크를 명확하게 형성
- **스팀펑크 미학의 완성도**: 시각/음향 디자인이 "차가운 세계에서의 따뜻한 생존"이라는 테마를 일관되게 지원
- **"한 법률만 더" 중독성**: Civ5의 "한 턴만 더"에 대응하는 Frostpunk만의 중독 루프

### 아쉬운 점
- **탑뷰 거리감**: 부감 시점이 시민 개개인과의 정서적 거리 생성 → 도덕적 선택의 감정적 무게 약화
- **리플레이 한계 (시나리오)**: 선형 내러티브로 정찰 이벤트가 동일 → 2회차 이후 원정의 서프라이즈 상실
- **실시간의 스트레스**: 위기 상황에서 시간이 흐르며 숙고 시간 부족 → 일시정지 의존도 높음
- **최적 빌드 수렴**: 경험 축적 후 "정답 루트" 고착 (스팀코어 우선, 특정 법률 순서 등)
- **후반 반복**: 시나리오 3막 이전 중반부가 자원 관리 반복으로 느려질 수 있음
- **AI 시민 한계**: 시민이 개별적 성격/스토리를 갖지 않아 "사회"보다 "자원"으로 느껴질 수 있음

## 5. Project_Sun 적용 포인트

> **핵심 참고 방향**: Frostpunk은 Project_Sun의 가장 가까운 레퍼런스. "추위 → 적 물량", "실시간 → 4타임 턴제"로 치환하면 핵심 설계가 거의 직접 대응됨.

### 직접 적용 가능
| 요소 | 적용 방법 | 우선순위 |
|---|---|---|
| 희망/불만 이중 게이지 | MoraleSystem에 Hope/Discontent 독립 게이지 구현. 법률/이벤트/전투 결과가 각각 수정자 발행. 임계치 도달 시 게임오버(반란/탈주). 변동량은 상대적 표현("약간"/"크게")으로 불확실성 유지 | 높음 |
| 법률 시스템 (비가역 분기) | BookOfLaws 시스템: 일정 주기(예: 3일마다)마다 법률 1개 제정. 이진 분기 + 비가역. 적응→목적(질서/신앙) 구조를 Project_Sun에 맞게 변형 (예: 군사노선 vs 외교노선, 생존주의 vs 인본주의) | 높음 |
| 원정대 시스템 | ExpeditionManager: 팀 편성(인력 배분) → Frostland 맵 대응 외부 맵 탐험 → 위치 도착 시 이벤트/선택지 → 자원/생존자/정보 획득. 타임 소비 기반 이동 (1~2타임). 스팀코어 같은 원정 전용 핵심 자원 설계 | 높음 |
| 3막 내러티브 구조 | 스테이지/챕터를 3막으로 구성: 1막(시스템 학습+기지 건설), 2막(내부 갈등+파벌 분열), 3막(대규모 적 총공세). 각 막이 다른 시스템을 강조 | 높음 |
| 자원 5종 체계 | Frostpunk의 5종(석탄/나무/철/식량/스팀코어) 참고하여 Project_Sun 자원 설계. 특히 "원정 전용 희귀 자원" 개념 차용 | 중간 |

### 변형하여 적용
| 요소 | 변형 내용 | 이유 |
|---|---|---|
| 제너레이터 히트존 → 방어 거점 범위 | 제너레이터 동심원 = **거점 방어탑의 방어 범위**. 범위 밖 건물은 적 공격에 취약. 방어탑 업그레이드로 범위 확장 (자원 소비 증가). 스팀허브 = 보조 방어탑 | 추위 대신 적 공격이 핵심 위협이므로, 열→방어로 1:1 치환 |
| 실시간 → 4타임 턴제 | Frostpunk의 실시간 24시간 주기 → **4타임(아침/점심/저녁/야간)으로 이산화**. 각 타임에 행동 배분. 법률 제정은 "야간 회의"에서만 가능 등 타임별 특화 가능 | Frostpunk의 "일시정지 의존" 문제 해결 + 턴제의 전략적 숙고 확보 |
| 온도 하강 → 적 웨이브 에스컬레이션 | 3막의 최종 폭풍(-150°C) → **최종 대규모 공세(DOTS 수천 엔티티)**. 시간 진행에 따라 적 물량/종류 에스컬레이션. 각 막 종료 시 보스급 웨이브 | Dome Keeper의 웨이브 방어 + Frostpunk의 3막 구조 결합 |
| 원정 이벤트 → 턴 소비형 탐험 | 원정 이동에 타임 소비. "아침에 출발 → 저녁에 도착" 같은 구조. 원정 중 인력 부재 = 도시 생산력 저하 → Frostpunk과 동일한 리스크-리워드. 이벤트는 **텍스트 어드벤처 형식**의 선택지 | Frostpunk의 리플레이 한계(고정 이벤트) 개선을 위해 이벤트 풀 랜덤화 |
| 시민 분류 확장 | 노동자/엔지니어/아이 → **전투원/기술자/민간인/특수(영웅 유닛)** 등 확장. 각 분류가 다른 건물/역할 전용. DOTS 기반 대규모 시민 시뮬레이션 | 전투가 핵심이므로 전투원 분류 필요. 민간인은 경영 시뮬 파트 담당 |
| 시민 개성 부여 | Frostpunk 약점(시민 = 자원 느낌) 보완: **핵심 NPC에 이름/성격/스토리** 부여. 파벌 리더, 핵심 엔지니어 등이 이벤트/법률에 반응 → 정서적 유대 강화 | 부감 시점의 거리감 보완. 법률/원정 선택의 감정적 무게 증가 |

### 영감만 참고
- **"최초의 사회 생존 게임"이라는 장르 정의** — Project_Sun도 단순 타워디펜스/경영 시뮬이 아닌, 사회 운영 시뮬레이션이라는 정체성 설정 가능
- **점진적 타락 내러티브** — 플레이어의 선택이 누적되어 의도치 않은 사회를 만들어가는 경험
- **스팀펑크 미학의 일관성** — 모든 시각/음향 요소가 하나의 테마를 지지하는 아트 디렉션
- **"이것이 정당했는가?" 엔딩 질문** — 게임 종료 시 플레이어의 선택 기록을 보여주며 성찰 유도
- **Frostpunk 1886 (2027)** — UE5로 리메이크 예정. 70% 원작 + 30% 신규 콘텐츠. 시각적 참고 가치
- **Civ5 교훈과의 시너지**: Frostpunk의 법률 비가역성 + Civ5의 기술 트리 구조 + Dome Keeper의 웨이브 방어 → Project_Sun의 3기둥
