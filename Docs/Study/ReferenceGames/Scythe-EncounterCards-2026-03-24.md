# Scythe 조우카드 시스템 레퍼런스 분석 (시스템 집중 모드)

- **분석일**: 2026-03-24
- **장르**: 4X 보드게임 / 엔진 빌딩 / 영역 지배
- **플랫폼**: 보드게임 (디지털 버전: Steam)
- **개발사**: Stonemaier Games (디자이너: Jamey Stegmaier)
- **출시일**: 2016
- **아트**: Jakub Rozalski (1920+ 유니버스)
- **분석 대상**: 조우카드(Encounter Card) 시스템

---

## 1. 게임 개요 (간략)

1920년대 대체 역사 유럽에서 거대 메카와 농민이 공존하는 세계. 5개 팩션이 영토 확장, 자원 관리, 전투, 건설을 통해 부(富)를 축적하는 4X 엔진 빌딩 보드게임. 조우카드는 맵 탐험 중 발생하는 **내러티브 선택 이벤트**로, 게임에 스토리텔링 레이어를 추가하는 핵심 시스템.

---

## 2. 조우카드 시스템 상세 분석

### 2.1 트리거 메커니즘

- **조우 토큰(Encounter Token)**: 게임 시작 시 맵의 특정 헥스에 배치
- **발동 조건**: 플레이어의 캐릭터(리더) 미니어처가 조우 토큰이 있는 헥스에 진입
- **이동 즉시 종료**: 조우 토큰이 있는 헥스에 도착하면 해당 턴의 이동이 종료 → 조우에 집중하도록 강제
- **일회성**: 토큰은 사용 후 맵에서 제거 → **게임 내 조우 횟수가 유한** → 희소성이 각 조우의 가치를 높임
- **타이밍**: 이동 액션이 완전히 끝난 후 카드를 뽑음 (다른 유닛 이동 완료 후)

### 2.2 카드 구조 — 3선택지 포맷

#### 기본판 (32장) 고정 포맷:

```
┌─────────────────────────────────────┐
│  [일러스트 - Jakub Rozalski 아트]    │
│                                     │
│  "당신은 폐허가 된 마을에서 굶주린   │
│   주민들을 발견합니다..."            │
│  (상황 묘사 텍스트)                  │
│                                     │
├─────────────────────────────────────┤
│  선택지 1: [무료 이득]               │
│  → 비용 없음, 소량 보상             │
│  예: 식량 2 획득                    │
│                                     │
│  선택지 2: [금화 지불 → 보상]        │
│  → 금화 비용, 중간 보상             │
│  예: 3금 지불 → 노동자 2 획득       │
│                                     │
│  선택지 3: [인기도 지불 → 보상]      │
│  → 인기도(평판) 비용, 큰 보상       │
│  예: 인기도 -2 → 전투력 +4          │
│                                     │
└─────────────────────────────────────┘
```

**핵심 설계 원리**:
| 선택지 | 비용 | 보상 | 설계 의도 |
|---|---|---|---|
| 1번 (안전) | 없음 | 소량 | 리스크 회피형 플레이어를 위한 보장된 이득 |
| 2번 (투자) | 금화 (물질적) | 중간 | 경제적 트레이드오프 — 자원 여유가 있을 때 유리 |
| 3번 (대가) | 인기도 (사회적) | 대량 | 도덕적/사회적 대가 — 게임 종료 시 인기도가 점수 배율에 영향 → 장기적 리스크 |

#### 확장판 Encounters (32장) 혁신 포맷:

기본판의 고정 포맷을 깨고 다양한 변형 도입:
- **비밀 선택지**: 괄호 안에 "비밀" 표시, 선택 시까지 다른 플레이어에게 비공개
- **지속 효과**: 일회성이 아닌 게임 잔여 기간 동안 유효한 능력 부여 (예: "매 전투 전 인기도 +1")
- **다른 플레이어 영향**: 다른 플레이어의 자원을 가져오거나 영향을 주는 선택지
- **팩션 능력 차용**: 다른 팩션의 고유 능력을 한시적으로 사용
- **카드 보관**: 조우카드를 버리지 않고 보관하여 나중에 사용

### 2.3 내러티브 통합 설계

- **아트 공유 의식**: 카드를 뽑으면 Jakub Rozalski의 일러스트를 **모든 플레이어에게 보여줌** → 테이블 전체가 장면을 목격
- **소리내어 읽기**: 상황 묘사를 소리내어 읽음 → 텍스트가 공유 서사가 됨
- **시각적 자유**: 일러스트에는 텍스트보다 더 많은 이야기가 담겨 있음 → 플레이어가 장면을 자유롭게 해석
- **세계관 전달**: 각 카드가 1920+ 세계의 일상(농민, 메카, 야생동물, 폐허)을 묘사 → 게임 메커니즘 밖의 세계 경험

### 2.4 수치/밸런스 분석

**기본판 선택지 밸런스 공식**:
```
선택지 1 기대가치 ≤ 선택지 2 기대가치 ≤ 선택지 3 기대가치
(비용 0)            (비용: 중간)           (비용: 높음)
```
- 모든 선택지가 "순이득" → 조우 자체가 보상 (패널티만 있는 카드 없음)
- 비용과 보상의 비율이 일정 → 어떤 선택이든 유효하지만, 게임 상황에 따라 최적이 달라짐
- **인기도 비용의 이중성**: 인기도는 게임 중 다른 용도로도 쓰이고, 게임 종료 시 점수 배율 → 장기적 기회비용

**카드 분포** (기본판 32장):
- 자원(식량/금속/오일/나무) 보상 카드: ~40%
- 군사(전투력/전투카드) 보상 카드: ~25%
- 인기도/노동자/이동 보상 카드: ~35%

### 2.5 상태 다이어그램

```
[맵 이동] → 조우 토큰 헥스 도착?
                │
          No    │    Yes
          │     ↓
          │   [이동 즉시 종료]
          │     ↓
          │   [조우 토큰 제거 (일회성)]
          │     ↓
          │   [카드 드로우]
          │     ↓
          │   [아트를 모든 플레이어에게 공개]
          │     ↓
          │   [상황 텍스트 소리내어 읽기]
          │     ↓
          │   [3개 선택지 확인]
          │     ├─ 비용 지불 가능?
          │     │   ├─ Yes → 선택 가능
          │     │   └─ No → 선택 불가 (회색 처리)
          │     ↓
          │   [선택지 1개 선택]
          │     ↓
          │   [비용 지불 + 보상 획득]
          │     ↓
          │   [카드 버림 (또는 확장: 보관)]
          │     ↓
          └───→ [턴 계속]
```

### 2.6 설계 철학 — Jamey Stegmaier의 원칙

1. **플레이어에게 항상 선택지를 줘라**: 선택지가 없는 강제 이벤트 금지. 에이전시(agency) 보장.
2. **모든 설정에 적용 가능해야 한다**: 특정 팩션/맵 위치에 의존하지 않는 범용 카드
3. **기존 조우와 일관성 유지**: 포맷의 일관성이 플레이어의 기대치를 형성 → 의사결정 속도 향상
4. **순수 패널티 금지**: 조우는 항상 보상 → 탐험에 대한 긍정적 인센티브
5. **비용 지불 불가 시에도 선택 가능**: 최소 1개 선택지(무료)는 항상 선택 가능

## 3. 아트/비주얼 (조우카드 관련)

### 비주얼 스타일
- **Jakub Rozalski 화풍**: 18~19세기 유럽 고전 회화 스타일 + 거대 메카 + 판타지 요소. 유화 질감의 따뜻한 색감. 넓은 풍경 속 인물과 메카의 스케일 대비.
- **조우카드 일러스트 특징**: 카드 상단 절반을 차지하는 대형 일러스트. 텍스트에 묘사된 것보다 더 많은 디테일 → 플레이어가 이야기를 상상하도록 유도. "몬스터가 항상 명확하지 않은" 은유적 장면 구성.
- **색감**: 따뜻한 황금빛 들판, 차가운 겨울 풍경, 안개 낀 폐허 등 다양한 분위기. 카드마다 고유한 톤.

### UI 구성 (카드 레이아웃)
- **상단**: 풀아트 일러스트 (카드 면적의 ~50%)
- **중단**: 상황 묘사 플레이버 텍스트 (이탤릭체, 2~3문장)
- **하단**: 3선택지 영역, 각 선택지에 비용 아이콘 + 보상 아이콘 + 간단한 설명
- **아이콘 시스템**: 자원/인기도/전투력 등을 직관적 아이콘으로 표시 → 텍스트 최소화
- **비용 불가 표시**: 비용을 지불할 수 없는 선택지는 시각적으로 구분

## 4. 장단점 분석 (조우카드 시스템)

### 잘한 점
- **3선택지 포맷의 명쾌함**: 무료/투자/대가라는 일관된 구조가 의사결정 프레임을 즉시 제공 → 분석 마비(analysis paralysis) 방지
- **모든 선택이 이득**: 순수 패널티 없음 → 조우를 피할 이유 없음 → 탐험 인센티브 극대화
- **비용 타입의 분리**: 물질적 비용(금화)과 사회적 비용(인기도)을 분리 → 서로 다른 전략 차원의 트레이드오프
- **내러티브와 메커니즘의 결합**: 일러스트+텍스트가 선택지에 감정적 맥락 부여 → 순수 수치 비교가 아닌 역할극적 선택
- **공유 서사 경험**: 아트 공개 + 소리내어 읽기 → 멀티플레이어에서 공동 스토리텔링
- **희소성의 가치**: 유한한 토큰 수 → 각 조우가 특별한 순간
- **확장성**: 팬 디자인 콘테스트로 확장판 제작 → 커뮤니티 참여 + 콘텐츠 확장 용이
- **비용 불가 시 대안 보장**: 돈이 없어도 1번 선택 가능 → 좌절감 방지

### 아쉬운 점
- **기본판 포맷 경직성**: 항상 무료/금화/인기도 순서 → 예측 가능 → 반복 플레이 시 서프라이즈 감소
- **솔로 플레이 약화**: 아트 공유/소리내어 읽기가 솔로에서는 의미 약화
- **맥락 독립성**: 카드가 맵 위치/팩션과 무관 → 세계관 몰입의 한계 (사막 헥스에서 숲 배경 카드가 나올 수 있음)
- **밸런스 고정**: 항상 3번이 "최고 보상"이므로, 인기도 여유가 있으면 3번만 고르는 최적 전략 고착
- **일회성 한계**: 카드 효과가 즉시 소비 → 장기적 서사 연결 부재 (확장판에서 일부 개선)

## 5. Project_Sun 적용 포인트

### 직접 적용 가능
| 요소 | 적용 방법 | 우선순위 |
|---|---|---|
| 3선택지 이벤트 포맷 | 모든 이벤트(원정대 + 3일 주기 정기 이벤트)에 3선택지 구조 적용. **①안전(무비용/소보상) ②투자(자원 비용/중보상) ③대가(희망 or 불만 비용/대보상)**. EventCard ScriptableObject에 3개 ChoiceData(비용 타입, 비용량, 보상 타입, 보상량, 희망/불만 수정자) 정의 | 높음 |
| 비용 타입 분리 | Scythe의 금화/인기도 분리 → Project_Sun에서 **물질적 비용(자원)과 사회적 비용(희망/불만 게이지)**으로 분리. 3번 선택지가 항상 희망 감소 or 불만 증가를 동반 → Frostpunk 법률 시스템과 시너지 | 높음 |
| 순수 패널티 없는 설계 | 모든 이벤트의 모든 선택지가 최소한의 이득 보장 → 이벤트 발생 자체를 긍정적으로 인식. "무시" 선택지 대신 "최소 이득" 선택지 | 높음 |
| 비용 불가 시 대안 보장 | 자원이 부족해도 1번(무료) 선택은 항상 가능 → 막다른 골목 방지. UI에서 비용 불가 선택지는 회색 처리하되, 1번은 항상 활성 | 중간 |
| 일러스트+텍스트 서사 | 각 이벤트에 고유 일러스트 + 2~3문장 상황 묘사 → 이벤트가 단순 수치 선택이 아닌 세계관 체험. 상단 일러스트(50%) + 하단 선택지(50%) 레이아웃 | 중간 |

### 변형하여 적용
| 요소 | 변형 내용 | 이유 |
|---|---|---|
| 고정 포맷 → 가변 포맷 | Scythe 기본판의 "항상 무료/금화/인기도" 고정 → **비용 타입을 카드별로 다르게**. 어떤 카드는 ①자원/②인력/③희망, 다른 카드는 ①무료/②시간(타임)/③자원+불만 등. Scythe 확장판의 혁신을 기본으로 채택 | 기본판의 예측 가능성 문제 해결. 매 이벤트마다 신선한 의사결정 |
| 일회성 → 이원화 (원정대 vs 정기) | **원정대 이벤트**: Scythe처럼 맵 위치 기반 일회성 → 희소성 + 탐험 보상. **3일 정기 이벤트**: 이벤트 풀에서 랜덤 드로우 → 반복 가능하되 조건부 필터링(시대/상황에 따라 풀 변경) | 원정대는 탐험 보상, 정기 이벤트는 거점 내부 사회 관리 역할 분리 |
| 맥락 독립 → 맥락 의존 | Scythe 약점(위치와 카드 무관) 보완: **이벤트에 발생 조건(Condition) 태그** 부여. 예: "전투 직후에만", "식량 부족 시에만", "야간에만", "특정 지역 원정 시에만". EventPool에서 조건 매칭으로 적절한 카드 필터링 | 세계관 몰입 강화. "왜 이 이벤트가 지금 발생했는지" 납득 가능 |
| 즉시 소비 → 지속 효과 혼합 | Scythe 확장판의 "카드 보관" + "지속 효과" 아이디어 채택. **일부 선택지는 영구 버프/디버프** (예: "이후 모든 전투에서 방어력 +5%, 대신 불만 지속 증가"). 법률 시스템(Frostpunk)의 비가역성과 연결 | 선택의 장기적 영향 → 결정의 무게감 증가 |
| 보드게임 공유 → 싱글 연출 | 아트 공유/소리내어 읽기 → **풀스크린 이벤트 연출**. 일러스트 페이드인 → 텍스트 타이핑 연출 → 선택지 순차 등장. BGM 변조(긴장/희망/슬픔)로 분위기 전달 | 싱글 플레이어 게임에서의 몰입감 극대화 |
| 3선택지 → 조건부 4번째 선택지 | 기본 3선택지 + **특정 조건 충족 시 숨겨진 4번째 선택지** 등장 (예: 특정 기술 연구 완료, 특정 NPC 동행, 이전 이벤트에서 특정 선택). Scythe 확장판의 "비밀 선택지" 발전형 | 리플레이 가치. "다음에는 4번째 선택지를 열어보자"는 동기 |

### 영감만 참고
- **팬 디자인 콘테스트 모델** — Scythe는 팬이 카드를 디자인하고 디자이너가 정제 → 커뮤니티 참여형 콘텐츠 확장 (모딩/UGC 시스템 고려 시)
- **아이콘 기반 비용/보상 표시** — 텍스트 최소화, 직관적 아이콘으로 비용/보상 즉시 파악 → UI 설계 원칙
- **Jakub Rozalski의 "은유적 장면"** — 조우 일러스트에서 위협이 항상 명확하지 않음 → 분위기와 긴장감을 직접 설명 대신 시각적으로 암시
- **비용 불가 시 그레이아웃** — 선택지를 숨기지 않고 표시하되 비활성화 → "저걸 선택하려면 자원이 더 필요하다"는 동기 부여

## 부록: Project_Sun 이벤트 시스템 설계 초안

Scythe 조우카드 + Frostpunk 법률/원정대를 결합한 Project_Sun 이벤트 시스템 구조:

### 이벤트 발생 체계
```
[3일 주기 정기 이벤트]          [원정대 이벤트]
  │                              │
  │ EventPool에서               │ 위치(Location)에
  │ 조건 매칭 드로우             │ 도착 시 트리거
  │                              │
  └──────────┬───────────────────┘
             ↓
    [이벤트 카드 표시]
    ┌─────────────────────┐
    │ [일러스트]           │
    │ [상황 묘사 텍스트]    │
    ├─────────────────────┤
    │ ① 안전 (무료/소보상) │ ← 항상 선택 가능
    │ ② 투자 (자원/중보상) │ ← 자원 부족 시 그레이아웃
    │ ③ 대가 (사회적/대보상)│ ← 희망/불만 영향
    │ ④ [숨김] 조건부 특수  │ ← 조건 충족 시에만 등장
    └─────────────────────┘
             ↓
    [선택 → 비용 지불 → 보상 획득]
             ↓
    [희망/불만 게이지 변동]
             ↓
    [일부: 영구 효과 적용 (법률과 연결)]
```

### EventCard ScriptableObject 구조 제안
```
EventCard
├── id: string
├── eventType: enum (Periodic / Expedition)
├── conditions: List<EventCondition>  // 발생 조건
│   ├── timeOfDay: enum (아침/점심/저녁/야간/Any)
│   ├── dayRange: (min, max)  // 게임 일차 범위
│   ├── requiredTech: TechNode (nullable)
│   ├── requiredState: GameState (전투직후/식량부족/등)
│   └── locationTag: string (원정 전용)
├── illustration: Sprite
├── narrativeText: string  // 상황 묘사
├── choices: List<EventChoice> [3~4개]
│   ├── choiceText: string
│   ├── costs: List<ResourceCost>
│   │   ├── type: enum (None/Resource/Hope/Discontent/Time)
│   │   └── amount: float
│   ├── rewards: List<ResourceReward>
│   │   ├── type: enum (Resource/Unit/Tech/Buff/Info)
│   │   └── amount: float
│   ├── moraleEffect: (hopeChange, discontentChange)
│   ├── duration: enum (Instant/Permanent/Timed)
│   ├── isHidden: bool  // 조건부 숨김 선택지
│   └── unlockCondition: EventCondition (nullable)
├── isOneTime: bool  // 일회성 여부
└── weight: float  // 드로우 가중치
```

### 비용-보상 밸런스 가이드라인 (Scythe 원칙 기반)
```
① 안전: 비용 0     → 보상 1.0x (기준)
② 투자: 비용 자원  → 보상 1.8~2.2x
③ 대가: 비용 사회적 → 보상 2.5~3.0x + 희망/불만 변동
④ 특수: 비용 혼합  → 보상 3.0~4.0x (조건부 해금)
```
- 모든 선택지의 **순기대가치 > 0** (순수 패널티 없음)
- ③의 사회적 비용은 장기적 기회비용 포함 → 표면적 보상률보다 실제 가치는 낮을 수 있음
- 이벤트 간 보상 총량은 게임 진행도에 비례하여 스케일링
