## 1. 핵심 컨셉

### 1.1 한줄 요약

> 낮에 내정과 탐사로 대비하고, 밤에 압도적 물량 공세를 가벼운 전술 조작으로 막아내는 턴제 기지 경영 + 실시간 타워디펜스 게임

### 1.2 엘리베이터 피치

플레이어는 허름한 기지의 지휘관이 되어, 낮(턴제) 동안 건물을 짓고 인력을 배치하며 원정대를 보내 정보를 수집한다. 밤이 되면 실시간으로 어마어마한 물량의 적이 밀려오고, 일시정지와 배속 조절을 활용해 분대를 지휘하며 낮 동안의 준비가 생존을 결정한다. 매 턴의 소소한 선택들이 쌓여 결정적 순간의 쾌감을 만들어내며, "한 턴만 더"를 멈출 수 없게 만든다. 각 기지는 독립된 이야기를 가지지만, 모든 기지의 이야기가 하나의 시대를 관통하는 서사로 연결된다.

## 2. 핵심 재미 이론

### 2.1 단기적 재미 (30초~5분)

즉각적으로 느끼는 재미 요소 -- **호기심과 선택의 긴장감**

- **선택의 고통**: 건물 슬롯에 무엇을 배치할지, 소수의 생존자 중 누구를 어떤 건물에 소켓 배치할지, 매 순간의 의사결정이 "사람을 미치게" 만드는 쾌감
- **간결한 인카운터**: Frostpunk식 도덕적 딜레마 기반의 빈번한 이진 의사결정으로 리듬감 유지 (V1의 Reigns 참조를 Frostpunk 이벤트로 대체)
- **건물 배치 피드백**: 스케일 팝 + 파티클 + 사운드로 한 클릭의 권능감 전달 (Thronefall UX 패턴 직접 차용, Cross-Reference-Matrix 섹션 14)
- **정보 발견**: 탐사/정찰 결과로 새로운 정보가 드러날 때의 호기심 충족

### 2.2 중기적 재미 (5분~1시간)

한 세션에서 느끼는 성장과 전략 검증 -- **안도감과 뿌듯함**

- **점진적 해금**: 관리 탭 -> 탐험 탭 순서로 기능이 열리며 가능성 확대. 건설 우선순위에 따라 방어력과 생산력을 확보
- **업그레이드 분기 = 전략 다양성**: 고정된 건물의 업그레이드 방향 선택이 기지 특색을 결정
- **전략 검증**: 낮의 준비가 밤의 결과로 입증될 때의 "내 선택이 맞았다!" 뿌듯함. 이 "준비->검증" 구조는 Thronefall(96% 리뷰)과 TAB(90만 판매)에서 독립적으로 검증된 재미 원천이다 (Cross-Reference-Matrix 섹션 9.1)
- **물량 방어 카타르시스**: 어마무시한 적 물량을 막아냈을 때의 안도감과 성취감

### 2.3 장기적 재미 (1시간~)

반복 플레이, 리텐션 요인 -- **호기심과 마스터리**

- **옴니버스 내러티브**: 각 기지 세션의 이야기가 연결되어 전체 시대의 서사를 형성. 연결 메커니즘은 텍스트 콜백(MVP) -> 초기 조건 변형(알파/베타) -> 공유 NPC(정식) 순으로 단계적 구현 (시스템 5.10 참조)
- **시대 전환**: 한 시대의 기지들을 모두 클리어하면 다음 시대로 -> 쇠퇴한 기지, 기술 발전 등 변화
- **최적화 탐색**: 더 매끄러운 운영을 시도하려는 동기 (복잡도 4~4.5이지만 최적화 여지 존재)
- **다른 선택 시도**: 패배 시 다른 기술 트리/정책 선택으로 재도전
- **기지별 고유 기술 분기**: 공통 트리 기반 위에 기지마다 다른 고유 분기가 다른 전략을 요구

### 2.4 재미의 우선순위 매트릭스

| 순위 | 재미 요소 | 감정 곡선 | 포기 가능 여부 | 검증 근거 |
|---|---|---|---|---|
| 1 | 선택의 긴장감 (매 턴 유의미한 의사결정) | 긴장 -> 결정 -> 확인 | 절대 불가 | Civ5 "한 턴만 더" 중독성, 15년 후 CCU 11K+ |
| 2 | 물량 방어 카타르시스 (DOTS 대규모 전투) | 불안 -> 긴장 -> 안도/쾌감 | 절대 불가 | TAB 최종 웨이브 바이럴, 90만 판매 |
| 3 | "한 턴만 더" 중독성 (다음 턴에 대한 호기심) | 완결 -> 호기심 -> 재진입 | 절대 불가 | Against the Storm 100만 판매, 12개월 잔존율 29% |
| 4 | 옴니버스 내러티브 연결 | 완결 -> 호기심 -> 연결 | 상황에 따라 축소 가능 | Frostpunk 시나리오 서사가 핵심 구매 동기 |
| 5 | 시대 전환 메타 구조 | 성장 -> 새로움 -> 기대 | 필요시 후순위 | Against the Storm 메타 진행 구조 (영감) |

## 3. 디자인 필터

### 3.1 수박 껍질 이론 (진입 장벽)

> "유저가 핵심 재미에 도달하기 전에 이탈하게 만드는 장벽은 무엇이고, 어떻게 최소화할 것인가?"

- **허용 가능한 장벽**: 건물/직책 슬롯에 시민 배치, 탐험대 구성, 방어 타워 배치 등 인력 배분의 복잡도
- **제거할 장벽**: 초반의 과도한 정보 노출, 복잡한 자원 관리, 고도의 APM을 요구하는 하드코어 RTS 조작
- **장벽 해소 전략**:
  - ~~**행정관 능력 기반 정보 접근 단계**~~ (폐기됨: 소켓 시스템이 충분히 직관적)
  - **탭 점진적 해금**: 1일차 건설만 -> 2일차 관리 탭 -> 이후 탐험 탭

**경쟁작 진입 장벽 벤치마크** (Cross-Reference-Matrix 섹션 9.2):

| 게임 | 진입 장벽 해소 전략 | 결과 | Project_Sun 적용 |
|---|---|---|---|
| Thronefall | 선택지 제거 (고정 슬롯, 단일 자원) | 96% 긍정 | 고정 슬롯 채택 |
| Against the Storm | 단기 런 구조 | 95% 긍정이나 긴 학습 곡선 | 반면교사 |
| Frostpunk | 점진적 기능 해금 없음 | 92% 긍정이나 튜토리얼 부족 지적 | 반면교사 |
| **Project_Sun** | **탭 점진적 해금** | **목표 90%+** | **점진적 해금** |

### 3.2 결핍-충족 이론

> "유저에게 무엇이 부족하다고 느끼게 하고, 그것을 어떻게 채워주는가?"

- **의도적 결핍**:
  - 인력 보너스 최적화: 소수의 생존자를 어떤 건물에 배치할지, 보너스 조합의 트레이드오프
  - 정보 부족: 적 구성을 완전히 알 수 없음 (탐사로 부분 해소)
  - 자원 제한: 기초/고급/유물 3종 자원이 항상 선택을 강제. 특히 유물은 매우 희소하여 어디에 투자할지 고민
- **충족 수단**:
  - 방어 성공 보상으로 자원/인력 획득
  - 탐사로 생존자 구출, 유물 발견, 기술 해금
  - 업그레이드 분기를 통한 기능 확장
- **충족 주기**: 매 턴(낮->밤->보상) 단위의 빠른 피드백 루프

### 3.4 레퍼런스 상충 해결 필터

Cross-Reference-Matrix 섹션 12에서 도출된 3가지 상충과 해결 원칙. 이후 모든 시스템 설계에서 이 원칙을 위반하는지 점검해야 한다.

| 상충 | 해결 원칙 | 위반 징후 |
|---|---|---|

**핵심 선언**: Project_Sun은 복잡도가 균일하지 않은 게임이다. 건설은 Thronefall 수준(복잡도 최소화), 인력 관리는 디아블로 보석 소켓식 직관적 깊이 (소수 인원의 최적 배치)(능력치 2~3종, 보너스 유형 5~8종, 숙련도 성장, 핵심 시민 5~8명만 개별 관리). 에이전시는 낮(턴제)의 의사결정과 밤(실시간, 일시정지 가능)의 가벼운 전술 조작(분대 3~5개, 이동/공격/정지 3종 명령, SC2 참고)에서 확보한다. 전체 복잡도는 4~4.5/10 (섹션 5.3.2 양립 논증 참조).

## 4. 핵심 플레이 루프

### 4.1 매크로 루프 (전체 게임)

```
[시대 1]
  |-- 기지 A (세션 1) -- 클리어
  |-- 기지 B (세션 2) -- 클리어   <-- 옴니버스: 각 기지 독립 서사, 전체 연결
  +-- 기지 C (세션 3) -- 클리어
        |
[시대 2] -- 기술 발전/쇠퇴 반영
  |-- 기지 D ...
  +-- ...
```

- 기지 세션 길이: 1~2시간 (Against the Storm 30~120분, The Last Spell 3~8시간의 중간 지점)
- 시대당 기지 수: 3~5개 (MVP에서는 1시대 1~2기지로 축소 검증)
- 시대 수: 3~5개 (정식 출시 기준)

### 4.2 마이크로 루프 (하루 = 1턴)

```
[낮 페이즈 -- 턴제, 무제한 시간]          [밤 페이즈 -- 실시간, 일시정지/배속 가능]
+-----------------------------+      +-------------------------------+
|  건설 탭: 슬롯 건설/업그레이드    |      |  타워디펜스 + 가벼운 RTS 마이크로  |
|  관리 탭: 인력 소켓 배치, 연구, 정책 |  ->  |  DOTS 웨이브 공세               |
|  탐험 탭: 원정대 편성/파견       |      |  분대(3~5개) 이동/공격/정지 지시   |
|  인카운터: 이진/3선택지 이벤트    |      |  속도: 1x/2x + 일시정지          |
+-----------------------------+      +-------------------------------+
                                              |
                                     [방어 결과 판정]
                                       /            \
                              [방어 성공]          [방어 실패]
                              전투 보상 획득       건물 손상 + 시민 부상
                              [다음 낮으로 ->]     [다음 낮으로 ->]
```

**밤 전투 조작 수준**: 분대 단위 가벼운 RTS 마이크로. 3~5개 분대를 이동/공격/정지 3종 명령(SC2 참고)으로 지시한다. 타워디펜스가 기본 뼈대(전투력 70~80%)이며, 유닛 분대(전투력 20~30%)를 방어 위치에 배치하거나 위협 지점으로 이동시킨다. 마이크로 없이도 전투는 자동 진행되며, 분대 조작은 승률을 높이는 보조 수단이다. 일시정지 중에도 명령 입력이 가능하므로 시간 압박이 없다. Thronefall처럼 주인공 캐릭터를 직접 조종하는 것이 아니며, 고도의 조작 숙련도를 요구하지 않는다. 상세 프레임워크는 섹션 5.3.1 참조.

### 4.2.1 패배 조건 (건물 손상 기반 점진적 패배 모델)

Project_Sun은 **건물 손상 기반 점진적 패배** 방식을 채택한다. They Are Billions식 즉사(한 마리 침입 = 게임오버)는 "하지 않을 것"에 가깝다. **패배 조건: 본부 파괴 = 게임오버.** 밤 전투에서 적이 본부까지 도달하여 본부 HP를 0으로 만들면 게임오버. 패배는 누적된 실패의 결과이며, 플레이어에게 회복을 시도할 기회를 준다.

| 단계 | 조건 | 결과 | 감정 설계 |
|---|---|---|---|
| **단기 피해 (턴 단위)** | 밤 방어에서 방어선 침투 발생 | 침투 지점의 건물 손상 + 해당 구역 시민 부상. 다음 턴 해당 구역 비활성화 | 긴장감 -- "이번 밤을 버텨야 한다" |
| **중기 위기 (복수 턴 누적)** | 연속 방어 실패로 건물 손상 누적 | 운영 가능 슬롯 감소로 생산/방어력 약화. 회복 난이도 상승 | 위기감 -- "지금 돌이키지 않으면 끝난다" |
| **최종 패배 (게임오버)** | 본부 HP 0 도달 | 본부 파괴 = 기지 세션 종료 (게임오버) | 아쉬움 + 재도전 욕구 -- "다음엔 다르게 해보겠다" |

점진적 패배 모델은 유지: 외곽 건물부터 손상 -> 생산/방어력 약화 -> 본부 위협 증가. 단, 시민 이탈/반란은 없음.

**설계 근거**:
- TAB의 즉사는 마이크로 컨트롤 부재 상태에서 지나치게 가혹하다.
- 점진적 패배는 "회복 가능성"이라는 여지를 주면서도, 연속 실패 시 불가역적으로 상태가 악화되어 긴장감을 유지한다.
- 밤 방어 실패 시에도 게임이 즉시 끝나지 않으므로, 전투 결과에 대한 수용성이 높아진다. 가벼운 RTS 마이크로로 플레이어에게 전투 개입 기회를 주되, 점진적 패배 구조로 한 번의 실수가 치명적이지 않게 설계한다.

**방어 결과 스펙트럼**:

```
[완벽 방어]          [경미 피해]          [대규모 피해]         [본부 파괴]
피해 0              건물 1~2 손상        건물 다수 손상         본부 HP 0 -> 게임오버
보너스 보상          기본 보상            보상 감소 + 시민 부상
```

### 4.3 루프 간 연결

- **탐사 -> 내정**: 원정 결과로 기술/정책 해금 방향 결정, 적 정보로 방어 배치 계획
- **내정 -> 방어**: 건물 배치/업그레이드/인력 소켓 배치가 밤 전투력 직결
- **방어 -> 다음 턴**: 전투 보상(기초/고급 자원, 유물)이 다음 턴의 내정/탐사 자원. 생존자 합류로 인력 보너스 옵션 확대
- **방벽 건설**: 자원 투자 -> 방어력 확보 -> 후반 웨이브 대응

**P0-core 검증 핵심** (Cross-Reference-Matrix 섹션 11): 위 세 연결 중 "내정->방어" 인과관계가 가장 중요하다. 낮의 인력 배분과 건설 결정이 밤의 전투 결과에 직결되지 않으면 모든 요소가 독립적으로 재미있어도 전체 루프가 성립하지 않는다. 밤의 가벼운 RTS 마이크로는 낮의 준비를 "미세 조정"하는 보조 수단이지, 준비 부족을 만회하는 수단이 아니다.

**프로토타입 2기둥 검증 선언**: 탐사 시스템은 P0-full로 분류되어 프로토타입 단계에서는 고정 이벤트 3개 스텁으로 대체된다. 따라서 **프로토타입은 3기둥이 아닌 2기둥(내정+방어)으로 핵심 루프를 검증**한다. 이것이 가능한 이유:
- 내정->방어 인과관계가 코어 루프의 본질이다. 탐사는 이 인과관계를 풍성하게 하지만, 없어도 "건설/인력 배치 -> 밤 방어 결과"라는 핵심 루프 자체는 성립한다.
- Thronefall이 탐사 없이 건설+방어 2기둥만으로 96% 긍정 리뷰를 달성한 것이 2기둥 루프의 독립적 재미를 증명한다.
- 탐사 시스템은 P0-full(알파 단계)에서 통합하며, 이때 3기둥 루프의 시너지를 별도로 검증한다.

### 5.2 인력 관리 시스템

| 항목 | 내용 |
|---|---|
| **Why** | "내 선택이 유의미하다"는 느낌의 핵심. 소수의 생존자를 어떤 건물에 소켓 배치하여 최적의 보너스 조합을 만드느냐가 핵심 전략 |
| **Objective** | 건물 소켓에 시민을 배치하여 보너스 부여 (스탯 증가, 특수 능력 해금). 건물은 건설 완료 시 자동 활성화되어 기본 기능이 작동하며, 인력 소켓 배치는 선택적 보너스를 추가한다. 각 시민의 고유 속성/스킬이 건물에 따라 다른 보너스를 제공. 배치된 건물에서 경험이 축적되어 해당 건물 보너스가 강화된다 |
| **Volume** | 시민 **5~15명/기지** (탐사로 생존자 구출하여 증원), **능력치 2~3종**, **보너스 유형 5~8종** |
| **Detail** | **디아블로 보석 소켓 모델** 변형. 건물은 건설 시 자동 활성화되어 기본 생산/기능이 보장된다. 시민을 건물의 인력 소켓에 배치하면 보너스가 부여된다 -- 스탯 증가(생산량 +20% 등) 또는 특수 능력 해금(자동 수리, 야간 경계 등). 핵심 시민 전원 개별 관리(이름, 고유 능력, 배치 보너스) -- 5~15명이므로 FTL 스케일과 유사하게 전원 관리 가능. 시민은 배치된 건물에서 경험을 축적하여 해당 건물의 보너스가 강화된다 (FTL식 "쓸수록 는다"). 시민 이동 = 이전 건물 보너스 상실 + 새 건물 보너스 획득 (기회비용). ~~행정관 정보 단계~~ (폐기됨: 소켓 시스템이 충분히 직관적). 복잡도는 디아블로 보석 소켓 수준의 직관적 깊이를 목표 (상충 해결 원칙 3.4 준수) |
| **레퍼런스** | **FTL: Faster Than Light**(숙련도 성장 모델) -- **직접 차용**, **Diablo**(보석 소켓 시스템) -- **직접 차용** |
| **리스크** | 소켓 보너스 조합의 밸런스 -- 프로토타입 검증 필요 |
| **우선순위** | P0-core |
| **검증 가설** | "인력 소켓 배치 의사결정이 핵심 재미가 되는가?" |

**디아블로 보석 소켓 모델 적용 근거** (v2.4 자원/인력 모델 개편):
- 건물 자체는 자동 가동된다. 인력은 필수 운영 요소가 아닌 선택적 보너스 제공자다.
- 소수 인원(5~15명)이므로 전원 개별 관리가 FTL 스케일과 유사하게 가능하다.
- "이 사람을 여기에 둘까 저기에 옮길까"의 의사결정은 유지되며, 보석 소켓처럼 조합 최적화가 핵심 퍼즐이 된다.
- 숙련도 성장은 시민에 대한 애착을 만들어내고, 손실 시 감정적 타격을 강화한다 (결핍-충족 이론 3.2 연동). 건물에서 오래 일할수록 해당 건물의 보너스가 강화되므로, 이동의 기회비용이 자연스럽게 생긴다.
- Civ5급 복잡도(능력치 3~5종, 직책 10~15종)는 인력 관리 자체가 별도 게임이 되는 수준이며 Project_Sun의 복잡도 4~4.5/10 목표에 과도하다.

#### 5.3.1 가벼운 RTS 마이크로 조작 프레임워크 (v2.3 신설)

비전 수준에서 "가벼운 RTS 마이크로"의 최소 스펙을 정의한다. 상세 설계는 시스템 GDD에서 확정하되, 이 프레임워크가 후속 설계자의 해석 기준이 된다.

| 항목 | 스펙 | 근거 |
|---|---|---|
| **유닛 그룹** | 3~5개 분대 (개별 유닛이 아닌 분대 단위) | 개별 유닛 선택은 SC2급 복잡도. 분대 단위는 Thronefall(복잡도 2)에 가까운 인지 부하. 3~5개는 한 화면에서 한눈에 파악 가능한 수 |
| **명령 유형** | 3종 (SC2 참고): (1) 이동 -- 분대를 지정 위치로 이동, (2) 공격 -- 지정 대상/지점을 공격, (3) 정지 -- 현재 위치에서 대기/사격 | 명령 3종은 SC2의 기본 조작 3종과 동등. 일시정지 가능하므로 APM 부담 없음 |
| **밤당 명령 횟수** | 제한 없음. 단, 배치만 잘하면 최소 개입으로도 클리어 가능 | 타워가 자동 공격하므로 유닛 분대는 보조 전력. 잘 배치하면 거의 안 건드려도 되지만, 위기 시 재배치로 대응 |
| **마이크로 없이 전투 진행 여부** | 가능. 타워디펜스가 기본 뼈대이므로 유닛 분대에 아무 명령을 내리지 않아도 전투는 자동 진행됨 | 핵심 원칙: 마이크로는 "승률을 높이는 보조 수단"이지 "필수 조건"이 아님. 낮의 준비(건설/인력 배치)가 전투 결과의 70~80%를 결정 |
| **속도 조절** | 1x / 2x + 일시정지 가능 | 일시정지 중에도 명령 입력 가능. Frostpunk식 "정지 후 판단" 패턴. 이것이 "가벼운"의 핵심 보장 -- 시간 압박이 없음 |
| **전투력 비율 (목표)** | 타워/건물 자동공격 70~80% + 유닛 분대 20~30% | 타워가 주력이므로 분대 마이크로가 전투 결과를 뒤집지 않음. 분대는 취약 지점 보강, 피해 최소화 등 "미세 조정" 역할 |

**핵심 설계 원칙**: "타워가 자동 공격하고, 유닛 분대는 플레이어가 방어 위치에 배치/이동. 잘 배치하면 거의 안 건드려도 되지만, 위기 시 재배치로 대응한다."

**밤 페이즈 시간 모델 선언**: 낮 = 턴제(무제한 시간), 밤 = 실시간(일시정지/배속 가능). "완전 턴제"라는 기존 표현은 낮 페이즈에만 해당하며, 밤 전투는 실시간으로 진행되되 일시정지와 배속 조절이 가능하여 시간 압박은 플레이어가 통제한다.

**V1 방향 전환 인정**: V1 Vision.md 섹션 7.2에서 "자동 전투 (플레이어 전투 간섭 보류)"를 확정 방향으로 명시했다. v2.2에서 디렉터 판단으로 이를 "타워디펜스 + 가벼운 RTS 마이크로"로 전환했다. 이 전환의 핵심 근거는 부록 B의 에이전시 이중 확보 논증(5가지)이며, 전환이 실패할 경우 폴백 3단계(7.3.1)에서 V1 방향(완전 자동)으로 복귀한다.

#### 5.3.2 복잡도 4~4.5/10 + RTS 마이크로 양립 논증 (v2.3 신설)

밤 전투에 RTS 마이크로를 추가하면 "동시 관리 시스템"이 증가하므로 복잡도 점수에 영향을 미칠 수 있다. 이 논증은 왜 복잡도가 4~4.5 범위에 머무르는지를 명시한다.

**핵심 논증**: 분대 단위 이동/공격/정지 3종 명령 + 일시정지 = 복잡도 추가 0.5~1.0

| 논거 | 설명 |
|---|---|
| **"동시"가 아닌 "교대"** | 건설/인력/탐사는 낮 페이즈에서만, 전투 마이크로는 밤 페이즈에서만 수행한다. 4개 시스템이 동시에 관리되는 것이 아니라 2개(낮) + 1개(밤)로 교대한다. Cross-Reference-Matrix 섹션 10.1의 복잡도 축 기준 "동시 관리 시스템 2~3개 = 3~4점"에 해당 |
| **Thronefall 선례** | Thronefall(복잡도 2)에서 밤 페이즈의 군주 직접 전투 조작은 복잡도를 거의 추가하지 않았다. 이유: (1) 명령이 단순(이동만), (2) 실시간이지만 적 물량이 소규모. Project_Sun의 분대 3종 명령(이동/공격/정지)은 SC2 기본 조작과 동등하나 일시정지 가능으로 APM 부담 없음 |
| **Rising Demacia 비교** | Rising Demacia(복잡도 1~2)는 완전 자동 전투. Project_Sun은 마이크로가 있지만 분대 단위로 축소되고 일시정지가 가능하므로, 실질 조작 부하는 "경영 게임에 미니게임이 추가된" 수준 |
| **일시정지가 복잡도를 상쇄** | 밤 전투에서 일시정지 가능하므로 시간 압박이 없다. 실시간이지만 실질적 인지 부하는 반실시간(Frostpunk 3점)보다 낮다. 일시정지 중 명령 입력이 가능하므로 APM이 아닌 판단력이 핵심 |

**결론**: 복잡도는 기존 4에서 4~4.5 수준으로 소폭 상향한다. 목표 범위(3~5) 내이며, "복잡한 4X에 피로감을 느낀 유저"가 코어 타겟이라는 전제를 훼손하지 않는다.

| **Detail** | DAG 기반 노드 그래프. **공통 트리 70% + 기지 고유 분기 30%** 구조로 콘텐츠 생산 비용을 절감하면서 기지별 전략 차별화를 유지. 공통 트리는 건설/관리/방어의 기본 업그레이드를 포함하며, 고유 분기는 기지의 환경/서사에 연결된 특수 기술(예: 사막 기지의 태양열 기술, 산악 기지의 채굴 기술). 기본 노드: 기초+고급 자원으로 연구. 강력한 노드: 기초+고급+유물 자원이 필요하여 전략적 투자 선택을 강제 |
| **레퍼런스** | Civilization V(DAG 기술 트리) -- 변형, Frostpunk(3탭 기술 트리) -- 변형 |
| **리스크** | 고유 분기 노드의 밸런스 검증 필요. 공통 트리 비중이 높으면 기지 간 차이가 희미해질 수 있음 |
| **우선순위** | P1 |
| **콘텐츠 볼륨 현실화** | 정식 출시(3시대 x 3기지 = 9기지) 기준: 공통 15~20 + 고유 45~72 = 총 60~92 노드. 기존 추정(135~625)에서 대폭 축소. MVP: 1시대 2~3기지로 검증 |

### 5.8 패배 조건 (v2.5 변경: 희망/불만 시스템 폐기)

희망/불만 게이지 시스템은 과도한 복잡도로 인해 폐기되었다.
**패배 조건: 본부 파괴 = 게임오버.** 밤 전투에서 적이 본부까지 도달하여 본부 HP를 0으로 만들면 게임오버.

본부는 게임 시작 시 맵 중앙에 존재하며, 건설/이동 불가. --> 건설 시스템 GDD (Construction.md) 섹션 3.1 참조.

점진적 패배 모델은 유지: 외곽 건물부터 손상 -> 생산/방어력 약화 -> 본부 위협 증가. 단, 시민 이탈/반란은 없음.

### 5.9 ~~방벽 확장 시스템~~ (폐기 -- 건설 시스템에 통합)

> **설계 변경 (2026-04-02)**: 방벽 확장을 별도 시스템으로 분리하는 개념이 폐기되었다. 방벽은 건설 시스템(5.1) 내 일반 방어 건물의 하나이며, 다른 건물과 동일하게 고정 슬롯에 지정되어 있다. 방벽 건설/업그레이드는 방어력(HP/범위)을 높이며, 다른 건물의 건설 가능 여부에 영향을 주지 않는다. 방벽 건설 비용 자체는 자원 Sink로 유효하며, 세부 스펙은 Economy-Model.md 섹션 5.2 방어 건물 참조.

| 후보 | 설명 | 구현 비용 | 플레이어 체감 |
|---|---|---|---|
| **A. 텍스트 콜백** | 이전 기지에서의 주요 선택/결과가 다음 기지의 이벤트 텍스트에 언급됨 (예: "서쪽 기지가 함락되었다는 소식이 전해졌다") | 낮음 | 서사적 연결감, 직접적 게임플레이 영향 없음 |
| **B. 초기 조건 변형** | 이전 기지의 원정 결과/생존 시민이 다음 기지의 초기 자원/인력/기술 해금에 영향 (예: 이전 기지에서 파견한 원정대가 다음 기지의 초기 시민에 포함) | 중간 | 게임플레이와 서사의 직접 연결, 리플레이 동기 강화 |
| **C. 공유 NPC/아이템** | 특정 NPC나 유물이 기지를 넘어 등장. 이전 기지에서의 선택이 해당 NPC/아이템의 상태에 반영 | 중간~높음 | 캐릭터 애착, 시대 관통 서사 |

### 5.11 자원 체계

자원 3종(기초/고급/유물)과 인력(보석 소켓) 시스템을 정의한다. 비전 수준에서 각 자원의 역할과 3기둥 연결을 명시한다.

| 자원 (후보명) | 역할 | 3기둥 연결 | 레퍼런스 |
|---|---|---|---|
| **기초 자원** (공용 기본) | 건설/업그레이드/연구의 기본 재화. 가장 풍부한 자원 | 내정(건설) + 방어(수리/강화) + 탐사(보급) | StarCraft 2 미네랄 |
| **고급 자원** (공용 고급) | 기초 자원과 함께 사용. 고급 건설/연구/업그레이드에 필수 | 내정(고급 건설) + 방어(고급 시설) + 탐사(장거리 보급) | StarCraft 2 가스 |
| **유물 자원** (해금 재화) | 강력한 연구 노드, 엘리트 업그레이드, 특수 기능 해금. 매우 희소 | 탐사(주 획득 경로) + 방어(보상) | 디아블로 희귀 재화 |

**인력** (자원이 아닌 "보너스 시스템"으로 분리):

| 인력 | 역할 |
|---|---|
| **시민** (보석 소켓) | 건물에 배치 시 보너스 부여. 5~15명/기지, 각자 고유 속성. 건물은 인력 없이도 가동 |

**설계 의도**:
- 기초/고급의 이중 자원이 "무엇을 먼저 짓고 업그레이드할 것인가"의 우선순위 결정을 강제한다.
- 유물은 "가장 강력한 기능 중 어디에 투자할 것인가"의 전략적 선택을 만든다.
- 인력은 자원이 아닌 보너스 제공자다. "누구를 어디에 배치할까"가 핵심 퍼즐이며, 자원 소비/생산과 독립적으로 의사결정 가치를 가진다.
- StarCraft의 미네랄/가스 구조보다 단순하게 유지하되, 유물로 전략적 깊이를 추가한다.
- Thronefall의 단일 자원(골드)보다는 복잡하지만, Frostpunk의 4종이나 TAB의 6종보다는 간결하다.

**미확정 사항** (시스템 GDD에서 결정):
- 자원명은 세계관에 맞춰 변경 가능 (예: 기초 자원 -> 광석, 고급 자원 -> 결정, 유물 -> 고대 유물)
- 유물 자원의 구체적 획득 경로 및 드롭 비율
- 자원 획득/소비의 구체적 수치

### B. 차별화 논증 요약

Cross-Reference-Matrix 섹션 13의 핵심 논거를 요약한다.

**왜 이 조합이 기존에 없었는가**:
- 기술적 한계: 턴제 기반에서 밤 실시간 전투로 수천 엔티티 시뮬레이션은 DOTS/ECS가 인디에 접근 가능해진 2022년 이후에야 실현 가능
- 설계 난이도: 가벼운 RTS 마이크로의 난이도 밸런스(너무 쉬우면 무의미, 너무 어려우면 스트레스)가 핵심 과제. Thronefall(2024)이 소규모로 자동방어를 검증했지만 대규모 물량+복잡한 내정+가벼운 마이크로의 결합은 미시도

**왜 이 조합이 재미있을 것인가**:

| 요소 | 독립 검증 근거 | 조합 시 시너지 |
|---|---|---|
| 낮 턴제 + 밤 실시간 타워디펜스(일시정지/배속) + 가벼운 RTS 마이크로 | Thronefall 96% 리뷰, SC2 유닛 이동 UX | 턴제(낮) + 실시간 대규모 물량(밤, 일시정지 가능) = 낮은 APM으로도 전술적 카타르시스 + 에이전시 |
| 대규모 물량 카타르시스 | TAB 최종 웨이브 바이럴 | 물량 + 복잡한 내정 = 정교한 준비로 큰 물량을 막는 성장 곡선 |
| FTL식 직관적 인력 관리 | FTL 숙련도 시스템, Civ5 8M+ 판매 | 내정 + 밤 전술 조작 = 낮의 모든 결정이 밤에 반영 + 밤의 미세 조정으로 에이전시 강화 |

**핵심 시너지**: "낮(턴제)에 내린 결정의 결과를 밤(실시간)의 압도적 물량 방어로 시각화하고, 일시정지하며 가벼운 전술 조작으로 결과를 미세 조정한다"는 독특한 감정 루프. TAB(고APM 필수), Thronefall(물량 소규모), Civ5(물량 방어 없음) 각각에서 부분적으로만 경험 가능했던 것을 통합.

**Thronefall 군주 조작과의 차이 인정**:

Thronefall의 밤 페이즈는 "완전 자동"이 아니다. 군주가 직접 전장을 이동하며 전투에 참여하고, 이 조작이 밤 페이즈의 에이전시에 기여한다 (Cross-Reference-Matrix 섹션 10.1: "Thronefall 실시간성 2점 -- 밤에 군주 조작"). 따라서 "Thronefall이 자동방어로 성공했다"는 논거는 정확하지 않으며, 이를 수정한다.

Project_Sun이 군주 직접 조종을 배제하고 **가벼운 RTS 마이크로**를 선택한 이유 (v2.2 방향 전환):
1. **에이전시 이중 확보**: 낮의 복잡한 내정(인력 관리, 기술 트리, 정책) + 밤의 가벼운 전술 조작(유닛 이동/배치)으로 양 페이즈에서 에이전시를 확보한다. v2.1의 "완전 자동 + 낮에서만 에이전시"보다 에이전시 원천이 넓어진다.
2. **마이크로 스트레스 vs 에이전시의 균형**: "어느 병력을 어디로 보낸다" 수준의 조작은 TAB/Frostpunk 유저 불만인 "고APM 실시간 스트레스"를 피하면서도, "내가 전투에 개입했다"는 에이전시를 준다.
3. **Thronefall 군주 조작과의 차이**: 주인공 캐릭터를 직접 움직여 전투하는 것이 아니라, 유닛 그룹 단위의 배치/이동 지시다. SC2의 유닛 이동 UX를 참고하되 고도의 숙련도는 요구하지 않는다.
4. **준비->검증+미세조정**: 밤의 전술 조작은 낮의 준비를 "미세 조정"하는 보조 수단이지, 낮의 준비 부족을 만회하는 수단이 아니다. 핵심 인과관계(낮의 준비 = 밤의 결과)는 유지된다.
5. **이 선택의 리스크를 인정한다**: 미검증 가설 1순위(7.3)로 분류되어 있으며, 조작이 너무 바쁘거나 무의미할 경우의 폴백 플랜(7.3.1)이 준비되어 있다.

**성공 가능성 근거**:
1. Thronefall이 낮밤 루프 + 자동방어(부분)를 96% 리뷰로 검증했다. 약점 "콘텐츠 부족"과 "내러티브 없음"이 Project_Sun 강점 방향과 일치한다.
2. TAB 커뮤니티에서 최종 웨이브 격퇴 영상이 가장 큰 바이럴 콘텐츠.
3. TAB/Frostpunk 유저의 공통 불만 "실시간 마이크로 스트레스"를 가벼운 RTS 마이크로(고APM 불필요)로 해소하면서도 전술적 에이전시를 주는 게임이 부재 (미충족 수요).
4. Rising Demacia가 완전 자동 전투의 캐주얼 수용성과 UX 패턴(사전 정보 공개, 세션 자유, 점진적 해금)을 검증했다. Project_Sun은 캐주얼 UX 패턴을 직접 차용하되, 전투에 분대 단위 가벼운 전술 조작(일시정지 가능)을 추가하여 코어 타겟(페르소나 A)의 에이전시 요구도 동시에 충족한다.

### C. 용어 사전

| 용어 | 정의 |
|---|---|
| 타임/페이즈 | 하루를 구성하는 단위. 낮 페이즈(내정+탐사)와 밤 페이즈(방어)로 구성 |
| 기지 | 하나의 플레이 세션 단위. 각 기지는 독립된 맵/기술 트리/내러티브를 가짐 |
| 시대 | 여러 기지를 포괄하는 메타 단위. 시대별 기술 수준/환경이 변화 |
| 슬롯 | 건물을 배치할 수 있는 고정된 위치 |
| 방벽 | 건설 시스템(5.1) 내 일반 방어 건물. 기지의 방어선 역할. 방벽 확장 시스템은 폐기됨(5.9 참조) |
| 인카운터 | 게임 진행 중 발생하는 이벤트. 일상(이진)과 중요(3선택지)로 구분 |
| ~~행정관~~ | *(폐기됨: 소켓 시스템이 충분히 직관적)* 이전 정의: 본부에 배치된 시민. 능력에 따라 열람 가능한 정보의 수준이 결정됨 |
| 노드 그래프 | 원정 맵의 구조. 지점(노드)과 경로로 구성된 탐험 맵 |
| P0-core | 프로토타입 단계에서 반드시 구현해야 하는 핵심 요소. 핵심 루프 성립 조건 |
| P0-full | EA 출시 전 완성 필요하지만 프로토타입에서는 스텁 대체 가능한 요소 |
| SOM | Serviceable Obtainable Market. 실제 점유 가능한 시장 규모 |
| 가벼운 RTS 마이크로 | 밤 전투(실시간, 일시정지/배속 가능)에서 3~5개 분대를 이동/공격/정지 3종 명령(SC2 참고)으로 지시하는 수준의 조작. 타워 자동공격(70~80%)이 주력이며 분대(20~30%)는 보조. SC2식 고APM 조작이 아닌, 일시정지 후 판단하는 전술 지시. 마이크로 없이도 전투 자동 진행 가능 |
| 숙련도 | FTL식 시민 성장 메커니즘. 시민이 배치된 직책에서 경험을 축적하면 해당 분야 효율이 증가 |
