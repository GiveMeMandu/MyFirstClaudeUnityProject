## 1. 화면 개요

탐사 노드 도달 시 발생하는 중요 인카운터의 선택지 팝업. Scythe의 3선택지 구조를 직접 차용한다. 블로킹 모달이므로 선택 전까지 다른 조작이 불가하다.

**화면 유형**: 모달 팝업 (블로킹)
**주요 행동**: 3개 선택지 중 하나 선택

### 발생 시점 및 흐름 (M3 명확화)

인카운터는 두 경로 중 하나로 발생하며, 경로에 따라 UX 흐름이 다르다.

**경로 A: 귀환 보고 내 포함 (인카운터가 귀환 직전 노드에서 발생한 경우)**

```
낮 페이즈 진입
    → 귀환 보고 팝업(PNL-05) 자동 표시
        → 보고 내용 확인 중 "인카운터 결과" 섹션 표시
        → [인카운터 처리] 버튼 클릭 시 인카운터 선택지 팝업(PNL-07) 열림
        → 선택지 선택 후 PNL-07 닫힘
        → 귀환 보고 팝업으로 복귀 (결과 반영된 상태)
        → [확인]으로 귀환 보고 종료
```

이 경우 모달 우선순위 2번(PNL-05) 안에서 1번(PNL-07)이 중첩 발생한다. PNL-07이 PNL-05 위에 레이어로 열리며, PNL-07 닫힘 후 PNL-05가 다시 활성화된다.

**경로 B: 귀환 턴 이전 별도 팝업 (원정대가 중간 경유지에서 인카운터 발생)**

```
낮 페이즈 진입 (또는 진입 직후)
    → 인카운터 선택지 팝업(PNL-07) 우선 표시 (모달 우선순위 1)
    → 선택지 선택 후 PNL-07 닫힘
    → 귀환 보고가 있을 경우 PNL-05 이어서 표시
    → 낮 페이즈 정상 진행
```

이 경우 인카운터가 귀환 보고보다 먼저 처리된다 (우선순위 1번 규칙 적용).

**Screen-Flow.md 섹션 6 모달 우선순위 규칙과의 관계**:
- 경로 A: PNL-05가 먼저 열리지만, PNL-07은 PNL-05 안에서 중첩 레이어로 발생한다. 우선순위 1번 규칙은 "동시 발생 시" 기준이며, PNL-05가 먼저 열린 후 그 안에서 PNL-07이 트리거되는 경우는 예외적 중첩으로 처리한다.
- 경로 B: 우선순위 1번 규칙 그대로 적용. PNL-07이 PNL-05보다 먼저 표시된다.

## 3. 선택지 구조 (Scythe 3선택지)

| 선택지 | 성격 | 전형적 보상/비용 | 리스크 |
|---|---|---|---|
| 1번: 안전 | 소소한 보상, 리스크 없음 | 기초 자원 소량 | 없음 |
| 2번: 투자 | 자원 소비 후 더 큰 보상 | 자원 소비 → 생존자 또는 더 큰 보상 | 낮음 (자원 비용) |
| 3번: 대가 | 큰 보상이지만 부상 위험 | 유물 또는 기술 → 부상 확률 증가 | 높음 |

## 4. 정보 요소

| 요소 | 위치 | 표시 내용 | 우선순위 |
|---|---|---|---|
| 인카운터 제목 | 상단 | 사건 이름 | 최고 |
| 상황 설명 | 제목 아래 | 2~3문장 플레이버 텍스트 | 높음 |
| 선택지 카드 (x3) | 중앙 3분할 | 번호 / 행동 설명 / 보상/비용 / 리스크 | 최고 |
| 보상/비용 강조 | 카드 하단 | 자원 아이콘 + 수치 | 높음 |
| 리스크 경고 | 카드 하단 | 부상 확률 등 부정적 결과 | 높음 |
| 참여 원정대 | 하단 | 시민 이름 + 숙련도 (관련성 있을 때) | 중간 |

### 4.1 조건부 4번 선택지 (P1)

특수 조건 달성 시 (예: 탐사 Lv3 시민 포함) 4번 선택지 칸이 잠금 해제.

```
|  +---------------------------------+|
|  |  4. (조건 달성)                  ||
|  |  "고수의 판단으로 안전하게..."   ||
|  |  [이 선택]                       ||
|  +---------------------------------+|
```

미달성 시 4번 칸은 회색 + 잠금 아이콘 + 달성 조건 툴팁.

## 5. 인터랙션

| 입력 | 동작 | 피드백 |
|---|---|---|
| 선택지 카드 마우스오버 | 보상/리스크 상세 툴팁 | 카드 가장자리 강조 |
| [이 선택] 버튼 클릭 | 선택 확정 | 선택 효과 텍스트 표시 후 팝업 닫힘 |
| Escape | 닫기 불가 (블로킹 모달) | "선택 후 계속" 안내 툴팁 |

**블로킹 이유**: 인카운터는 탐사의 핵심 의사결정 순간이다. 실수로 창을 닫아서 선택을 건너뛰는 상황을 막는다. 선택지는 항상 세 가지이며, 아무것도 고르지 않는 옵션은 존재하지 않는다.

## 6. 선택 후 피드백

선택 즉시 결과 텍스트를 카드 위에 오버레이로 표시 (0.5초). 이후 팝업이 닫히고 귀환 보고 팝업(PNL-05) 또는 낮 페이즈로 복귀.

```
|  +-----------------+|
|  |  2. 투자        ||
|  |                 ||
|  |  [선택됨!]      ||    <-- 0.5초 오버레이
|  |  생존자 합류    ||
|  |  기초 자원 -10  ||
|  |                 ||
|  +-----------------+|
```

## 7. 인카운터 결과의 시민 영향 표시

일부 인카운터는 참여 시민의 숙련도나 패시브에 영향을 준다. 영향이 있을 경우 선택지 카드에 미리 표시.

```
|  +-----------------+|
|  |  3. 대가        ||
|  |  ...            ||
|  |  유물 +1        ||
|  |  부상 확률 +30% ||
|  |  [시민B: 탐사   ||
|  |   숙련도 +보너스]||  <-- 시민 개인 영향
|  | [이 선택]       ||
|  +-----------------+|
```

## 8. 설계 의도

3선택지 구조는 "무조건 좋은 선택"이 없도록 설계된다. 각 선택에 트레이드오프가 존재해야 하며, 세 선택지 중 어느 하나가 80% 이상 선택된다면 보상/비용 밸런스를 재조정해야 한다 (Exploration.md 섹션 9.1의 밸런스 지표).

선택지 순서는 고정이다. 항상 안전 → 투자 → 대가 순으로 배치하여, 플레이어가 패턴을 학습하고 더 빠르게 의사결정을 내릴 수 있게 한다.
