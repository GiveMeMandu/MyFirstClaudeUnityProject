## 1. 화면 개요

본부(HQ)가 파괴되었을 때 표시되는 게임오버 화면. 이번 런의 결과를 통계로 정리하고, 플레이어에게 재도전 또는 메인 메뉴 복귀를 선택하게 한다.

Vision.md 4.2.1의 패배 감정 설계 목표: **"아쉬움 + 재도전 욕구"**. 화면은 실패를 단죄하는 분위기가 아니라, "여기까지 잘 버텼다"는 결과 정리와 "다음엔 이렇게 하면 되겠다"는 전략적 힌트를 전달하는 방향으로 설계한다.

**화면 유형**: 전용 풀스크린
**주요 행동**: 결과 통계 확인 + 재시도 / 메인 메뉴 복귀
**이전 화면**: 밤 페이즈 HUD (HQ HP 0 도달 시 "본부 파괴" 연출 후 자동 전환)
**다음 화면**: 낮 페이즈 허브 (재시도) 또는 메인 메뉴

```

|                                                                  |
+------------------------------------------------------------------+
|  생존 기록                                                        |
|  버텨낸 밤: 12야   최장 완벽 방어 연속: 3회                        |
|  총 격파한 적: 1,847   총 건설한 건물: 9                           |
|                                                                  |
+------------------------------------------------------------------+
|  마지막 웨이브 (13야)                                              |
|  피해 비율: 78%   손상 건물: 7개   부상 시민: 2명                  |
|  약점: 북쪽 방어선이 반복적으로 돌파됨                              |
+------------------------------------------------------------------+
|  도달한 진척                                                      |
|  해금된 탭: 건설 / 관리 / 탐험                                     |
|  탐사 완료 노드: 5개   획득 유물: 3개                              |
+------------------------------------------------------------------+
|                                                                  |
|             [다시 시도]              [메인 메뉴로]                 |
|                                                                  |
+==================================================================+
```

## 3. 정보 요소

| 요소 | 표시 내용 | 우선순위 | 설명 |
|---|---|---|---|
| 게임오버 배너 | "본부가 파괴되었습니다" | 최고 | 항상. 0.5초 지연 후 등장 |
| 생존 기록 | 버텨낸 밤 수 / 완벽 방어 연속 횟수 / 총 격파 적 수 / 총 건설 건물 수 | 최고 | 항상 |
| 마지막 웨이브 요약 | 패배한 밤의 피해 비율 + 손상 건물 + 부상 시민 + 돌파 방향 약점 힌트 | 높음 | 항상 |
| 도달한 진척 | 해금된 탭 / 탐사 완료 노드 수 / 획득 유물 수 | 높음 | 항상 |
| [다시 시도] 버튼 | 동일 기지 설정으로 1일차부터 재시작 | 최고 | 항상 |
| [메인 메뉴로] 버튼 | 메인 메뉴(SCR-01)로 복귀 | 높음 | 항상 |

### 3.1 약점 힌트

"약점" 항목은 패배의 패턴을 분석하여 짧은 텍스트로 표시한다. 최대 1줄, 단정적이지 않은 관찰형 문장.

| 조건 | 힌트 텍스트 |
|---|---|
| 동일 방향에서 2회 이상 피해 | "북쪽 방어선이 반복적으로 돌파됨" |
| HQ가 직접 피해를 받음 | "본부까지 적이 도달함 — 방어선 재점검 필요" |
| 분대 전원 탈락 | "분대가 모두 탈락한 상태에서 피해 집중" |
| 건물 수리 없이 연속 손상 | "수리되지 않은 건물에서 연속 손상 발생" |

## 4. 인터랙션

| 입력 | 동작 | 피드백 |
|---|---|---|
| [다시 시도] 버튼 | 동일 기지 설정으로 1일차 낮 페이즈 시작 | 화면 전환 페이드 |
| [메인 메뉴로] 버튼 | 메인 메뉴(SCR-01)로 전환 | 화면 전환 페이드 |
| Enter | [다시 시도]와 동일 | 화면 전환 |
| Escape | [메인 메뉴로]와 동일 | 화면 전환 |

**자동 전환 없음**: 플레이어가 버튼을 누르기 전까지 이 화면을 유지한다. 결과를 읽을 시간을 보장한다.

## 5. 시각 연출 방향

| 단계 | 연출 |
|---|---|
| HQ HP 0 도달 (밤 HUD) | "본부 파괴" 풀스크린 적색 플래시 + 폭발음 (Night-Phase-HUD.md 섹션 7) |
| 게임오버 화면 진입 | 어두운 배경 + 잔해/먼지 파티클 페이드인 |
| 배너 등장 | 0.5초 지연 후 적색 텍스트 페이드인 |
| 통계 표시 | 배너 등장 후 0.5초 뒤 순차적 항목 등장 (카운트업 애니메이션) |
| 버튼 등장 | 통계 표시 완료 후 0.5초 뒤 페이드인 |

**감정 설계 의도**: 적색 플래시와 폭발음은 패배의 무게를 전달하지만, 이후 화면은 침착한 결과 정리 분위기로 전환된다. "이 런에서 이만큼 했다"는 성취 기록을 먼저 보여준 후 재시도 버튼을 제시하여, 패배를 단죄가 아닌 학습의 기회로 프레이밍한다.
