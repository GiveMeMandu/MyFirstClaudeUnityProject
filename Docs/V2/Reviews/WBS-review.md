## 잔존 또는 신규 이슈

### Major (R1 잔존 -- 강력 권장이나 승인 차단 아님)

1. **[M-01 잔존] M4(P2 Beta) SF 부족: 12~16주 마일스톤에 SF 4개** -- M4 DoD(147~153행)의 "한국어/영어/일본어 로컬라이징 완성", "Steam EA 빌드 제출" 등을 커버하는 SF가 없다. 로컬라이제이션 파이프라인, Steam 통합, 다기지 레벨 데이터 SF가 M3/M4 착수 전까지 추가되어야 한다. 다만 M4는 36~56주 후이므로 현재 시점에서 Critical은 아님.

2. **[M-02 잔존] 아트 에셋 파이프라인 SF 전무** -- 건물 10종+본부, 적 12종, 분대 유닛 등의 3D 에셋 조달 전략(자체 제작/구매/외주)이 WBS에 없다. M2 플레이테스트 시 시각적 품질이 피드백 신뢰도에 영향을 미친다. [SPECULATION] 프로그래머 아트로 M2까지 진행할 수 있으나, M3 이전에 에셋 전략 확정 SF를 추가해야 한다.

3. **[M-03 잔존] SF 크기 추정의 맨먼스 환산 부재** -- S/M/L/XL의 주 단위 범위만 있고 "1인 + Claude Code 기준 실작업일"이 없다. 스프린트 과적 여부 판단이 여전히 주관적이다.

4. **[M-06 잔존 + 확장] S1-1/S1-2 스프린트도 과적** -- S1-1에 XL 1개(SF-WD-001) + L 3개(SF-WD-002, SF-WD-007, SF-WD-008) + M 3개(SF-WD-003, SF-WD-004, SF-WD-018) = 중앙값 합산 약 14주를 2주 스프린트에 배정. S1-2도 L 1개 + M 6개 + S 2개 = 약 11주 분량. R1의 [C-01]은 S1-3에 한정하여 지적했으나, S1-1/S1-2에도 동일 패턴이 존재한다. 단, S1-1/S1-2의 SF 대부분은 M0 병렬 트랙에서 상당 부분 선행 완료되거나(건설/인력/경제 로직은 S0-2~S0-4에서 진행), ECS 스파이크의 연속이므로 실질적 "신규 작업"은 축소된다는 점을 감안해야 한다. 이 맥락에서 Critical이 아닌 Major로 유지한다.

5. **[M-07 신규] S2-1 스프린트 과적** -- S2-1(572행)에 EXP 6개(M+S+M+M+L+S) + WD 2개(M+M) + ECO 2개(M+S) + WF 1개(S) = L 1개 + M 5개 + S 4개 = 중앙값 약 12주를 2주에 배정. EXP-005(탐험 탭 UI, L)만으로 2~3주가 소요된다. S2-1을 2개 스프린트로 분할하거나, EXP-005를 S2-2로 이동시키는 것을 권장한다.

6. **[M-05 잔존] M0 병렬 트랙 통합 SF 부재** -- 병렬 매트릭스 690행에 "통합 브랜치"라는 표현만 있고, 통합 작업 자체의 SF(데이터 구조 정합성 검증, 네이밍 통일, 머지 충돌 해소)가 정의되지 않았다. S1-1의 "Bridge Layer 구현"이 사실상 통합 역할을 하지만, worktree 머지 비용이 별도로 산정되어야 한다.

### Minor (R1 잔존 + 신규)

1. **[m-03 잔존] SF-CON-011 우선순위/마일스톤 불일치** -- P1인데 M2 배정. 의도적이라면 근거를 명시해야 한다.

2. **[m-04 잔존] M3 S3-6 과적** -- "전체 P1 통합 테스트 + 밸런싱 이터레이션 + 텍스트 완성 + EA 빌드 준비"를 2주에 수행(586행). 최소 2개 스프린트로 분할 권장.

3. **[m-05 잔존] SF-WD-015 마일스톤 "M1/M2" 모호** -- v1.1에서 S1-4에 "(핵심 4종 우선)"이라고 명시(563행)하여 부분 해소되었으나, 나머지 적 종류가 어느 시점에 구현되는지는 여전히 불명확.

4. **[m-06 신규] SF-AUD-002의 마일스톤 표기 불일치** -- SF-AUD-002는 표(361행)에서 M1으로 배정되었으나, 병렬 매트릭스(691행)에서는 S1-3 병렬 트랙에 배치. SF-AUD-002는 SF-WD-006(M1)에 의존하므로 M1이 맞지만, SF-AUD-001(M1)이 S1-5에 배정(564행)된 상태에서 SF-AUD-002가 S1-3에 오는 것은 의존 순서 위반이다. SF-AUD-001이 SF-AUD-002보다 먼저 완료되어야 한다(AUD001 -> AUD002 의존, 473행).

5. **[m-07 신규] SF-EXP-010과 SF-CON-003의 마일스톤 시차** -- SF-EXP-010은 M2, SF-CON-003은 M1. CON-003이 EXP-010의 이벤트를 "수신"한다고 했는데, EXP-010이 아직 존재하지 않는 M1에서 CON-003이 수신 로직을 구현하려면 인터페이스 스텁이 필요하다. 이는 기술적으로 가능하나 문서에 명시가 없다.

## Pre-Mortem: 실패 시나리오 Top 3

### 시나리오 1: 스프린트 과적 패턴의 반복으로 M1-M2 일정 누적 지연

- **확률**: 중간 (R1에서 높음 -> S1-3 해소로 중간으로 하향)
- **영향**: 중대
- **근거**: S1-3 과적은 해소되었으나, S1-1(XL 1+L 3+M 3 = ~14주), S1-2(L 1+M 6+S 2 = ~11주), S2-1(L 1+M 5+S 4 = ~12주)이 각각 2주 스프린트에 배정(560~572행). 문서 자체의 SF 크기 기준(0.2절, 27~32행)과 스프린트 배분 간의 수학적 불일치가 S1-3 이외의 스프린트에서도 존재한다. 단, M0 병렬 트랙에서 상당수 SF가 선행 진행되는 점(병렬 매트릭스 684~693행)을 감안하면 실제 "스프린트 진입 시점의 잔여 작업량"은 표면적 합산보다 작을 수 있다.
- **완화**: (1) 각 스프린트에 "병렬 트랙에서 이미 진행된 SF"와 "해당 스프린트에서 신규 착수하는 SF"를 분리 표기한다. (2) S1-1/S1-2 진입 시점에서 M0 병렬 트랙의 완료 SF를 실측하고 잔여 작업량 기반으로 스프린트를 재조정한다.

### 시나리오 2: 후반 마일스톤(M3/M4)의 미정의 작업이 일정을 잠식

- **확률**: 높음
- **영향**: 중대
- **근거**: M4 DoD(147~153행)에 "한국어/영어/일본어 로컬라이징 완성"과 "Steam EA 빌드 제출"이 명시되어 있으나 이를 위한 SF가 없다(R1 M-01). 아트 에셋 파이프라인(R1 M-02)도 SF 미정의. M3 S3-6에 "전체 P1 통합 테스트 + 밸런싱 이터레이션 + 텍스트 완성 + EA 빌드 준비"가 2주에 압축(586행). 이 미정의 작업들이 M3/M4에서 계획에 없던 SF로 추가되면 8~16주의 일정 확장이 발생할 수 있다.
- **완화**: (1) M3 착수 전까지 로컬라이제이션 파이프라인(텍스트 키 시스템, .csv 추출, 폰트 관리) SF 3개 + 아트 에셋 전략 SF 1개를 추가 정의한다. (2) M4 SF를 현재 4개에서 최소 8~10개로 확장한다. (3) 이 SF들을 확정하지 않더라도 "M3/M4 SF 확장 예정 -- M2 완료 시점에 구체화"라는 플레이스홀더를 문서에 명시하여 의도적 지연임을 기록한다.

### 시나리오 3: SF-AUD-001/002 의존 순서 위반으로 오디오 통합 지연

- **확률**: 낮음
- **영향**: 경미
- **근거**: 병렬 매트릭스(691행)에서 SF-AUD-002(전투 핵심 SFX)가 S1-3에 배치되었으나, SF-AUD-001(AudioManager 인프라)은 S1-5(564행, 693행)에 배치됨. Mermaid 그래프(473행)에서 AUD001 -> AUD002 의존이 명시되어 있으므로, AUD-001이 완료되지 않으면 AUD-002는 착수 불가. 이 불일치가 해소되지 않으면 오디오 통합이 M2로 밀린다.
- **완화**: SF-AUD-001을 S1-3 이전(S0-4 또는 S1-1/S1-2)으로 당기거나, 병렬 매트릭스에서 SF-AUD-002를 S1-5 이후로 이동한다. 병렬 매트릭스 686행에 "SF-AUD-001 시작 (인프라)"가 1~2주에 있으므로, S1-5가 아닌 M0 기간에 AUD-001이 완료되는 것이 원래 의도였을 수 있다 -- 이 경우 스프린트 계획(564행)의 S1-5 배정이 병렬 매트릭스와 모순이므로 스프린트 계획을 수정해야 한다.

## WINQ: 개선 가능 영역

### What Works

1. **C-01 해소 방식이 정확하다**: L 크기 SF 4개를 3개 스프린트로 분산하고, M1 기간을 6~10주로 상향 조정한 것은 SF 크기 기준과 정합적이다. S1-4(L 2개)와 S1-5(L 1개+M 1개)의 배분이 합리적이며, 크리티컬 패스 갱신(501행)도 정확하다.

2. **C-02 해소가 체계적이다**: 단순히 오디오 SF를 추가한 것이 아니라, Vision 6.3의 핵심 경험 3종과의 매핑을 명시적으로 기술했다(366~369행). P0/P1 분배도 적절하다 -- 핵심 SFX(P0)와 전체 사운드스케이프(P1)의 구분이 우선순위와 일치한다.

3. **C-03 해소가 양방향이다**: SF-EXP-010(발행 측) 추가뿐 아니라 SF-CON-003(수신 측)의 설명도 갱신하고, Mermaid 그래프에 양방향 엣지를 추가했다. 인터페이스 계약 이행의 완전성이 보장된다.

4. **SF 총계 정합성**: 변경 이력(720행)에서 "106 -> 112 (P0 76->80, P1 26->28)"로 정확하게 기록. 7.1절(647~667행)의 시스템별 SF 통계가 실제 SF 목록과 일치함을 확인.

5. **병렬 매트릭스 확장**: UI/오디오 트랙 열을 추가(684행)하여 3-트랙 병렬 구조를 명시한 것이 이전 2-트랙 구조보다 현실적이다.

### Improvement Needed

1. **스프린트별 SF 크기 합산 검증표 추가 (R1 WINQ 1번 잔존)**: S1-1/S1-2/S2-1의 과적이 수치로 즉시 드러나도록 각 스프린트의 "배정 SF 크기 중앙값 합산 vs 스프린트 기간" 비교표를 추가해야 한다. 특히 "M0 병렬 트랙에서 이미 진행된 SF"를 분리 표기하면 실질적 잔여 작업량이 명확해진다.

2. **SF-AUD-001 마일스톤 불일치 해소**: 스프린트 계획(564행)에서 S1-5로 배정되었으나, 병렬 매트릭스(686행)에서 1~2주에 시작으로 기술. 실제로 M0 기간에 완료하는 것이 의도라면 스프린트 계획에서 S1-5 배정을 제거하고 M0 병렬 트랙 산출물로 명시해야 한다.

3. **M3/M4 SF 확장 계획 플레이스홀더**: 현재 M4에 SF 4개만 있는 것이 의도적 지연(M2 완료 후 구체화)이라면 그렇다고 명시해야 한다. 아무 설명 없이 12~16주 마일스톤에 SF 4개만 있으면 "빠뜨림"과 구별이 안 된다.

### New Ideas

1. **스프린트 "실질 작업량" 열 추가**: 각 스프린트 테이블에 "해당 스프린트에서 실제 착수하는 SF" vs "이전 스프린트/병렬 트랙에서 이미 진행 중인 SF"를 구분하면, 표면적 과적 vs 실질적 과적을 구별할 수 있다.

2. **"일정 압박 시 스코프 컷 후보" 열**: 각 마일스톤에서 DoD 달성에 반드시 필요한 SF와 "있으면 좋지만 빼도 DoD 통과하는 SF"를 구분 표기. 예: S1-3의 SF-ECO-011(자원 로그, S)은 "개발 편의"이지 DoD 필수가 아니다.

### Questions

1. **SF-AUD-001의 실제 개발 시점은 M0인가 M1 S1-5인가?** 병렬 매트릭스(686행)와 스프린트 계획(564행)이 서로 다른 시점을 가리킨다.

2. **S1-1/S1-2의 SF가 M0 병렬 트랙에서 얼마나 선행 완료되는가?** 현재 문서에서는 병렬 트랙의 "진행" 범위가 "시작"인지 "완료"인지 불명확하다. 예: S0-3 병렬 트랙에 "SF-CON-004/005/006/007 진행"이라 했을 때, S1-2 진입 시점에서 이들이 몇 % 완료 상태인가?

3. **로컬라이제이션 텍스트 키 시스템은 언제 도입하는가?** M3 DoD(132행)에 "한국어/영어 텍스트 1차 완성"이 있는데, 텍스트 키 시스템 없이 하드코딩하면 M4 3개국어 확장 시 비용이 폭발한다.

## 긍정적 평가

- **1차 리뷰 Critical 3건의 해소 품질이 높다.** 단순 수정이 아니라 구조적 반영(스프린트 분할 + M1 기간 조정, 오디오 섹션 신설 + Vision 매핑, EXP-010 + CON-003 양방향 갱신)이 이루어졌다.
- **변경 이력(720행)의 변경 추적이 정확하다.** v1.0 -> v1.1의 변경 내용이 실제 문서 변경과 일치하며, SF 총계 변동(106->112)도 정확하게 기록되었다.
- **기술 리스크 선행 배치 + 폴백 게이트 설계**가 건재하다. 이 전략은 프로젝트 최대 리스크(DOTS)에 대한 가장 효과적인 대응이다.
- **112개 SF의 전체 분해 완성도**가 V2 기획 파이프라인 산출물 중 가장 높은 수준이다. P0 80개, P1 28개, P2 4개의 분포가 Vision의 우선순위 계층과 정확히 일치한다.
- **오디오 시스템의 P0/P1 분리**가 현실적이다. M1에 AudioManager 인프라와 전투 핵심 SFX만 넣고, BGM 크로스페이드와 전체 사운드스케이프는 M3으로 미룬 것이 1인 개발 제약에 맞다.
