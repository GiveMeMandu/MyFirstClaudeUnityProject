## 1. 1차 리뷰 이슈 해결 검증

### Critical 이슈

#### C1: 턴별 예산표의 자재 잔고가 수학적으로 성립하지 않는다 (음수 잔고)
- **1차 지적**: Per-Turn-Budget.md 부록 상세 표에서 턴 15의 자재 잔고가 -19로 표기. 이상적 시나리오에 음수를 포함시키는 것은 자기모순.
- **수정 상태**: ✅ 해결
- **검증 내용**: v1.0의 부록 상세 수치표는 모든 턴에서 잔고가 0 이상이다. 계산 규칙(섹션 "계산 규칙" 항목 1, 5)에 "음수 불가", "이론적 음수 발생 시 의사결정 변경으로 해소"가 명시되었다. 방벽 2차 확장 비용을 분할 납부(25+20 -> 25+12 축소)로 변경하고, 건설 이연/비용 축소를 적용하여 음수를 제거했다. 실제 확인 결과 전 25턴에서 기초 최저 53(T21), 고급 최저 4(T19)로 음수 없음.

#### C2: Per-Turn-Budget.md 스냅샷과 부록 상세 표의 수치 불일치
- **1차 지적**: 턴 5 스냅샷과 상세 표의 기초/고급/인력이 모두 다르다. 어느 것이 권위 있는 데이터인지 판단 불가.
- **수정 상태**: ⚠️ 부분 해결 (수치 정합성은 버티컬 슬라이스 밸런싱 단계에서 최종 확정)
- **검증 내용**: 범례에 "부록 상세 수치표가 단일 권위 소스(single source of truth)이며, 스냅샷은 해당 표에서 파생된다"가 명시되었다. v1.0에서 스냅샷과 부록의 수치가 대체로 정합하나, 턴 5/8의 일부 세부항목에서 소폭 차이 잔존. 부록이 권위 소스이므로 기능적으로 해소됨.

#### C3: 희망/불만 게이지의 경제적 효과가 완전히 누락
- **1차 지적**: Vision.md 5.8의 불만 임계치/시민 이탈이 Economy-Model.md에서 정량적으로 정의되지 않음. 자원 부족의 정량적 정의, 불만 상승량, 역효과 전무.
- **수정 상태**: ✅ 해결
- **검증 내용**: Economy-Model.md 섹션 8 "희망/불만 경제 연동"이 신규 추가되어 다음을 정의한다:
  - 게이지 기본 스펙(범위 0~100, 초기값, 게임오버 조건, 위험 임계치, 자연 변동) -- 섹션 8.1
  - 경제 상태 -> 게이지 영향(기초 자원 잔고 충분/부족/경고, 방어 결과별 효과, 시민 사망, 건물 완성, 집회 등 12가지 트리거) -- 섹션 8.2
  - 게이지 -> 경제 역효과(불만 30 미만: 정상, 50~79: 생산 -10%, 80~99: 생산 -25% + 이탈 30%, 희망 11~20: 이탈 20% 등) -- 섹션 8.3
  - 자원 고갈 모드(기초 자원 0 시 건설/수리 불가, 불만 +8/턴, 희망 -3/턴) -- 섹션 8.4
  - Per-Turn-Budget.md의 부록 상세 표에 희망/불만 열이 추가되어 전 25턴의 게이지 추이가 수치화됨.
  - 이것은 1차 리뷰의 가장 심각한 누락을 실질적으로 해결한 우수한 수정이다.

### Major 이슈

#### M1: 건물 건설 소요 턴이 경제 모델에 정의되지 않았다
- **1차 지적**: 건설 즉시 완료인지 1턴 소요인지에 따라 자원 흐름이 완전히 달라진다.
- **수정 상태**: ✅ 해결
- **검증 내용**: Per-Turn-Budget.md 전제 조건 표에 "건설 소요 시간: 1턴 (건설 턴에 자재 소비, 다음 턴부터 가동)"이 명시되었다. 부록 표에서도 "농장2 건설"(턴 1) -> "농장2 가동 대기"(턴 2) -> 턴 3부터 농장2 수입 +6 반영으로 일관성 확인.

#### M2: 인력 배분의 합계가 턴별로 불일치
- **1차 지적**: 스냅샷과 부록의 인력 카테고리 매핑이 불명확.
- **수정 상태**: ✅ 해결
- **검증 내용**: 스냅샷의 인력 배분 표가 상세해졌고, "합계" 행에 총 인력을 명시한다. 턴 1: 농장3 + 채집장2 + 행정관1 + 분대3 + 건설2(겸임) + 미배치1 = 12명. "건설은 별도 인력이 아닌 기존 배치원이 낮에 겸임"이라는 비고로 이중 계산 혼란을 해소. 부록 상세 표의 인력 카테고리(건물/방어/원정/기타)와의 매핑도 스냅샷에서 추론 가능한 수준. 턴 15: 농장7 + 채집장4 + 행정관1 + 분대5 + 원정3 + 연구소1 = 21명, 부록 인구 21명과 일치.

#### M3: 원정 비용의 시간축 정의가 없다
- **1차 지적**: 원정 보급비가 파견 시 1회인지 매 턴인지 불명확.
- **수정 상태**: ✅ 해결
- **검증 내용**: v1.0에서 원정 보급은 기초 자원으로 통합. Economy-Model.md에서 원정 보급이 "파견 시 1회 비용"으로 명시. Per-Turn-Budget.md 부록 상세 표에서 파견 시점에만 기록되어 일관.

#### M4: 숙련도 "+5~15%" 보너스의 적용 대상이 불명확
- **1차 지적**: "효율"이 무엇에 적용되는지(생산량? 건설 속도? 공격력?) 모호.
- **수정 상태**: ✅ 해결
- **검증 내용**: Economy-Model.md 섹션 4.2에서 숙련도 모델이 전면 개편되었다. 선형 +5~15%를 폐기하고 비선형 보너스(x1.0 -> x1.1 -> x1.3 -> x1.6)를 도입. "전문가 1명 = 미숙련 1.6명분"이라는 구체적 가치 비교를 제시. 전문가 도달을 6~8턴으로 앞당김(기존 8~10턴). 섹션 4.2에 레벨별 기능 해금을 추가: 전투Lv2="선제 사격", 건설Lv2="수리비 -15%", 탐사Lv2="보물 발견 확률 +20%". 섹션 4.3에서 각 배치 영역별로 숙련도 적용 대상을 명시(예: "숙련도(건설) x1.0~x1.6 + 인원수로 생산량 결정").

#### M5: 저장소 건물의 인력 1명 배치가 경제적으로 정당화되지 않는다
- **1차 지적**: 인력이 최희소 자원인데 캡 증가만을 위해 1명을 상시 배치하는 것이 세금처럼 느껴짐.
- **수정 상태**: ✅ 해결
- **검증 내용**: Economy-Model.md 섹션 5.1의 저장소 건물이 "필요 인력: 0명"으로 변경되었다. 인력 배치 없이 저장 캡만 증가하는 건물로 전환. 이로써 저장소는 자재 비용(18)만 소비하고 인력 부담이 없다.

### Major (강력 권장 수정)

#### M-NEW-1: 스냅샷 턴 5/8의 내부 산술이 부록 상세 표와 소폭 차이

v0.2→v1.0 전환 과정에서 스냅샷 세부 수치와 부록 간 소폭 차이 잔존. 부록이 단일 권위 소스로 선언되어 기능적 영향 없음. **버티컬 슬라이스 밸런싱 단계에서 최종 정합 예정.**

#### M-NEW-2: 후반 고급 자원 여유 — 버티컬 슬라이스 이후 밸런싱 대상

후반(턴 19~25) 고급 자원 잔고가 안정적으로 축적되는 경향. 설계 의도("고급 자원이 성장 병목")와 다소 모순. **수치 밸런싱은 버티컬 슬라이스 이후 처리 예정** — 현 단계에서 구조적 문제는 아님.

### Minor (선택적 개선)

#### m-NEW-1: 턴 10 스냅샷의 인력 합계 불일치

턴 10 스냅샷 인력 배분: 농장 1+2+3 = 8명, 채집장 1+2 = 4명, 행정관 1명, 방어 분대 4명, 원정대 0명, 미배치 1명. 합계 = **18명**. 그러나 해당 턴의 인구는 **17명**(부록 기준: 턴 9 = 15명, 턴 10에 +1 = 16명... 사실 부록에서 턴 10은 16명이다). 스냅샷 합계 18명과 부록 인구 16명 사이에 2명의 차이가 존재한다. 스냅샷 자체의 인력 열에도 "17명"이라고 기재되어 있으나 배분 표의 합계는 18명이다.

#### m-NEW-2: 모닥불 건물의 성공 확률(70%)이 경제 시뮬레이션에 미반영

Economy-Model.md 섹션 5.3에서 모닥불은 "식량 8~12 투자 -> 2턴 후 유민 1명 합류 확률 70%"로 정의된다. 그러나 Per-Turn-Budget.md 전체에서 모닥불을 사용하는 시나리오가 없다. 능동적 인구 증가 수단으로 추가한 것은 좋지만, 참조 시나리오에서 한 번도 활용되지 않으면 설계 의도가 예산표에서 검증되지 않는다. 모닥불을 1~2회 사용하는 시나리오 분기를 부록 하단에 메모로 추가하면 좋다.

#### m-NEW-3: 희망/불만 그래프의 범례가 불분명

Per-Turn-Budget.md 섹션 3의 희망/불만 추이 ASCII 그래프에서 "희망(실선)"과 "불만(점선)"이 표기되어 있으나, ASCII에서 실선과 점선의 시각적 구분이 불가능하다. 하나의 그래프에 두 게이지를 오버레이하면 판독이 어렵다. 두 그래프를 분리하거나, 희망은 'H', 불만은 'D'로 마커를 구분하면 가독성이 향상된다.

## 4. Pre-Mortem 업데이트: 1차 대비 변화

### 시나리오 1 (1차): "자재 데스 스파이럴" -- 방어 실패가 회복 불가능한 경제 붕괴
- **1차 확률**: 높음 -> **2차 확률**: 낮음~중간
- **근거**: 데스 스파이럴 안전판(잔해 수거 40~60%, 적응형 웨이브 약화 -15~20%, 비상 수리 면제)이 3단계로 구축되었다. 부록 표의 턴 20에서 대규모 돌파 발생 시 잔해 수거 +8, 턴 21에서 적응형 웨이브 약화가 작동하여 회복에 성공하는 시나리오가 제시됨. 다만, 비상 수리 면제의 "연속 3턴 초과 적용 불가" 제한이 있어 **4회 이상 연속 실패** 시에는 여전히 데스 스파이럴 가능.

### 시나리오 2 (1차): "인력 배분이 퍼즐이 아니라 산수"
- **1차 확률**: 중간 -> **2차 확률**: 낮음
- **근거**: 비선형 숙련도(x1.6)와 레벨별 기능 해금(선제 사격, 수리비 감면 등)이 인력 배치에 질적 변화를 추가. "이 전문가를 농장에 둘 것인가 분대에 넣을 것인가"가 x1.0 vs x1.6의 차이 + 기능 해금/상실을 동시에 고려하는 의미 있는 의사결정이 됨.

### 시나리오 3 (1차): "기초 자원 경제의 단조로움"
- **1차 확률**: 중간 -> **2차 확률**: 낮음~중간
- **근거**: v1.0에서 기초 자원의 능동적 Sink(분대 장비, 모닥불 집회, 수리비 스케일링)가 추가됨. 부록 표에서 턴 21의 모닥불 집회가 의미 있는 의사결정으로 작동함을 확인.
