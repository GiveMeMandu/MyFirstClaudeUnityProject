## 2. 핵심 루프

```
[게임 시작 — 팩션/플레이어매트 선택]
      ↓
[턴 진행 — 4개 행동 중 하나 선택]
  ├─ 이동(Move): 유닛 이동 + 인카운터 발동 가능
  ├─ 생산(Produce): 타일 위 워커로 자원 생산
  ├─ 무역(Trade): 자원 구매 + 인기도 상승
  └─ 볼스터(Bolster)/확장(Enlist 등): 전투력/엔진 강화
      ↓
[인카운터 헥스 진입 시 — 인카운터 카드 발동]
  ├─ 카드 공개 → 3가지 선택지 제시
  ├─ 플레이어 1가지 선택
  └─ 결과 적용 (자원/인기도/능력 중 하나)
      ↓
[별(Star) 조건 충족으로 게임 종료]
  ├─ 전투 승리, 목표 달성, 업그레이드 완료 등
  └─ 최종 점수 계산 (인기도 트랙 곱수 × 자원/영토/별 합산)
```

**핵심 구조**: "엔진빌딩 + 인카운터를 통한 내러티브 스포트". 게임의 주요 루프는 자원 엔진 구축과 영토 확장이지만, 인카운터가 각 플레이어의 세션에 고유한 이야기와 비대칭 보상을 삽입.

## 3. 시스템 분석

### 3.1 인카운터 시스템 (핵심 분석)

- **메커니즘**: 플레이어의 유닛(또는 메크)이 인카운터 토큰이 있는 헥스 타일에 진입할 때 발동. 해당 헥스의 토큰을 제거하고 인카운터 카드 덱에서 1장을 공개. 카드에는 항상 3가지 선택지가 제시되며, 플레이어는 그 중 하나만 선택. 선택 후 토큰은 제거되어 동일 헥스에서 재발동 불가.

- **3선택지 구조 원칙**:

  인카운터 카드의 3선택지는 일반적으로 다음 계층으로 설계됨:

  | 선택지 번호 | 유형 | 특징 |
  |---|---|---|
  | 선택지 1 | 즉각 자원 획득 (안전) | 비용 없음, 인기도 소모 없음. 팝(인구)/코인/전투력 중 하나를 소량 획득. 리스크 없음 |
  | 선택지 2 | 능력 강화 또는 건물 배치 | 중간 비용 또는 조건부 획득. 자원 투자로 더 큰 지속 효과 |
  | 선택지 3 | 영구 업그레이드 또는 고자원 (도전) | 상당한 비용(인기도 소모 등) 또는 조건 필요. 가장 강력하거나 독특한 효과 |

- **수치 데이터**:
  - 기본 게임 인카운터 카드 수: 28장 [공식 — Stonemaier Games 공식 구성 목록]
  - 확장팩 "Scythe Encounters" 추가 카드: 32장 프로모 카드 [공식]
  - 인카운터 토큰 수: 맵의 특정 헥스에 배치 (플레이어 수에 따라 조정) [공식]
  - 발동 조건: 유닛이 인카운터 토큰이 있는 헥스 진입 시 (진입 턴에만 발동) [공식]
  - 동일 헥스 재발동: 불가 (토큰 1회 제거) [공식]
  - 게임당 총 인카운터 발생 횟수: 플레이어 수 × 평균 2~4회 (맵 탐험 범위에 의존) [추정 — BGG 리뷰 기반, 밸런스 직접 사용 금지]

- **선택지 콘텐츠 유형 예시** (커뮤니티 리뷰 기반):

  | 보상 유형 | 예시 내용 |
  |---|---|
  | 팝(인기도 트랙) | 인기도 +1~2 (최종 점수 곱수에 영향) |
  | 자원 | 식량/목재/금속/석유 2~4개 즉각 획득 |
  | 코인 | 골드 +2~5 |
  | 워커(인력) | 워커 1명 즉각 배치 |
  | 업그레이드 토큰 | 업그레이드 액션 1회 무료 실행 |
  | 특수 능력 | 이번 게임 한정 고유 능력 부여 (팩션 의존) |

- **선택의 비가역성**: 선택 직후 즉시 적용되며 취소 불가. 단, 일부 선택지는 "이번 턴 추가 이동력 +1"처럼 단기 효과를 주기도 하며 이는 실질적으로 비가역이 아님. 영구 업그레이드나 팝 변화는 완전 비가역.

- **조건부 4번째 선택지**: Vision.md에서 언급된 "조건부 4번째 선택지"는 확장팩(The Wind Gambit 등)에서 도입된 팩션 고유 능력 또는 특정 조건(특정 자원 보유, 특정 타일 인접 등)을 만족할 때만 활성화되는 추가 선택지로, 기본 게임에는 존재하지 않음 [위키 — BGG 확장팩 FAQ 기반, 추정].

- **플레이어 경험**: "이 카드를 지금 당장 안전하게 가져갈 것인가, 아니면 더 큰 것을 위해 자원을 투자할 것인가"의 상황 판단. 각 선택지가 명확히 다른 전략적 방향을 가리키므로 선택의 무게감이 크지 않으면서도 고유한 이야기를 만든다. Matt Jarvis(Tabletop Gaming 리뷰어)는 인카운터를 "게임의 소우주를 마무리하는 미시적 인간의 순간"이라고 표현.

- **우리 게임 적용**: 직접 — Vision.md 시스템 5.5 인카운터/이벤트 시스템 "중요 인카운터" 구조의 원형. "안전(1번) vs 투자(2번) vs 대가(3번)" 3계층 구조를 Project_Sun 중요 인카운터에 직접 적용.

### 3.2 인기도(Popularity) 트랙 — 선택의 사회적 비용

- **메커니즘**: 인기도는 0~18 트랙으로, 최종 점수 계산 시 곱수 역할(인기도 낮으면 별 1개당 2점, 높으면 5점). 인카운터 선택지 중 일부는 자원을 많이 주지만 인기도를 소모함. "나쁜 행동을 하면 단기 이득, 장기 손해"를 자연스럽게 구현.

- **수치 데이터**:
  - 인기도 트랙: 0~18, 3구간으로 분리 (0~3, 4~12, 13~18) [공식]
  - 구간별 점수 곱수: 0~3구간 = 별 1개당 2~3점, 13~18구간 = 별 1개당 5점 [공식]
  - 인카운터에서 인기도 소모: 최대 -2~-3 (선택지 유형에 따라) [추정 — 커뮤니티 기반, 밸런스 직접 사용 금지]

- **우리 게임 적용**: 영감 — Project_Sun의 희망/불만 게이지가 이 구조와 유사. 인카운터 선택의 사회적 비용(인기도 소모 = 불만 상승)과 물질적 이득(자원 획득) 간의 트레이드오프는 직접 참고.

### 3.3 비대칭 팩션 시스템 — 고유 인카운터 반응

- **메커니즘**: 7개 팩션 각각이 고유 능력을 보유. 일부 인카운터 선택지는 팩션 능력에 의해 더 강력해지거나 추가 옵션이 활성화됨. 이로 인해 동일한 인카운터 카드가 팩션에 따라 다른 최적 선택을 만든다.

- **수치 데이터**:
  - 기본 팩션 수: 5개 + 2개 추가 확장 = 7개 [공식]
  - 팩션별 고유 능력: 각 팩션당 1~2개 패시브 + 1개 액티브 [공식]

- **우리 게임 적용**: 영감 — Project_Sun에서 기지별 고유 기술 트리가 인카운터 선택지의 최적화를 바꾸는 구조로 유사하게 구현 가능.

## 4. UX/UI 패턴

- **카드 레이아웃**: 인카운터 카드는 상단에 플레이버 텍스트(상황 설명), 하단에 3가지 선택지가 번호+비용+보상으로 명확히 분리. 선택지 간 시각적 계층이 명확.
- **선택지 비용 표기**: 선택지 2~3번에 자원 비용 아이콘이 즉시 표시되어 비용-보상 관계를 직관적으로 파악.
- **인카운터 발동 시각화**: 유닛이 토큰 헥스에 진입할 때 명확한 시각/청각 피드백으로 인카운터 발동 통지.
- **디지털판(Steam)**: 카드가 화면 중앙에 애니메이션으로 등장, 선택지 마우스 호버 시 결과 미리보기 제공.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **인카운터의 내러티브 가치**: "단순 보상이 아닌 작은 이야기" — 각 카드의 플레이버 텍스트가 세계관 몰입을 강화
- **3선택지의 명확한 계층**: "어떤 상황에서 어떤 선택이 좋은가"가 직관적으로 이해됨
- **선택의 고통이 적음**: 3선택지 모두 어느 정도 유효하여 "무조건 손해"인 선택이 없음 → 분노 유발 없이 전략적 갈등
- **팩션 비대칭으로 재발견**: 같은 카드가 팩션에 따라 다른 의미 → 높은 리플레이 가치

### 약점 (자주 비판받는 점)
- **인카운터 빈도가 낮음**: 게임당 발동 횟수가 적어 인카운터의 내러티브 영향이 작다는 비판 (BGG 커뮤니티)
- **운의 요소**: 인카운터 카드 순서가 랜덤이어서 강력한 카드가 나쁜 타이밍에 등장하거나 약한 카드만 연속 등장하는 경우
- **선택지 3번의 진입 장벽**: 가장 강력한 선택지가 조건/비용을 요구하여 초보자가 놓치는 경우 많음
- **디지털판 UI**: 보드게임 원본의 물성적 피드백(카드 만지기)이 디지털에서 재현되지 않아 체감이 약해진다는 의견

## 6. 적용 포인트

| 요소 | 적용 방식 | 우선순위 |
|---|---|---|
| 3선택지 계층 구조 (안전/투자/대가) | 직접 — Vision.md 시스템 5.5 중요 인카운터의 핵심 원형. 선택지 1=안전(소보상), 선택지 2=투자(자원 소모→큰 보상), 선택지 3=대가(희망/불만 영향→특수 보상) | P1 |
| 비용 타입 분리 (물질적 vs 사회적) | 직접 — 인카운터 선택 비용을 "자원 소모" vs "희망/불만 변화"로 분리 설계. Scythe의 "자원 vs 인기도" 구조 차용 | P1 |
| 인카운터 토큰 1회 발동 규칙 | 직접 — 동일 노드에서 중요 인카운터 재발동 방지. 탐사로 새 노드 개방 시에만 발동 | P1 |
| 플레이버 텍스트로 세계관 전달 | 직접 — 각 인카운터 카드에 짧은 상황 설명으로 기지별 고유 서사 전달 | P1 |
| 조건부 4번째 선택지 | 변형 — 행정관 능력 단계(Lv2)에서 특정 인카운터의 4번째 선택지 해금. 고수 플레이어에게 추가 선택지 제공 | P2 |
| 인카운터 빈도 조절 | 영감 후 보완 — Scythe의 인카운터 빈도가 낮아 내러티브 영향 약함을 반면교사. Project_Sun은 일상 인카운터(빈번, 이진)와 중요 인카운터(드물지만 강력)로 이원화하여 내러티브 밀도 확보 | P1 |

## 7. Scythe 인카운터 vs Project_Sun 인카운터 직접 비교

| 항목 | Scythe 인카운터 | Project_Sun 중요 인카운터 |
|---|---|---|
| 발동 조건 | 유닛이 인카운터 토큰 헥스 진입 | 원정대가 특정 노드 도달 |
| 선택지 수 | 기본 3개 (+조건부 4번째) | 3개 (+행정관 Lv2 조건부 4번째) |
| 선택지 1 (안전) | 즉각 자원/코인 소량 | 즉각 자원 소량 획득 |
| 선택지 2 (투자) | 자원 투자→능력/건물 강화 | 자원 소모→기술 해금/방어력 강화 |
| 선택지 3 (대가) | 인기도 소모→강력한 효과 | 희망 감소 또는 불만 증가→특수 효과 |
| 비가역성 | 즉시 적용, 취소 불가 | 즉시 적용, 취소 불가 |
| 발동 후 토큰 제거 | 예 (재발동 불가) | 예 (해당 노드 인카운터 소진) |
| 내러티브 플레이버 | 카드 상단 플레이버 텍스트 | 인카운터 시나리오 텍스트 |
| 팩션/기지 비대칭 | 팩션별 고유 반응 | 기지별 고유 이벤트 풀 + 기술트리 연동 |
