## 2. 핵심 루프

```
[서바이벌 모드 — 랜덤 생성 맵]
      ↓
[초반 탐색 & 기지 구축 페이즈]
  ├─ 자원 채취 건물 건설 (채석장, 목재소, 식량 시설)
  ├─ 군인 유닛 생산 (병영에서)
  ├─ 기지 외곽 방어선 구축 (목재 성벽 → 석재 성벽 → 방어탑)
  └─ 맵 정찰/클리어 (좀비 무리 제거 → 생존 공간 확보)
      ↓
[웨이브 주기] ← 일정 시간마다 발생, 점점 강해짐
  ├─ 방향 표시기로 웨이브 진입 방향 미리보기
  ├─ 기지 외곽 방어선 + 군인 유닛으로 대응
  ├─ 실시간 군인 재배치 가능
  └─ 한 마리라도 건물에 닿으면 연쇄 감염 → 게임오버 위험
      ↓
[최종 웨이브 (Final Swarm)]
  ├─ 맵 전체에서 동시 공격
  ├─ 맵 위 살아남은 모든 좀비 합류
  ├─ 규모: 이전 웨이브의 수십~100배
  └─ 격퇴 시 시나리오 클리어
```

**핵심 구조**: "철저한 준비 → 물량 공세 격퇴의 카타르시스". 방어선 구축의 섬세함과 물량 스케일의 압도감이 핵심 감정.

## 3. 시스템 분석

### 3.1 대규모 물량 웨이브 시스템

- **메커니즘**: 일정 시간(게임 내 일수)마다 좀비 웨이브가 발생. 진입 방향이 사전 공지되며 규모는 회차마다 증가. 최종 웨이브는 이전 모든 웨이브와 비교해 압도적으로 큰 규모로 도달. 맵 위에 남아있는 청소되지 않은 좀비 무리도 최종 웨이브에 합류.
- **수치 데이터**:
  - 게임 기간 설정: 80일(5웨이브) ~ 150일(10웨이브) [위키 — TAB Fandom Wiki 기반]
  - 예시 웨이브 규모 (80일 게임 기준): W1: 5마리 → W2: 12 → W3: 18 → W4: 24 → W5(최종): ~140마리 이상 [추정 — Steam 토론 기반, 밸런스 직접 사용 금지]
  - 최종 웨이브 규모: 이전 웨이브 대비 약 10~100배 급증 (비선형 스케일링) [추정 — 커뮤니티 관찰, 밸런스 직접 사용 금지]
  - 최종 웨이브 구성: 기본 좀비 + Harpy(비행), Venom(독), Chubby(고체력) 특수 유닛 다수 포함 [위키]
  - 4방향 동시 공격 + 맵 잔존 좀비 합류 [위키]
- **플레이어 경험**: "최종 웨이브 전까지는 잘 막아왔는데 갑자기 100배가 온다"는 충격과 공포. 단, 준비가 충분했다면 압도적 물량을 막아냈을 때의 카타르시스는 최상급. 스트리머 콘텐츠로 바이럴에 매우 적합.
- **우리 게임 적용**: 직접 — Vision.md "물량 방어 카타르시스" + "DOTS 대규모 물량" 레퍼런스의 핵심. Project_Sun은 이 규모를 턴제로 구현하는 것이 핵심 기술 목표.

### 3.2 기지 건설 및 방어선 시스템

- **메커니즘**: 자유 배치 방식. 건물을 어디에 놓을지 플레이어가 완전히 결정. 성벽/방어탑의 배치와 연결이 방어 전략의 핵심. 성벽에 구멍이 생기면 좀비가 쏟아지고, 한 마리가 건물에 닿으면 연쇄 감염으로 기지 전체가 무너질 수 있음.
- **수치 데이터**:
  - 건물 종류: ~30~40종 (자원, 생산, 방어, 특수) [위키]
  - 자원 종류: 식량, 목재, 석재, 철, 골드, 에너지 (~6종) [공식 — 게임 내 시스템]
  - 방어 건물: 목재 성벽, 석재 성벽, 타워(궁수/대포 등), 함정 [위키]
  - 군인 유닛: 병사, 궁수, 루시, 레인저, 탱커 등 ~6~8종 [위키]
  - 건물 체력: 목재 성벽 HP ~2~3, 석재 성벽 HP ~5~8 [추정 — 커뮤니티 관찰, 밸런스 직접 사용 금지]
- **플레이어 경험**: 방어선 설계가 레벨 디자인이자 퍼즐. "어디에 성벽을 쌓고 어디에 타워를 놓을 것인가"가 끝없는 최적화 동기를 자극.
- **우리 게임 적용**: 영감 — Project_Sun은 고정 슬롯 방식으로 자유 배치 대신 업그레이드/배치 순서로 단순화. 단, TAB의 "방어선 구축의 레이어드 설계" 개념(앞선 외곽 방어 → 내부 타워 백업)은 적용 가능.

### 3.3 자원 경제 시스템

- **메커니즘**: 6종 자원(식량/목재/석재/철/골드/에너지)의 복합 경제. 자원 채취 건물이 자원 소모 건물(인구 유지, 군인 유지)보다 많아야 생존 가능. 인구 증가 → 식량 소모 증가 → 더 많은 농장 필요 → 방어선 확장 필요 → 자원 더 필요의 순환 고리.
- **수치 데이터**:
  - 식량: 시민 1명당 ~0.5식량/틱 [추정 — 커뮤니티 데이터마이닝, 밸런스 직접 사용 금지]
  - 골드: 건물 건설/업그레이드 비용 [공식]
  - 에너지: 일부 건물 작동에 필요 (발전기 필요) [공식]
  - 목재/석재/철: 건설 재료 [공식]
- **플레이어 경험**: "확장하면 할수록 유지비가 늘어난다"는 경제적 압박. TAB의 가장 어려운 측면 중 하나로 지목되며, 이 복잡성이 높은 학습 곡선을 만든다.
- **우리 게임 적용**: 영감 후 단순화 — Project_Sun은 자원 3종 이하를 목표. TAB의 6종 자원 복잡도는 "제거할 진입 장벽"의 사례. 단, "확장하면 유지비가 늘어나는 트레이드오프" 구조는 방벽 확장 시스템에 반영 가능.

### 3.4 연쇄 감염 게임오버 메커니즘

- **메커니즘**: 좀비가 건물에 닿으면 해당 건물 내 인구가 즉시 좀비로 전환되어 인접 건물로 연쇄 전파. 단 1마리의 침투가 기지 전체를 멸망시킬 수 있음. 성벽의 단 한 칸 구멍이 치명적.
- **수치 데이터**: 감염 전파 속도: 몇 초 내 연쇄 [추정 — 커뮤니티 관찰, 밸런스 직접 사용 금지]. 방어선 구멍 → 기지 전멸까지 ~30초~2분 [추정 — 커뮤니티 관찰, 밸런스 직접 사용 금지].
- **플레이어 경험**: 극도의 긴장감과 스트레스. "완벽한 방어"를 강제하는 설계. 스트리머 콘텐츠(참사 장면)에 최적화. 단, 일반 유저에게는 과도한 스트레스 요인으로 작용.
- **우리 게임 적용**: 영감 후 완화 — Project_Sun은 턴제이므로 "즉각 연쇄 전파"는 없음. 그러나 "한 지점이 뚫리면 전략적 불이익"이 발생하는 구조는 유지해야 카타르시스가 생김.

## 4. UX/UI 패턴

- **웨이브 방향 표시**: 다음 웨이브의 진입 방향을 화살표+색상으로 미리 표시. 방어 준비를 계획할 수 있는 핵심 정보 제공.
- **건설 그리드**: 건물 배치 시 배치 가능 영역을 그리드로 표시. 겹침/충돌 방지.
- **유닛 명령 UI**: 군인 선택 → 이동 명령의 간단한 RTS 방식. 그룹 선택 가능.
- **리소스 HUD**: 상단에 6종 자원 수치 + 틱당 변화량 표시. 자원 흐름을 실시간으로 파악 가능.
- **카메라**: 탑뷰 2.5D. 마우스 스크롤로 줌/패닝.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **압도적 물량 스케일**: 수천 마리 좀비의 시각적 임팩트가 업계 최고 수준
- **준비-검증 카타르시스**: 꼼꼼한 준비 → 최종 웨이브 격퇴의 쾌감이 강력
- **스트리머 친화성**: 참사/역전 장면이 시청자 반응을 유발 → 바이럴 마케팅에 효과적
- **서바이벌 모드 다양성**: 랜덤 맵 생성으로 높은 리플레이 가치

### 약점 (자주 비판받는 점)
- **극도의 실시간 스트레스**: 실수 하나로 30분+ 플레이가 날아감 → 좌절감이 강함 (Steam 부정 리뷰 다수)
- **캠페인 밸런스 실패**: 정식 출시 후 캠페인 미션 밸런스가 불균일하다는 커뮤니티 비판
- **개발사 사후지원 부족**: 정식 출시 후 업데이트 거의 없음. 커뮤니티 불만 다수
- **성능 이슈**: 고사양 PC에서도 대규모 좀비 처리 시 프레임 저하 (2021년 기준 최상급 PC에서도 60fps 불안정)
- **비선형 최종 웨이브 밸런스**: "이전 웨이브를 막으면 최종도 막을 수 있겠지"라고 생각하게 만드는 설명 부재
