## 2. 핵심 루프

```
[시대 시작 — 문명 & 맵 선택]
      ↓
[턴 진행 사이클]
  ├─ [도시 관리]
  │   ├─ 시민 타일/전문가 배치 (생산력/식량/금 최적화)
  │   ├─ 생산 큐 관리 (건물, 유닛, 불가사의 건설 순서)
  │   └─ 전문가 배치 (위인 포인트 생성)
  ├─ [외교]
  │   ├─ 다른 문명과의 협상/동맹/전쟁 선언
  │   └─ 도시국가와의 관계 관리
  ├─ [연구]
  │   └─ 기술 트리 다음 연구 선택
  ├─ [사회 정책]
  │   └─ 문화 포인트로 사회 정책 트리 해금
  ├─ [군사]
  │   └─ 유닛 이동/전투/도시 포위
  └─ [탐험]
      └─ 정찰대 파견, 미발견 지형 탐색
      ↓
[다음 턴] ← "한 턴만 더" 심리
      ↓
[승리 조건 달성]
  ├─ 과학(우주선 발사)
  ├─ 문화(관광 지배)
  ├─ 외교(유엔 표결)
  ├─ 지배(모든 원래 수도 점령)
  └─ 시간(서기 2050년 도달)
```

**핵심 구조**: "매 턴 작은 선택들이 쌓여 장기 전략을 결정"하는 복리 의사결정 구조. "한 턴만 더"의 원형.

## 3. 시스템 분석

### 3.1 시민 타일/전문가 배치 시스템

- **메커니즘**: 각 도시에 인구(시민)를 타일 또는 전문가 슬롯에 배치. 타일은 식량/생산력/금을 생산하며 지형에 따라 수치가 다름. 전문가는 특정 건물(대학, 공장, 사원 등)에서만 배치 가능하며, 대신 위인 포인트를 생성. 자동 관리(AI 거버너) 또는 수동 관리 선택 가능.
- **수치 데이터**:
  - 기본 타일 산출: 평원 = 1식량 + 1생산력, 언덕 = 0식량 + 2생산력, 강 인근 = +1금 [공식 — Civ5 공식 가이드]
  - 개선 타일 (농장/광산): 식량 +1, 생산력 +1~3 추가 [위키 — CivFanatics Wiki]
  - 전문가 종류: 상인(+3금), 기술자(+2생산력+위인포인트), 과학자(+3과학+위인포인트) [공식]
  - 위인 포인트 임계: 대과학자 100~300포인트 / 대예술가 150~400포인트 등 (연구/정책에 따라 변동) [추정 — 범위가 넓어 실질적 밸런스 참고 가치 낮음, 밸런스 직접 사용 금지]
  - 전문가 배치 조건: 해당 건물 존재 + 인구 수 충분 [공식]
- **플레이어 경험**: "이 도시를 생산력 특화로 키울 것인가, 과학 특화로 키울 것인가"의 도시별 역할 분담. 인구 성장에 따라 새 시민을 어디에 배치할지 매 턴 결정. 인구가 많을수록 전문가 배치 여유가 생겨 위인 생산 가속화.
- **우리 게임 적용**: 직접 — Vision.md 시스템 5.2 "인력 관리 시스템"의 직접 레퍼런스. "시민을 건물/직책/원정대/방어에 배치. 각 시민의 능력치/적성이 배치 효율에 영향"이 Civ5 시민 배치 시스템의 Project_Sun 버전.

### 3.2 기술 트리 (테크 트리) 시스템

- **메커니즘**: DAG 구조의 기술 트리. 각 기술은 과학(Beakers) 포인트 축적으로 연구. 기술 연구로 새 건물/유닛/개선 해금. 게임 후반으로 갈수록 연구 비용이 기하급수적으로 증가. 사회 정책(Social Policy)은 별도 트리로 문화 포인트로 해금.
- **수치 데이터**:
  - 기술 트리 총 기술 수: ~82개 (기본 게임 기준) [위키 — CivFanatics Wiki]
  - 초기 기술 연구 비용: ~20~60 과학 포인트 [위키]
  - 후기 기술 연구 비용: ~3,000~10,000+ 과학 포인트 [위키]
  - 과학 생산: 도시 인구 × 1 + 도서관/대학/연구소 건물 보너스 [공식]
  - 사회 정책 트리: 5개 트리 계열 (전통/자유/명예/상업/합리주의 등), 각 트리당 5개 정책 [공식]
- **플레이어 경험**: "군사 기술에 집중할 것인가, 과학에 집중할 것인가"의 연구 방향 결정. 다른 문명이 먼저 기술을 연구하면 불이익 → 경쟁 압박.
- **우리 게임 적용**: 직접 — Vision.md 시스템 5.6 "기술 트리 시스템"에 "Civ5 DAG 기술 트리" 명시. Project_Sun은 기지별 고유 트리 + 탐사 연동 해금으로 변형.

### 3.3 "한 턴만 더" 중독성 설계

- **메커니즘**: Civ5의 중독성은 여러 메커니즘의 복합 작용:
  1. **미니 목표의 연쇄**: "다음 턴에 기술 완료", "3턴 후 도시 성장", "5턴 후 불가사의 완성" 등 항상 목전의 작은 목표가 존재
  2. **낮은 진입 장벽**: "다음 턴" 버튼 누르는 비용이 없음
  3. **복리 의사결정**: 초반의 작은 선택이 수십 턴 후 큰 결과로 나타남 → 검증 호기심
  4. **정보 부분 공개**: 안개 속 미발견 지역 → 탐험 동기
  5. **비대칭 진행**: 다른 문명의 상황이 정보 불완전 → 불확실성 긴장감
- **수치 데이터**:
  - 평균 게임 길이: ~300~500턴 [위키]
  - 실제 플레이 시간: 초보 10~30시간 / 숙련 5~15시간 / 게임당 [추정 — Steam 리뷰/커뮤니티 기반]
  - "한 턴만 더" 효과: 커뮤니티에서 Civ 시리즈 중독의 원형으로 인용 [위키 — CivFanatics 커뮤니티]
  - 현재 CCU: 11,000+ (2026-04, 출시 15년 후) [위키 — Steambase 기반]
- **플레이어 경험**: "세 번 더 하고 자야지" → 실제로 세 시간이 지나 있는 경험. 시간 압박 없이 각 턴이 완결되는 구조가 멈추기 어렵게 만듦.
- **우리 게임 적용**: 직접 — Vision.md 핵심 재미 3번 "한 턴만 더 중독성"의 직접 원형. Project_Sun의 미니 목표 설계(다음 날 웨이브 전 탐사 결과 확인, 기술 연구 완료 예상 날짜 표시 등)에 Civ5 원칙 직접 적용.

### 3.4 유닛 헥사 그리드 전투 시스템

- **메커니즘**: 6각형 헥사 타일 기반 턴제 전투. 유닛당 이동력과 전투력이 있으며, 1타일 1유닛 규칙으로 포지셔닝이 중요. 도시 공격 시 포위 공격(Siege)으로 순차적으로 체력 감소. 비전투 지원 유닛(전사제, 의무관 등)이 전투 유닛 지원.
- **수치 데이터**:
  - 유닛 전투력: 1~100 범위, 문명 진행에 따라 강한 유닛 해금 [공식]
  - 이동력: 유닛당 2~4 이동력/턴 [위키]
  - 전투 계산: 공격력 vs 방어력 + 지형 보너스 + 플랭킹 보너스 [공식]
  - 도시 체력: 20~50 HP, 포위 공격으로 매 턴 감소 [위키]
- **플레이어 경험**: 전략적 포지셔닝과 유닛 조합이 중요. 그러나 Project_Sun에는 직접 적용되지 않음.
- **우리 게임 적용**: 영감만 — Project_Sun의 전투는 자동이므로 직접 적용 불가. 단, 지형 보너스 개념(방어 진지 효과 등)을 방어탑 배치 시스템에 응용 가능.

## 4. UX/UI 패턴

- **도시 생산 큐**: 도시 클릭 시 팝업으로 건설 가능 건물/유닛 목록. 드래그로 우선순위 조정 가능.
- **시민 배치 화면**: 도시 내 타일을 클릭하여 시민 수동 배치 또는 AI 자동 배치 전환.
- **기술 트리 UI**: 좌→우 나무 구조, 현재 연구 중인 기술 하이라이트, 각 기술 마우스 호버 시 해금 건물/유닛 표시.
- **턴 종료 전 경고**: 행동하지 않은 유닛이 있으면 경고 표시. 모든 행동 완료 확인 후 "다음 턴" 클릭.
- **통지 로그**: 화면 우측에 상대 문명의 기술 완료, 도시 성장, 불가사의 완성 등 이벤트 통지 목록.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **전략 깊이와 접근성의 균형**: "배우기 쉽지만 마스터하기 어렵다"는 게임 설계의 황금 기준
- **"한 턴만 더" 중독성**: 게임 업계에서 시간 소비 게임의 대명사
- **높은 리플레이 가치**: 23개 문명, 랜덤 맵으로 매번 다른 경험
- **출시 15년 후에도 강한 CCU**: 콘텐츠 완성도와 장기 생명력의 증거

### 약점 (자주 비판받는 점)
- **AI 외교의 비일관성**: AI 외교가 예측 불가능하고 불합리한 결정을 내리는 경우 많음
- **후반부 지루함**: 게임이 길어질수록 매 턴 처리해야 할 유닛/건물 수가 급증 → 관리 피로
- **전쟁 집착 AI**: 평화 플레이를 원해도 AI가 무작위로 전쟁 선포
- **시민 전문가 시스템 복잡도**: 최적화를 위해 수동으로 관리해야 하지만 번거롭다는 평가. Civ6에서는 자동화(District 시스템)로 대폭 단순화됨 (단, 이 분석의 레퍼런스는 Civ5 시스템 기준임. Project_Sun의 인력 관리 원형은 Civ5의 수동 시민 배치 방식이며, Civ6의 자동화 방향은 참고 방향성으로만 활용)

| 시민 배치 = 인력 관리 시스템 원형 | 직접 — Vision.md 인력 관리 시스템의 핵심 레퍼런스 | P0 |
