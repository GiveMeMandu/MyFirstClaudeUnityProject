## 2. 핵심 루프

```
[LoL/TFT 매치 플레이]
        ↓
[은방패 축적] (일일 500~1000개 / ARAM 아수라장 기준)
        ↓
[미니게임 접속 — LoL 클라이언트]
        ↓
[턴 수행] — 1턴당 은방패 10 소모
    ├── 건설/업그레이드 (목재, 석재, 금속, 페트라이트 소비)
    ├── 유닛 훈련 (병영에서)
    ├── 군대 이동 (정착지 간)
    └── 연구 진행 (용맹 소비)
        ↓
[턴 종료] → 자원 생산, 건설 완료, 건설 완료 처리
        ↓
[영토 확장] — 인접 타일 점령 시도 → 자동 전투 (정찰 가능)
        ↓
[이벤트/위협] — 무작위 적 침공 → 방어 전투
        ↓
[챔피언 해금] (뽀삐 → 모르가나 → 케일 → 갈리오, 챕터별)
        ↓
[목표 달성] → 인게임 보상 (스킨, 이모트 등)
        ↓
[LoL/TFT 복귀 — 다음 은방패 적립]
```

**메타 루프 (일간 단위)**:
1. LoL 매치 플레이 → 은방패 적립
2. 미니게임 접속 → 턴 소진 → 진척도 확인
3. 다음 날 재접속

**Act 구조**: Act 1 (데마시아 건국) → Act 2 (위협으로부터의 방어). Act 2는 Act 1 클리어 후 진입 가능.

## 3. 시스템 분석

### 3.1 자원 시스템

**메커니즘:**
5종의 자원이 계층적으로 연결된다. 기본 자원(목재/석재/금속)은 건물에서 턴당 자동 생산되고, 상위 자원(페트라이트)은 특정 건물과 연구를 통해서만 획득 가능하다. 은방패는 게임 외부(LoL 플레이)에서만 수급된다.

| 자원 | 수급 방법 | 주요 용도 |
|---|---|---|
| 목재 | 목재소(Lumberyard) — 15/30/75/150/턴 (레벨 1~4) | 초반 건설 전반 |
| 석재 | 채석장(Quarry) — 10/20/50/100/턴 (레벨 1~4) | 중반 건설, 방어 건물 |
| 금속 | 대장간(Forge) — 5/10/25/50/턴 (레벨 1~4) | 군사, 고급 건물 |
| 페트라이트 | 페트라이트 방앗간 — 1/2/3/턴 (레벨 1~3, 수도 전용) | 고급 연구, 최상위 건물 |
| 용맹(Valor) | 정착지 건설/업그레이드, 전투 승리, 이벤트 달성 | 연구 잠금 해제 |
| 은방패 | LoL/TFT 매치 플레이 | 턴 진행 (1턴 = 10 소모) |
| 식량(Food) | 농장 — 1/2/3/5/턴 (레벨 1~4) | 인구/유닛 유지 |

**수치 데이터 [위키]:**
- 목재소 레벨1: 무료 건설 → 레벨4 비용: 석재 1000 + 금속 500 + 페트라이트 15
- 채석장 레벨1: 목재 100 → 레벨4: 목재 1750 + 금속 500 + 페트라이트 15
- 페트라이트 방앗간 레벨3 (최고): 목재 7500 + 석재 3500 + 금속 2500
- 목재:석재:금속 생산 권장 비율: 5:3:2 [추정 — 커뮤니티 공략 기반]

**플레이어 경험:**
초반에는 목재와 석재만으로 진행되다가, 게임이 진행될수록 페트라이트가 "병목 자원"으로 부상한다. 생산 건물 업그레이드가 곧 전략의 핵심이 된다. 자원 부족으로 인한 선택 압박이 생기는 구조다.

**Project_Sun 적용:** 변형 적용. 페트라이트처럼 특정 구간에서만 등장하는 "희소 자원 병목" 설계는 후반 전략 심화에 효과적이다. 단, 본 게임처럼 외부 플레이 의존 자원(은방패)은 Project_Sun에 부적합 — 자체 생산 턴 경제로 대체 권장.

### 3.2 건물/건설 시스템

**메커니즘:**
정착지(Settlement) 타일마다 건설 슬롯이 고정되어 있다. 자유 배치가 아닌 "정착지 업그레이드 → 슬롯 해금 → 건물 선택"의 계층 구조다. 건물은 연구(Research)로 먼저 잠금 해제해야 지을 수 있다.

**건물 분류:**

| 분류 | 대표 건물 | 특징 |
|---|---|---|
| 자원 생산 | 목재소, 채석장, 대장간, 농장, 페트라이트 방앗간 | 레벨 1~4 업그레이드, 지형 보너스 있음 |
| 군사 | 병영(Barracks), 민병대 사령부 | 병영 레벨2=전 유닛 해금, 민병대는 자동 배치 |
| 방어 | 망루(Watchtower), 병참장(Quartermaster) | 망루: 위협 지연 +2~4턴, 병참장: 인근 전투 피해 +25% |
| 특수/랜드마크 | 베일의 숙녀 사당, 듀랑드 공방, 왕실 무기고 | 전역 버프 제공, 건설 시간 5턴으로 길음 |
| 경제 강화 | 시장(Marketplace), 학당(Academy) | 인접/동일 지형 자원 +5~10% |

**수치 데이터 [위키]:**
- 베일의 숙녀 사당: 목재 1000 + 석재 1000 + 금속 1000, 건설 5턴 → 전 전투 첫 5초 피해 35% 감소
- 시장 레벨3: 목재 12500 + 금속 5000 + 페트라이트 40 → 인접 정착지 자원 +10%
- 왕실 무기고: 석재 10000 + 페트라이트 50 → 전투 보상 +2 용맹, 자원 50% 증가

**플레이어 경험:**
어떤 건물을 먼저 짓느냐의 선택이 실질적이지만, 슬롯 고정으로 인해 "어디에 배치할까"의 공간적 고민은 없다. 의사결정이 "무엇을 먼저?"로 단순화되어 인지 부담이 낮다.

**Project_Sun 적용:** 직접 적용. 고정 슬롯 + 건물 선택 구조는 Thronefall과 동일한 패턴으로 Project_Sun 이미 채택. 건물별 "건설 시간(턴 소모)" 개념은 변형 적용 가능 — 중요 건물에 복수 턴 건설 비용을 부과해 선택 무게 부여.

### 3.3 연구(Research) 시스템

**메커니즘:**
4개 색상 카테고리로 구분된 트리 구조. 연구는 용맹(Valor) 자원을 소모하며, 선행 연구가 완료되어야 후속 연구가 잠금 해제된다. 연구로 새 건물 유형, 유닛 업그레이드, 챔피언 영입이 이루어진다.

| 색상 | 카테고리 | 대표 효과 |
|---|---|---|
| 파란색 (Blue) | 왕국/효율 | 건물 해금, 자원 생산 강화 |
| 빨간색 (Red) | 전투 강화 | 유닛 스탯 업그레이드 |
| 금색 (Gold) | 챔피언 | 챔피언 영입, 챔피언 업그레이드 |
| 녹색 (Green) | 시스템 | 슬롯 확장, 특수 기능 해금 |

**플레이어 경험:**
"다음에 뭘 연구할까"가 매 세션의 핵심 의사결정이다. 카테고리 색상 구분으로 방향성이 직관적이며, 챔피언 해금이 이정표 역할을 해 진척감을 명확히 준다.

**Project_Sun 적용:** 변형 적용. 색상 구분 연구 카테고리 → Project_Sun 기술 트리의 "내정/탐사/방어 3축" 구조와 유사. 챔피언 해금처럼 기술 트리의 특정 노드를 달성하면 명확한 서사 피드백이 나오는 방식은 직접 차용 가능.

### 3.4 군사 유닛 시스템

**메커니즘:**
병영에서 자원을 소비해 유닛을 훈련한다. 유닛마다 전방/후방 배치가 시스템에 의해 강제된다 (보병=전방, 궁수=후방). 챔피언은 별도 슬롯으로 영입되며 일반 유닛과 함께 전투에 참여한다.

**유닛 종류 (확인된 것):**
- 경비병(Guards) — 기본, 초반 전용
- 보병(Soldiers) — 전방 배치 전사
- 궁수(Rangers/Archers) — 후방 배치, 후반 DPS 핵심
- 기동대(Raptor Knights) — 중후반 추가 유닛
- 챔피언 유닛: 뽀삐, 모르가나, 케일, 갈리오 (챕터별 순차 해금)

**추천 조합 [추정 — 커뮤니티 공략]:**
- 초반: 보병 + 궁수 + 뽀삐/모르가나
- 후반: 궁수 2 + 기동대 2 + 챔피언 (갈리오/케일 포함)

**Project_Sun 적용:** 영감. "소수의 명확한 역할 유닛 + 챔피언 클래스" 구조는 단순하면서도 전략적 선택을 만들어낸다. Project_Sun의 방어 자동전투 유닛 구성에서 "전방/후방 포지션 고정" 개념 차용 가능.

### 3.5 전투 시스템 (자동 전투)

**메커니즘:**
전투는 완전 자동 진행(Auto-Battle). 플레이어가 할 수 있는 것은 전투 전에 어느 정착지에 어떤 유닛을 배치하느냐를 결정하는 것뿐이다. 전투 시작 후에는 속도 조절(빠르기 버튼)만 가능하다.

**전투 발생 조건:**
1. 플레이어가 적 점령 타일을 공격할 때 (능동 공격)
2. 무작위 적 침공 위협이 발생할 때 (수동 방어)

**사전 정보 시스템:**
- 적 타일 공격 전: 정찰(Scout) 기능으로 적 병력 규모 확인 가능
- 침공 위협: 몇 턴 전 사전 경고 + 망루(Watchtower)로 지연 시간 추가 확보

**주요 특수 건물 버프:**
- 베일의 숙녀 사당: 모든 전투 첫 5초 피해 35% 감소
- 병참장: 인근 정착지 전투 피해 +25% 증가
- 왕실 무기고: 전투 보상 +2 용맹

**플레이어 경험:**
전투 자체는 "결과를 지켜보는 것"에 가깝다. 긴장감은 "사전 배치와 준비" 단계에서 발생한다. 실시간 조작 피로가 전혀 없으며, 배치가 잘못되면 지기 때문에 전략적 고민은 유지된다.

**Project_Sun 적용:** 영감(전투 방식) + 직접(간접 전략 레이어). Rising Demacia의 완전 자동 전투는 캐주얼 수용성을 검증한 레퍼런스이며, Project_Sun은 사전 배치 결정 구조를 차용하되 전투 중 가벼운 RTS 마이크로(분대 단위 이동/배치)를 추가하여 에이전시를 강화한다. "망루로 위협 지연"처럼 방어 건물이 전투 타이밍을 조정하는 간접 전략 레이어는 직접 차용 가능.

### 3.6 이벤트 연동 구조 (메타게임 설계)

**메커니즘:**
미니게임 자체가 LoL 시즌 이벤트의 "메타게임"으로 설계되었다. 핵심 특징은 세 가지다:

1. **외부 연료 시스템**: LoL/TFT 매치를 플레이해야 은방패를 얻고 그것으로 턴을 진행한다. 미니게임만 독립적으로 즐길 수 없다.
2. **장기 이벤트 기간**: 약 2~3개월에 걸쳐 천천히 진행. 일일 플레이 압박이 낮다.
3. **보상 동선 통합**: 미니게임 목표 완료 → 인게임 스킨/이모트 보상. 보상이 메인 게임 자산으로 연결된다.

**플레이어 경험:**
"오늘 롤 하면서 은방패 쌓아뒀으니까 잠깐 왕국 발전시켜야지"라는 일상적 습관 루프를 형성한다. 의무감은 낮고, 자발적 체크인 유도가 강하다.

**커뮤니티 비판 포인트:**
- 은방패 소모 속도 대비 수급이 느려 진행이 막히는 느낌 ("조금 하다가 접게 됨")
- LoL 플레이 의무감이 부담 ("그 지겨운 롤을 병행해서 해야함")
- 순수 미니게임만 즐기려는 플레이어에게 장벽

**Project_Sun 적용:** 영감 (역학습). 외부 게임 연동 연료 시스템은 독립형 게임에 적용 불가하나, "일일 자연 회복 자원으로 플레이 세션을 제한"하는 구조는 Project_Sun의 하루 턴 수 제한 설계의 참고점이 될 수 있다. 단, 연료 병목으로 인한 불만은 명백한 실패 패턴으로 인지.

## 4. UX/UI 패턴

### 4.1 화면 구성

- **맵 뷰**: 헥스 타일 기반 영토 지도. 현재 소유 정착지와 적/미탐색 타일이 색상으로 구분
- **정착지 상세**: 타일 클릭 → 해당 정착지의 건물 슬롯, 유닛, 자원 현황 표시
- **자원 바**: 화면 상단 항상 표시 — 목재/석재/금속/페트라이트/은방패 잔량
- **연구 패널**: 색상 구분된 4개 카테고리 트리
- **턴 종료 버튼**: 태양 아이콘 (Sun icon) — 중앙 하단 고정

### 4.2 정보 제공 방식

- 건물 건설 전: 완성 후 효과 미리보기 (구매 전 정보 완전 공개)
- 전투 전: 정찰 기능으로 적 병력 숫자/종류 사전 확인 가능
- 위협 발생: 몇 턴 후 침공인지 미리 통보 (망루 연구 시 지연 가능)
- 자원 생산: 턴 종료 시 "이번 턴에 생산된 자원" 요약 화면 표시

### 4.3 인터랙션 단순성

- 턴당 행동 수: 제한 없음 — 은방패가 있으면 턴 종료를 미루고 계속 행동 가능
- 취소/되돌리기: 턴 종료 전까지 대부분 취소 가능
- 자동 저장: 각 행동 후 자동 저장 (세션 이탈 후 재접속 시 이어 플레이)
- 전투 속도: 자동 전투 재생 속도 조절 버튼

## 5. 강점/약점 (커뮤니티 기반)

### 강점

- **낮은 진입 장벽**: LoL 이용자라면 클라이언트에서 즉시 접근 가능. 설치 없음
- **IP 친밀감**: Lux, Garen, Kayle 등 친숙한 챔피언이 등장해 세계관 몰입 유도
- **명확한 이정표**: 챔피언 해금이 챕터별로 설정되어 진척감이 직관적
- **자동 전투의 캐주얼함**: 실시간 조작 피로 없이 전략적 긴장감 유지
- **장기 이벤트 구조**: 매일 조금씩, 의무감 없이 즐길 수 있는 속도
- **사전 정보 공개**: 정찰/위협 사전 통보로 "억울한 패배" 최소화
- **서사 연결**: Lux의 성장 서사가 게임 진행과 맞물려 몰입도 부여

### 약점

- **은방패 병목**: 미니게임 내 진행 속도가 외부 게임 플레이량에 종속. 독립적 플레이 불가
- **깊이의 한계**: 장기 플레이 시 "연구 트리 다 올리고 나면 끝"이라는 콘텐츠 상한이 명확
- **불분명한 실패 조건**: "게임 오버"가 없어 실패감이 낮지만, 동시에 긴장감도 낮음
- **초기 버그**: 출시 첫날 로딩 오류로 서비스 다운 — 신뢰도 타격
- **순수 전략 깊이 부재**: 배치 선택지는 있지만 카운터 전략, 시너지 연구 등 고급 레이어가 없어 코어 전략 플레이어에게 피상적으로 느껴짐
- **반복성**: 정착지 구조가 모두 유사해 후반부 건설이 단조로움

## 6. 캐주얼함의 원천 분석

Project_Sun 설계에서 가장 중요한 질문: **"무엇이 이 게임을 캐주얼하게 만드는가?"**

| 캐주얼 요소 | 구현 방식 | Project_Sun 적용 여부 |
|---|---|---|
| **조작 복잡도 제거** | 전투 완전 자동 — 클릭 한 번으로 전투 해결 | 영감 (Project_Sun은 가벼운 RTS 마이크로 채택하되, 분대 단위 2종 명령으로 조작 복잡도를 최소화) |
| **실패 없는 실패** | 전투 패배해도 게임 오버 없음, 일부 자원 손실만 | 변형 — Project_Sun은 기지 붕괴 있지만 재도전 구조로 완화 |
| **세션 길이 자유** | 1턴만 하고 나갈 수도 있음. 자동 저장 완비 | 직접 적용 (턴제 + 세션 중단 가능) |
| **정보 완전 공개** | 건물 효과, 적 병력 정찰, 위협 사전 통보 | 직접 적용 |
| **명확한 이정표** | 챕터별 챔피언 해금 — 목표 항상 명확 | 변형 적용 (기지별 클리어 목표) |
| **시각적 성장 피드백** | 정착지가 확장되고 지도가 채워지는 시각적 변화 | 직접 적용 (기지 건설 시각 피드백) |
| **낮은 진입 장벽** | IP 친밀감 + 클라이언트 내 즉시 접근 | 영감 (Steam 첫 10분 경험 설계) |
| **서술 동반** | 매 챕터마다 챔피언 대사와 서사가 진행 동반 | 변형 적용 (행정관 캐릭터 서사) |

| 건물 완성 시 서사 대사 | 변형 | P1 | 특수 건물 완성 시 캐릭터 대사 트리거. 몰입감과 이정표 결합 |

## 8. Project_Sun 포지셔닝 대비 분석

| 비교 항목 | 떠오르는 데마시아 | Project_Sun (목표) |
|---|---|---|
| 장르 | 클라이언트 내 웹 메타게임 (이벤트 한정) | 독립 PC 전략 게임 (Steam) |
| 플레이 시간 | 이벤트 기간 수개월, 일일 5~15분 | 기지당 1~2시간 |
| 세션 구조 | 무제한 (은방패 제한) | 턴제 기지 1회 플레이 |
| 전투 방식 | 완전 자동 | 타워디펜스 + 가벼운 RTS 마이크로 (낮=턴제, 밤=실시간+일시정지/배속 가능) |
| 건설 | 고정 슬롯 정착지 | 고정 슬롯 기지 |
| 실패 조건 | 사실상 없음 (자원 손실만) | 기지 함락 (재도전 가능) |
| 내러티브 | IP 기반 서사 (Lux 교육) | 오리지널 옴니버스 서사 |
| 탐사 | 영토 타일 확장 | 노드 맵 원정대 |
| 연구 | 색상 구분 4카테고리 트리 | 내정/탐사/방어 3축 트리 |
| 캐주얼 깊이 | 낮음 (코어 부재) | 중간 (캐주얼 + 전략 심화 공존) |

**핵심 통찰**: 떠오르는 데마시아는 전투 자동화와 정보 공개를 통해 캐주얼 접근성을 확보하는 데 성공했으나, 전략적 깊이는 의도적으로 낮게 유지했다. Project_Sun은 이 캐주얼 UX 레이어를 기반으로, 탐사-내정-방어 3기둥 연동과 인력 관리 시스템을 통해 코어 레이어를 추가해야 한다. 즉, 데마시아 라이징의 "껍데기 캐주얼함"이 아닌 "접근 가능한 전략성"을 목표로 삼는 것이 차별화 방향이다.
