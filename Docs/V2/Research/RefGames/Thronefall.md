## 2. 핵심 루프

```
[맵 선택] → 각 맵은 고정된 슬롯 레이아웃 + 적 스폰 방향 고정
      ↓
[낮 페이즈 — 건설 & 경제]
  ├─ 고정 슬롯에 건물 배치 (순서만 선택 가능, 위치 선택 불가)
  ├─ 경제 건물 (농장, 광산, 어촌 등): 골드 생산
  ├─ 방어 건물 (병영, 타워 등): 야간 전력
  ├─ 업그레이드 선택: 농장 → 대규모 농장 or 시장 (분기)
  └─ 군주(Monarch) 직접 이동: 건설 속도 보너스, 전투 참여
      ↓
[밤 페이즈 — 자동 방어]
  ├─ 적 웨이브 진입 (고정 경로 + 다방향)
  ├─ 배치된 방어 건물 자동 공격
  ├─ 군주 직접 조작으로 긴급 지원 가능
  └─ 생존/실패 판정
      ↓
[라운드 보상] → 다음 낮 시작 시 골드 + 업그레이드 포인트 획득
      ↓
[반복 — 점점 강해지는 밤 웨이브]
```

**핵심 구조**: "낮=계획/건설, 밤=검증/자동방어"의 명확한 이원 구조. Project_Sun의 핵심 루프와 가장 유사한 원형.

## 3. 시스템 분석

### 3.1 고정 슬롯 건설 시스템

- **메커니즘**: 맵마다 미리 정해진 위치에만 건물을 배치할 수 있다. 플레이어가 결정하는 것은 "무엇을 먼저 지을 것인가"(순서)와 "어떤 업그레이드를 선택할 것인가"(분기)뿐. 자유 배치가 없어 인지 부하를 극도로 낮춤.
- **수치 데이터**:
  - 맵당 건물 슬롯: 약 8~15개 (맵에 따라 상이) [추정 — 커뮤니티 플레이 기반]
  - 건물 종류: 경제 (~5종) + 방어 (~6종) + 특수 (~3종) = 총 ~14종 [위키 — Steam 게임 가이드 기반]
  - 업그레이드 분기: 각 건물당 2~3가지 업그레이드 경로 [위키]
  - 골드 생산: 기본 가정(House) 1기당 ~2골드/주기 [추정 — 커뮤니티 추정, 밸런스 직접 사용 금지]
  - 업그레이드 비용: 기본 건물 50~150골드, 업그레이드 100~300골드 범위 [추정 — 커뮤니티 추정, 밸런스 직접 사용 금지]
- **플레이어 경험**: "어떤 순서로 지을 것인가"의 선택만 남겨두어 전략적 의사결정이 명확해짐. "뭘 어디에 놓을지 고민"하는 피로 없이 "뭘 먼저 지을 것인가"에만 집중.
- **우리 게임 적용**: 직접 — Vision.md 건설 시스템 "Thronefall식 고정 슬롯"이 명시적으로 레퍼런스로 지정됨. Project_Sun의 핵심 건설 메커니즘의 원형.

### 3.2 낮/밤 사이클 루프

- **메커니즘**: 낮 페이즈에서 제한된 골드로 건물을 배치/업그레이드하고, 밤 페이즈에서는 자동 방어가 실행된다. 군주(플레이어 아바타)는 밤에도 직접 이동하며 전투에 참여 가능하지만, 방어의 핵심은 배치된 건물.
- **수치 데이터**:
  - 낮 페이즈 길이: 자유 시간 (플레이어가 "시작" 버튼으로 밤 페이즈 돌입) [공식 — 게임 내 메커니즘]
  - 밤 페이즈 길이: 웨이브 격퇴 시 자동 종료 [공식]
  - 웨이브 라운드 수: 맵에 따라 7~10라운드 (마지막 라운드가 보스급 규모) [위키]
  - 적 규모 스케일링: 라운드마다 약 1.3~1.5배 증가 [추정 — 커뮤니티 추정, 밸런스 직접 사용 금지]
- **플레이어 경험**: "낮의 결정이 밤에 검증된다"는 명확한 인과관계. 밤 페이즈는 "내 준비가 충분했는가"의 긴장감 + 시각적 카타르시스.
- **우리 게임 적용**: 직접 — Project_Sun 마이크로 루프의 가장 직접적인 원형. "낮=내정/탐사, 밤=자동방어"는 Thronefall에서 검증된 구조.

### 3.3 경제/자원 시스템

- **메커니즘**: 단일 자원(골드)으로 모든 건설과 업그레이드를 처리. 경제 건물이 매 라운드 골드를 생산하며, 방어 건물은 지속적으로 전투력을 제공. "경제 vs 방어" 투자 배분이 핵심 전략 결정.
- **수치 데이터**:
  - 자원 종류: 1종 (골드) [공식]
  - 골드 생산 건물 예시: House +2/주기, Farm +5/주기, Mine +8/주기 [추정 — 대략적 수치, 맵 변수 있음, 밸런스 직접 사용 금지]
  - 방어 건물 예시: Barracks(병사 생산), Archer Tower(원거리 공격), Wall(장벽) [위키]
  - 업그레이드 ROI: 경제 업그레이드의 골드 회수 기간은 약 3~5라운드 [추론 — 분석가 계산, 밸런스 직접 사용 금지]
- **플레이어 경험**: 단일 자원이 의사결정을 명확하게 만든다. "경제에 투자해 나중에 강해질 것인가, 지금 당장 방어에 투자할 것인가"가 매 라운드의 핵심 질문.
- **우리 게임 적용**: 영감 — Project_Sun은 자원 3종(Vision.md)으로 Thronefall보다 복잡. 단, Thronefall의 "자원 최소화로 의사결정 명확화" 원칙은 자원 설계 시 반드시 참고해야 함. 3종 이상으로 가지 말 것.

### 3.4 미니멀리즘 복잡도 계층 설계 (시스템 메타)

- **메커니즘**: 게임 개발자 인터뷰(GDC/GameDeveloper.com 2024)에서 Jonas Tyroller/Paul Schnepf가 명시한 설계 철학: "기존 전략 게임에서 핵심 재미 이외의 모든 것을 제거한다." 구체적으로는 "플레이어가 선택하는 것의 수를 최소화하되, 남은 선택의 무게를 최대화"하는 방식.
  - 제거한 것: 건물 위치 선택 → 고정 슬롯
  - 제거한 것: 복수 자원 → 단일 골드
  - 남긴 것: 건설 순서, 업그레이드 분기, 군주 이동 타이밍
- **수치 데이터**: 출시 후 96% 긍정 리뷰 (Steam, ~22,200건) [공식 — Steam 스토어]. EA 14개월 후 정식 출시 [공식]. 총 판매 500K+ 추정 [추정 — EA 기간 315,000+ 공개 기준, 밸런스 직접 사용 금지].
- **플레이어 경험**: "복잡하지 않은데 깊이가 있다"는 리뷰 다수. "10분 안에 배울 수 있는 게임인데 숙달까지는 수십 시간" — 진입 장벽 최소화 + 마스터리 존재.
- **우리 게임 적용**: 직접 — Project_Sun의 진입 장벽 설계 원칙. Vision.md "수박 껍질 이론"의 직접 구현 사례. 특히 건설 시스템의 "고정 슬롯"과 자원 "3종 이하" 원칙의 근거.

## 4. UX/UI 패턴

- **건설 피드백**: 건물 배치 시 스케일 팝 애니메이션 + 파티클 효과 + 명확한 사운드 피드백. "배치하는 행위 자체가 보상감"을 줌.
- **경로 미리보기**: 밤 페이즈 시작 전 적의 이동 경로가 지형에 표시되어 방어 배치를 사전 계획할 수 있음. (Vision.md 웨이브 방어 시스템에 "Thronefall 경로 미리보기" 명시)
- **단순 UI**: 골드 수치 + 웨이브 카운터 + 건물 선택 패널. 화면에 표시되는 정보가 극도로 적음.
- **군주 조작**: 탑뷰 + 직접 이동. 건설 시 군주가 해당 위치로 이동해 건설 속도 향상 → 플레이어의 직접 개입감 유지.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **즉각적 진입**: "5분 만에 게임을 이해할 수 있다"는 리뷰 압도적
- **높은 리뷰 점수 유지**: 96% — 복잡한 게임에 비해 이탈 없이 높은 만족도
- **깔끔한 비주얼 & 피드백**: 미니멀하지만 건설/방어 피드백이 만족스러움
- **짧은 세션 길이**: 맵 클리어에 20~40분, 짧은 여가 시간에도 플레이 가능
- **낮/밤 루프의 명확성**: 무엇을 해야 하는지 매 순간 명확함

### 약점 (자주 비판받는 점)
- **콘텐츠 볼륨 부족**: 맵 수가 적고 클리어 후 재도전 동기가 약함 (Steam 부정 리뷰 최빈 키워드)
- **내러티브 거의 없음**: 세계관/스토리 요소가 극히 빈약
- **반복성**: 같은 맵을 여러 번 하면 패턴이 반복됨
- **긴 세션 유인 약함**: "한 판 더"의 중독성이 Against the Storm에 비해 낮다는 평가
