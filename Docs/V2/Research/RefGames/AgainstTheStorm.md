## 2. 핵심 루프

```
[메타 진행 — 왕국 트리 (Citadel)]
  ├─ 개별 런 성과 → 왕국 포인트 획득
  ├─ 왕국 포인트 → 영구 해금 (신규 건물, 퍽, 이벤트 등)
  └─ 왕도 등급 상승 → 새 런에서 더 많은 선택지
      ↓
[런 시작]
  ├─ 랜덤 지형 생성 (바이옴 6종)
  ├─ 랜덤 종족 조합 (인간/비버/도마뱀/기타)
  ├─ 초기 블루프린트 선택 (랜덤 3개 중 선택)
  └─ 목표 할당 (여왕의 임무)
      ↓
[도시건설 & 경제 운영]
  ├─ 자원 채취 (벌목꾼, 채석장 등)
  ├─ 생산 체인 구성 (원자재 → 중간재 → 완성품)
  ├─ 종족별 요구 충족 (식량 종류, 주거 타입 다름)
  ├─ 주민 사기(Resolve) 관리
  └─ 글레이드(Glade) 탐험 파견
      ↓
[업그레이드 카드 선택] ← 일정 시간마다 제시
  ├─ 3장의 랜덤 카드 중 1장 선택
  └─ 지형/자원/종족 조합에 최적화된 선택이 핵심
      ↓
[목표 달성 또는 Blightstorm 도달]
  ├─ 달성: 왕국 포인트 → 다음 런
  └─ 실패: 부분 보상 → 메타 진행 후 재도전
```

**핵심 구조**: "단기 도시건설 + 장기 메타 성장"의 이중 루프. 각 런이 짧고 완결되지만 메타 진행이 장기 리텐션을 보장.

## 3. 시스템 분석

### 3.1 로그라이크 메타 진행 시스템

- **메커니즘**: 각 런은 독립적인 정착지 건설 세션. 런 클리어 시 왕국 포인트를 획득하여 왕도(Citadel) 트리에서 영구 해금. 해금 항목은 새 건물 블루프린트, 새 이벤트, 새 퍽, 신규 종족 등. 런 실패 시에도 부분 포인트 획득.
- **수치 데이터**:
  - 왕도 트리 노드: ~100+ 개 [위키 — ATS Fandom Wiki]
  - 런당 소요 시간: 초보 ~60~120분, 숙련 ~30~60분 [추정 — Steam 리뷰/커뮤니티 기반]
  - 프레스티지(난이도) 레벨: 0~20단계 [공식 — 게임 내 시스템]
  - 바이옴 종류: 6개 (숲, 습지, 설원, 사막 등) [공식]
  - 종족 조합: 매 런 다름 (인간 필수 + 2종 랜덤) [공식]
- **플레이어 경험**: "이번 런에서 배운 것을 다음 런에 적용"하는 학습 루프. 각 런이 짧아 "한 판 더" 동기 강함. 실패해도 "무언가는 해금됨"으로 좌절 최소화.
- **우리 게임 적용**: 영감 — Project_Sun의 옴니버스 기지 메타구조와 개념적 유사성. 각 기지 세션이 독립적이면서 전체 서사로 연결되는 구조가 Against the Storm의 "런 독립 + 메타 연결" 구조와 같은 원리.

### 3.2 종족별 요구 관리 시스템 (Resolve)

- **메커니즘**: 인간/비버/도마뱀/기타 종족이 서로 다른 식량, 주거, 서비스 요구 사항을 가짐. 이 요구를 충족시키면 사기(Resolve) 상승, 미충족 시 하락. Resolve가 임계값 이하로 떨어지면 주민 이탈 → 경제 붕괴.
- **수치 데이터**:
  - 사기 임계값: 하락 시 경고 → 계속 하락 시 런 실패 [공식]
  - 식량 종류: 고기, 생선, 빵, 채소, 곤충 등 ~8~10종 [위키]
  - 종족별 선호 식량: 도마뱀 = 고기 선호, 비버 = 채소 선호 등 [공식 — 게임 내 메커니즘]
  - 서비스 건물: 선술집, 사원, 교육기관 등 → 종족별 효과 상이 [위키]
- **플레이어 경험**: "같은 자원으로 다른 종족을 동시에 만족시키는" 최적화 퍼즐. 매 런마다 다른 종족 조합으로 다른 전략이 필요.
- **우리 게임 적용**: 영감 — Project_Sun의 인력 관리 시스템에서 "시민마다 다른 능력치/적성"으로 유사한 개인화 요소 구현. 단, 종족 시스템보다 개인 시민 단위의 차별화가 더 Project_Sun의 방향.

### 3.3 글레이드 탐험 및 이벤트 시스템

- **메커니즘**: 맵 위의 미탐험 지역(Glade)을 개방하면 자원 또는 이벤트 발생. 이벤트는 선택지 형태로 제시되며, 잘못된 선택은 해당 Glade가 위험 상태로 변해 지속 피해를 줌. 탐험 자원(연료)을 소모하여 Glade 제거 가능.
- **수치 데이터**:
  - Glade 종류: 일반(자원), 수상(보상/위험), 봉인(고위험 고보상) [공식]
  - 이벤트 선택지: 대부분 2~3가지 옵션 [위키]
  - 탐험 연료: Glade 개방에 소모 (자원 투자 필요) [공식]
  - 봉인된 Glade 이벤트 실패 시: 해당 구역이 지속 Resolve 페널티 적용 [위키]
- **플레이어 경험**: "탐험하면 보상이 있지만 위험도 있다"는 리스크-보상 구조. 탐험 타이밍이 전략적 의사결정.
- **우리 게임 적용**: 영감 — Project_Sun 탐사/원정 시스템의 "노드 탐험 → 이벤트/보상" 구조와 유사. 특히 "잘못된 선택 → 지속 페널티" 구조는 인카운터 이벤트 설계에 참고.

### 3.4 업그레이드 카드(Cornerstone) 시스템

- **메커니즘**: 정착지 레벨업 시 3장의 카드(Cornerstone) 중 1장 선택. 카드는 생산량 증가, 특수 건물 해금, 비용 감소, 새로운 메커니즘 추가 등 다양. 매 런 랜덤 제시로 다른 전략 강제.
- **수치 데이터**:
  - 카드 풀: 100+ 종 이상 [위키]
  - 선택 횟수: 런당 ~5~10회 (난이도/런 길이에 따라 상이) [추정 — 커뮤니티 기반]
  - 제시 방식: 3장 중 1장 선택 (때로 재굴림 기회 제공) [공식]
- **플레이어 경험**: "이번 런의 자원 조합에 가장 맞는 카드는 무엇인가"의 적응적 전략 수립. 덱빌딩의 재미를 도시건설에 접목.
- **우리 게임 적용**: 영감 — Project_Sun의 기술 트리 시스템에서 "탐사 결과로 다른 연구 방향 결정"하는 구조가 유사. 단, Project_Sun은 기지 고유 트리로 고정이므로 Against the Storm처럼 완전 랜덤하지 않음.

## 4. UX/UI 패턴

- **생산 체인 시각화**: 건물 클릭 시 해당 건물의 입력/출력 자원이 연결선으로 표시. 생산 체인을 직관적으로 파악.
- **목표 진행 표시**: 화면 우측에 현재 런의 목표와 달성도 상시 표시. "무엇을 해야 하는가"가 항상 명확.
- **사기(Resolve) 바**: 종족별 Resolve 수치가 독립 표시. 어느 종족이 불만인지 즉시 확인 가능.
- **Blightstorm 카운트다운**: 런 종료 시간이 남은 시간으로 표시. 압박감을 가시화.
- **왕도 트리 UI**: 해금 가능 항목이 나무 형태로 표시. 다음 목표가 명확해 장기 플레이 동기 유지.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **장르 혼합의 완성도**: "로그라이크 + 도시건설이 이렇게 잘 맞는지 몰랐다"는 반응 다수 (95% 긍정 리뷰 근거)
- **단기 런의 집중도**: 긴 도시건설 게임의 "후반부 지루함"이 없음
- **높은 리플레이 가치**: 매 런마다 다른 종족/지형/카드로 다른 경험
- **개발사의 꾸준한 업데이트**: EA부터 정식까지 커뮤니티 피드백 반영 업데이트 지속
- **분위기와 사운드**: 차분하고 아늑한 음악 + 아트 스타일로 "편안한 스트레스"

### 약점 (자주 비판받는 점)
- **진입 장벽**: 시스템이 많아 "처음 10~15시간은 뭐가 뭔지 모른다"는 리뷰 다수
- **반복성**: 100시간 이상 플레이 시 런의 패턴이 유사해지는 느낌
- **전투/방어 요소 부재**: 타워디펜스 팬에게는 방어 전투가 없어 아쉬움
- **느린 초반**: 생산 체인이 안정화되기까지 초반 플레이가 다소 느리게 느껴짐

## 6. 적용 포인트

| 요소 | 적용 방식 | 우선순위 |
|---|---|---|
| 옴니버스 메타 진행 구조 | 영감 — 기지별 세션 독립 + 시대 전환 연결 구조의 원형 | P1 |
| 업그레이드 카드 랜덤 선택 | 영감 — 기술 트리에 랜덤성 부여 시 참고 | P2 |
| 글레이드 탐험 이벤트 | 영감 — 탐사 이벤트의 "리스크-보상 구조" 설계에 참고 | P1 |
| 런 실패 시 부분 보상 | 영감 — ATS의 "부분 보상 구조"라는 설계 철학만 참고. Project_Sun의 실패 보상 구조는 별도 설계 | P1 |
| 진입 장벽 반면교사 | 영감 — ATS의 "10~15시간 진입 장벽" 원인 분석: 생산 체인 복잡성 + 종족 요구 동시 학습 + UI 정보 과부하의 복합. Project_Sun은 탭 점진적 해금으로 이 원인들을 순차적으로 노출하여 해결 | P0 |
| 종족별 요구 → 시민별 능력치 | 변형 — 종족 단위 대신 개인 시민 단위의 능력치 차별화로 변형 | P1 |
| 목표 달성형 런 구조 | 영감 — 각 기지 세션에 명확한 클리어 조건(=여왕 임무 구조) 부여 | P1 |
