## 2. 핵심 루프

```
[메타 진행 — Haven (기지) 강화]
  ├─ 각 런 클리어 → 영구 해금 포인트
  ├─ 새 영웅, 스킬, 건물, 마법 해금
  └─ 난이도 조정 (Magebreaker 등급)
      ↓
[런 시작 — Haven 선택]
      ↓
[낮 페이즈 — 생산(Production) 페이즈]
  ├─ 건물 건설/업그레이드 (목재/금속/식량 소모)
  ├─ 부상 영웅 치료
  ├─ 방어 구조물 수리/강화
  ├─ 기술 연구
  └─ 상인 거래
      ↓
[밤 페이즈 — 방어(Combat) 페이즈]
  ├─ 적 웨이브 진입 (다방향, 물량)
  ├─ 영웅 3명의 턴제 행동 (이동 + 스킬 사용)
  ├─ 방어 구조물 자동 공격
  ├─ 행동 포인트(AP) 소모 → 자원 재사용 대기
  └─ 모든 적 격퇴 시 낮 페이즈로 복귀
      ↓
[Day N 도달 → 최종 결말]
  ├─ 마법 완성 → 세계 정화 → 클리어
  └─ Haven 함락 → 런 실패
```

**핵심 구조**: "낮=기지 강화, 밤=턴제 전투". Project_Sun과 장르 구조가 가장 유사하지만, 영웅 3명 중심의 전술 RPG vs Project_Sun의 기지 중심 대규모 자동전투라는 차별점이 있다.

## 3. 시스템 분석

### 3.1 턴제 전투 시스템 (영웅 기반)

- **메커니즘**: 밤 페이즈에서 영웅 3명을 턴제로 조작. 각 영웅은 행동 포인트(AP)를 소모하여 이동, 공격, 스킬 사용. 적은 영웅 턴 종료 후 자동으로 이동/공격. 턴 순서는 영웅 → 방어 구조물 → 적 순서.
- **수치 데이터**:
  - 영웅 수: 런당 3명 (메타 진행으로 최대 확장 가능) [공식]
  - AP: 영웅당 기본 3~5 AP/턴 [위키 — TLS Fandom Wiki]
  - 이동 비용: 1 타일 = 1 AP [공식]
  - 공격 비용: 스킬에 따라 1~4 AP [위키]
  - 적 종류: ~30~40종 (기본~보스) [위키]
  - 방어 구조물: 성벽, 함정, 포탑, 투석기 등 (~15~20종) [위키]
  - 웨이브 규모: 첫 날 밤 ~20~50마리 → 최종 밤 ~500~2,000마리 이상 [추정 — 커뮤니티 관찰, 밸런스 직접 사용 금지]
- **플레이어 경험**: "영웅 한 명 한 명의 포지셔닝과 스킬 사용이 전투의 핵심". 직접 조작하는 영웅이 있어 자동전투만 있는 Thronefall보다 더 강한 에이전시 감각.
- **우리 게임 적용**: 영감 후 전환 — Project_Sun은 영웅 3명 중심이 아닌 기지 전체 자동전투. The Last Spell의 전투 직접 조작이 제공하는 에이전시를 우리 게임에서는 "낮 페이즈의 준비 결정"으로 대체해야 함.

### 3.2 낮 페이즈 건설/업그레이드 시스템

- **메커니즘**: 낮 페이즈(생산 페이즈)에서 건물 건설 및 기존 건물 업그레이드. 목재/금속/식량 3종 자원 소모. 부상 영웅 치료, 방어 구조물 수리. 상인에게 자원 거래 가능. 연구소에서 기술 연구 진행.
- **수치 데이터**:
  - 자원 종류: 목재, 금속, 식량 (3종) + 영웅 경험치 [공식]
  - 건물 종류: 생산 건물(~8종) + 방어 구조물(~15~20종) + 지원 건물(~5종) [위키]
  - 건물 건설 비용: 목재 10~80 + 금속 0~60 범위 [위키]
  - 업그레이드 비용: 건설 비용의 ~50~100% 추가 [추정 — 커뮤니티 기반, 밸런스 직접 사용 금지]
  - 건물 수리 비용: 파손된 건물 복구 = 건설 비용의 ~30~50% [추정 — 커뮤니티 기반, 밸런스 직접 사용 금지]
- **플레이어 경험**: "오늘 밤 어떤 적이 몇 마리 오는가"를 기준으로 무엇을 건설/업그레이드할지 결정. 직전 밤의 결과가 다음 낮의 우선순위를 결정.
- **우리 게임 적용**: 직접 — Project_Sun의 낮 페이즈 건설 시스템과 구조적으로 동일. 자원 3종 구성(목재/금속/식량 → Project_Sun의 3종 자원)도 참고 가능.

### 3.3 영웅 RPG 성장 시스템

- **메커니즘**: 각 런에서 영웅이 전투 경험으로 레벨업. 레벨업 시 랜덤 2세트의 스탯 중 하나 선택. 스탯: 힘/민첩/지능/생명력 등. 스킬 슬롯이 확장되며 스킬 조합으로 특화 빌드 구성. 런 종료 시 영웅 능력치 초기화(로그라이크 구조).
- **수치 데이터**:
  - 영웅 레벨: 런당 최대 ~15~20레벨 달성 가능 [위키]
  - 레벨업 선택: 2세트의 랜덤 스탯 중 1세트 선택 [공식]
  - 스킬 종류: 영웅당 ~30~50종 스킬 풀 [위키]
  - 스킬 슬롯: 레벨업으로 확장, 최대 ~8~10개 [위키]
  - 메타 해금: 런 성과로 영구 새 스킬/영웅 해금 [공식]
- **플레이어 경험**: 매 런 다른 빌드를 시도할 수 있는 덱빌딩 감각. "이 영웅을 근거리 탱커로 키울 것인가, 원거리 딜러로 키울 것인가"의 선택.
- **우리 게임 적용**: 변형 — Project_Sun의 인력 관리에서 "시민 능력치 성장" 및 "행정관 능력 단계"로 변형. 영웅 중심 RPG 성장보다 집단(기지 전체) 성장으로 스케일 전환.

### 3.4 로그라이크 메타 진행 (Haven 영구 강화)

- **메커니즘**: 런 성과에 따라 Haven 영구 개선 포인트 획득. 포인트로 새로운 건물 블루프린트, 영웅 스타팅 장비, 마법 성능 강화 등 영구 해금. 난이도 Magebreaker 등급을 통해 스스로 도전 강도 조절 가능.
- **수치 데이터**:
  - 메타 진행 포인트: 런 클리어 시 규모에 따라 100~500+ 포인트 획득 [추정 — 커뮤니티 기반, 밸런스 직접 사용 금지]
  - 영구 해금 항목: 수십~100+ 종 [위키]
  - Magebreaker 등급: 0~최대 (높을수록 적 강화, 보상도 증가) [공식]
- **플레이어 경험**: "이번 런은 특정 빌드를 해금하기 위한 도전"이라는 목적 의식으로 실패해도 동기 유지.
- **우리 게임 적용**: 영감 — Project_Sun의 시대 전환 메타구조 및 기지별 고유 기술 트리 설계 시 참고. 단, Project_Sun은 영구 진행이 아닌 서사적 연결로 차별화.

## 4. UX/UI 패턴

- **전술 그리드**: 타일 기반 전투 맵. 영웅 이동 가능 범위 하이라이트, 스킬 공격 범위 프리뷰.
- **낮/밤 전환 UI**: 낮 페이즈 종료 버튼 클릭 시 밤 전환. 진입 전 "이번 밤 적 구성 미리보기" 가능 (탐사 수준에 따라 정보 제공).
- **영웅 상태 패널**: 화면 하단에 영웅 3명의 HP/AP/스킬 쿨다운 상시 표시.
- **건물 건설 UI**: 낮 페이즈에 건설 가능 위치가 하이라이트. 자원 부족 건물은 회색 처리.
- **메탈 사운드트랙**: 25+ 트랙의 헤비메탈 배경음악이 밤 전투의 박진감을 고조.

## 5. 강점/약점 (커뮤니티 기반)

### 강점 (자주 칭찬받는 점)
- **턴제+기지방어 혼합 성공**: "이 장르 조합이 이렇게 재미있을 줄 몰랐다"는 반응 (92% 긍정)
- **영웅 빌드 다양성**: 매 런 다른 빌드로 신선함 유지
- **직접 조작의 에이전시**: 내가 직접 영웅을 움직이는 만족감
- **헤비메탈 사운드트랙**: 독특한 음악이 분위기를 크게 강화

### 약점 (자주 비판받는 점)
- **기지 경영의 깊이 부족**: "영웅 RPG에 기지방어가 얹힌 것"으로 기지 경영 시스템이 단순하다는 평가
- **물량 스케일 한계**: 영웅 3명 중심이라 TAB처럼 수천 마리의 물량 카타르시스는 없음
- **세션 길이 길음**: 한 런이 수 시간에 달해 짧은 세션 플레이 불가
- **메타 진행 그라인드**: 영구 해금을 위한 반복 플레이가 과도한 그라인드로 느껴지는 경우
- **튜토리얼 부재**: 시스템이 많지만 설명이 부족해 초반 학습 부담

## 6. 적용 포인트

| 요소 | 적용 방식 | 우선순위 |
|---|---|---|
| 낮/밤 페이즈 이원 구조 | 직접 — Project_Sun 마이크로 루프의 직접 선례. "낮=준비, 밤=방어" 구조 검증 사례 | P0 |
| 자원 3종 (목재/금속/식량) | 직접 참고 — Project_Sun 자원 3종 설계 시 TLS의 3종 구성을 벤치마크 | P0 |
| 밤 시작 전 적 구성 미리보기 | 직접 — 탐사 수준에 따라 적 구성 정보 제공량 조절 | P1 |
| 건물 수리/복구 시스템 | 직접 — 방어 실패 시 건물 피해 → 낮에 수리 필요 구조 | P1 |
| 영웅 RPG → 행정관 능력 | 변형 — 영웅 개인 성장을 행정관 능력 단계 시스템으로 변형 | P1 |
| 메타 진행 그라인드 반면교사 | 직접 — 과도한 메타 그라인드 없이 서사적 연결로 리텐션 확보 | P1 |
| "기지경영 깊이 부족" 반면교사 | 영감 — TLS의 기지 경영이 왜 깊이가 부족한가: (1) 자원 3종이 영웅 AP로 대부분 해결되어 자원 배분 압박이 낮음, (2) 건물 건설이 낮 페이즈의 부수적 활동이지 주 전략이 아님, (3) 방어 구조물이 영웅 포지셔닝에 종속되어 독립적 설계 불필요. Project_Sun은 이 3가지 원인을 모두 역전: (1) 자원 3종이 인력 배분과 독립적으로 압박 생성, (2) 건설/인력 배치가 낮의 핵심 활동, (3) 방어 구조물이 완전 자동이므로 배치 자체가 핵심 전략 | P0 |
