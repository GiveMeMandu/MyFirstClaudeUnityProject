1. 리소스  등록
1.1 리소스를  등록하는  과정
아래의  절차는  " 문제  없이 " 리소스를  등록할  수  있는  방법임 , 가능한  빠르고  정확하게  등록해  주어야  함
해당 Step이  개선될  수  있는지  구현  문의  필요함
사유 : 모델링 / 기능  구현  폴리싱을  위해  빠르게  보는  것이  필요하며 , 한번에  리소스를  등록하기  위해  모아두는  경우  이후  다른  작업자의  작업물과  충돌하기도  함
충돌 사례
실제 사례  1
1. A작업자가  의뢰한  리소스  " 코코 " 가  완성됨  → A 작업자가  등록하는  것이  옳으나  A 작업자는  다른  작업을  먼저  하기  위해  등록을  미룸
2. B작업자  역시  해당  리소스  " 코코 " 가  필요한  상황  → B 작업자가  대신  등록함
3. 이후 맥락을  모르는  A 작업자가  다시  등록하여  리소스  " 코코 " 가  복사됨
4. 해당 맥락을  모르는  C 작업자가  두개의  리소스  " 코코 " 중  한개를  제거함
5. D작업자가  작업한  맵에  배치되어  있던  리소스 " 코코 " 가  사라지나  D 작업자는  해당  사실을  모름
6. 맵에서  사라졌다는  사실을  먼  훗날  알게  됨
실제 사례  2
1. A작업자가  의뢰한  리소스 " 코코 " 가  완성됨  → A 작업자가  등록을  미룸
2. B작업자가  일괄  등록하여  맵에  리소스 " 코코 " 배치까지  마침
3. 리소스 "코코 " 에서  버그가  발생하여  관련된  마데 , 구현  코드를  수정하고  리소스 " 코코 " 를  재배치함
4. A작업자가  리소스 " 코코 " 를  재  등록함
5. 마데는  통과했으나 , 맵툴  미  작동  or 해당  오브젝트는  이전의  버그  상태를  유지함
6. 작업자  B 는  원인을  찾아  모델러 / 프로그래머에게  방문함
등록 절차  : 일반적인  타일오브젝트를  등록  할  때
